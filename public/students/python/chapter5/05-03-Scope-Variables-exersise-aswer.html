SET A — Conceptual Questions (1–25)

1. Define variable scope in Python.
Scope means the region of a program where a variable can be accessed or used.
2. What is a local variable?
A variable declared inside a function, accessible only there.
3. What is a global variable?
Declared outside all functions and accessible anywhere.
4. Explain the LEGB rule.
Python looks for variables in Local → Enclosed → Global → Built-in order.
5. Why can't local variables be accessed outside the function?
Because they are destroyed once the function execution ends.
6. What happens if local and global variable names match?
Local variable shadows the global inside the function.
7. Define variable shadowing.
When a local variable hides a global variable of the same name.
8. Explain enclosed scope.
Variable in outer function accessible to inner function.
9. What is the lifetime of a local variable?
From function start to function end.
10. Lifetime of global variables?
Exists until program ends.
11. Why are global variables risky?
They cause unintended side effects across functions.
12. Does indentation affect scope?
Yes, indentation defines function blocks which create scope.
13. Explain why global scope is dangerous.
A single change can affect entire program behavior.
14. How does a nested function access outer variable?
Via enclosed scope (the E in LEGB).
15. Built‑in vs Global scope?
Built‑in contains Python defaults like print(); global contains user-defined variables.
16. What is namespace?
A container mapping variable names to values.
17. Side effects of globals?
Unexpected interactions between functions.
18. Why use return instead of modifying globals?
It ensures clean, reusable, and modular code.
19. Example where global is needed.
Game score, configuration flags, constants.
20. Does loop create new scope?
No, loop variables exist in same function scope.
21. Do if-blocks create scope?
No, Python does not create block scopes.
22. try-block scope?
Same as surrounding function.
23. Constants and scope?
Declared globally using uppercase names.
24. Scope vs Lifetime?
Scope = where accessible; Lifetime = duration in memory.
25. Recursion & scope?
Each recursive call has its own local scope.
