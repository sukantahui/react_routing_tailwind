Unit 1.1 — Basics of Programming

In this unit, you’ll clearly understand what a program is, how algorithms and flowcharts are used to design solutions, which steps are followed during problem solving, and how structured programming differs from object-oriented programming — with multiple simple, exam-focused examples.

Target: NIELIT O Level / Python Beginners Focus: Concept Clarity + Examples
1 What is a Program?

A program is the bridge between a real-world problem and the computer. It tells the computer exactly what to do, step by step.

Concept + Examples

Definition: A program is a finite, ordered set of instructions written using a programming language, which the computer can understand and execute to solve a specific problem.

Key idea: A program is not magic. It is only a collection of instructions like: “do this, then this, then that…”

1.1 Features of a Good Program

  • Correctness: Produces correct output for all valid inputs.
  • Clarity: Easy to read and understand (good variable names, comments).
  • Efficiency: Uses time and memory properly.
  • Maintainability: Easy to modify and update later.
  • Reusability: Common tasks are written as functions/modules.

1.2 Real Life View vs Program

Real-life “instructions” are all around us — cooking recipes, travel directions, exam question papers. A program is the same concept, but for a computer.

Example 1 – Simple Interest Program

Problem: Calculate Simple Interest given Principal (P), Rate (R), Time (T).

Logic: SI = (P × R × T) / 100

# Python-style program idea (for understanding) P = input("Enter Principal: ") R = input("Enter Rate: ") T = input("Enter Time in years: ") SI = (P * R * T) / 100 print("Simple Interest =", SI)
Example 2 – Student Passed or Failed

Problem: Read marks and decide whether the student passed (≥ 40) or failed.

1. Input marks 2. If marks ≥ 40 Print "Pass" Else Print "Fail"
Example 3 – Even or Odd

Problem: Check if a number is even or odd.

1. Input number N 2. If N % 2 == 0 Print "Even" Else Print "Odd"
2 Algorithm — Step-by-step Solution

Before writing any program, we design an algorithm — a clear list of steps to solve the problem.

Logic Design

Definition: An algorithm is a finite sequence of well-defined steps that solve a particular problem.

2.1 Characteristics of a Good Algorithm

  • Input: Clearly mentions required inputs.
  • Output: Clearly mentions expected output(s).
  • Definiteness: Every step is clear and unambiguous.
  • Finiteness: Algorithm must end after a finite number of steps.
  • Effectiveness: Each step is simple and can actually be done.

2.2 Algorithm Example 1 — Sum of Two Numbers

Algorithm: Sum of Two Numbers
Step 1: Start Step 2: Input A Step 3: Input B Step 4: SUM ← A + B Step 5: Print SUM Step 6: Stop

2.3 Algorithm Example 2 — Largest of Three Numbers

Algorithm: Largest of Three Numbers (A, B, C)
Step 1: Start Step 2: Input A, B, C Step 3: If A ≥ B and A ≥ C, then Largest ← A Else if B ≥ A and B ≥ C, then Largest ← B Else Largest ← C Step 4: Print Largest Step 5: Stop

2.4 Algorithm Example 3 — Factorial of a Number N

Algorithm: Factorial (N!)
Step 1: Start Step 2: Input N Step 3: FACT ← 1 Step 4: For i from 1 to N do FACT ← FACT × i Step 5: Print FACT Step 6: Stop

2.5 Algorithm Example 4 — Check Prime Number

Algorithm: Prime Check
Step 1: Start Step 2: Input N Step 3: If N < 2 then Print "Not Prime", Stop Step 4: Set flag ← 1 Step 5: For i from 2 to N - 1 do If N mod i = 0 then flag ← 0 Break the loop Step 6: If flag = 1 then Print "Prime" Else Print "Not Prime" Step 7: Stop
3 Flowchart — Visual Representation of Algorithm

A flowchart turns an algorithm into a diagram using standard symbols. This makes the logic easy to see and debug.

Visual Logic

Definition: A flowchart is a graphical representation of the sequence of steps in an algorithm, using symbols connected by arrows.

3.1 Standard Flowchart Symbols

Symbol Name Shape (Description) Meaning
Terminator Oval / Rounded rectangle Start or End of the flowchart
Process Rectangle Computation / processing step
Input / Output Parallelogram Taking input or showing output
Decision Diamond Yes/No or True/False condition check
Arrow Line with arrow head Shows direction of flow

3.2 Flowchart Example 1 — Sum of Two Numbers (Text Description)

Flowchart (described)
  • Start → (Oval)
  • Input A, B → (Parallelogram)
  • SUM = A + B → (Rectangle)
  • Output SUM → (Parallelogram)
  • End → (Oval)

3.3 Flowchart Example 2 — Pass / Fail

Flowchart (described)
  • Start
  • Input MARKS
  • Decision: MARKS ≥ 40 ?
    • Yes → Print "PASS"
    • No → Print "FAIL"
  • End

3.4 Flowchart Example 3 — Factorial N!

Flowchart (described)
  • Start
  • Input N
  • Set FACT = 1, i = 1
  • Decision: i ≤ N ?
    • Yes:
      • FACT = FACT × i
      • i = i + 1
      • Go back to decision
    • No:
      • Print FACT
      • End
4 Standard Problem-Solving Steps in Programming

All successful programs follow a disciplined approach — understand, plan, design, code, test, and refine.

Development Process

4.1 General Steps

  1. Problem Definition: Clearly describe what needs to be solved.
  2. Analysis: Identify inputs, outputs, constraints, and required processing.
  3. Algorithm Design: Write step-by-step solution in plain language.
  4. Flowchart / Pseudocode: Draw visual or structured representation.
  5. Coding: Convert algorithm into a program (e.g., in Python).
  6. Testing: Run program with different test cases.
  7. Debugging: Find and correct errors (logic, syntax, runtime).
  8. Documentation & Maintenance: Write comments, user guide, and update program when needed.

4.2 Example — Student Result Processing

Problem: For a student, read marks of 3 subjects, calculate total, percentage and display “Pass/Fail” (Pass if percentage ≥ 40).

Step 1: Identify Inputs & Outputs

  • Inputs: Marks in Subject1, Subject2, Subject3.
  • Outputs: Total marks, Percentage, Result (“Pass” or “Fail”).

Step 2: Algorithm

Algorithm: Student Result
Step 1: Start Step 2: Input M1, M2, M3 Step 3: TOTAL ← M1 + M2 + M3 Step 4: PERCENT ← TOTAL / 3 Step 5: If PERCENT ≥ 40 then RESULT ← "PASS" Else RESULT ← "FAIL" Step 6: Print TOTAL, PERCENT, RESULT Step 7: Stop

Step 3: Simple Flowchart (Text Description)

  • Start
  • Input M1, M2, M3
  • Compute TOTAL, PERCENT
  • Decision: PERCENT ≥ 40 ? → Yes: “PASS” | No: “FAIL”
  • Output TOTAL, PERCENT, RESULT
  • End

Step 4: Rough Python Code (for later unit)

M1 = float(input("Enter marks in Subject 1: ")) M2 = float(input("Enter marks in Subject 2: ")) M3 = float(input("Enter marks in Subject 3: ")) TOTAL = M1 + M2 + M3 PERCENT = TOTAL / 3 if PERCENT >= 40: RESULT = "PASS" else: RESULT = "FAIL" print("Total =", TOTAL) print("Percentage =", PERCENT) print("Result =", RESULT)
5 Structured Programming vs Object-Oriented Programming

Both are ways to organize programs. Structured programming focuses on functions; OOP focuses on objects.

Programming Paradigms

5.1 Structured Programming

Structured programming is a programming style where the solution is divided into small parts called functions or procedures and uses control structures like sequence, selection, and iteration.

  • Follows a top-down approach.
  • Program is mainly a collection of functions.
  • Focus on how to solve the problem step by step.
  • Uses constructs like if, else, for, while.
Example – Structured Thinking (Pseudo-code)
main() { read marks result = calculate_result(marks) print result } calculate_result(marks) { if marks >= 40 then return "PASS" else return "FAIL" }

5.2 Object-Oriented Programming (OOP)

OOP organizes the program around objects that contain data (attributes) and functions (methods) together.

  • Follows a bottom-up approach.
  • Program is a collection of interacting objects.
  • Focus on who/what is acting (student, account, car, etc.).
  • Important concepts: Class, Object, Encapsulation, Inheritance, Polymorphism.
Example – Simple OOP View (Conceptual)
Class Student Attributes: name, roll, marks Methods: input_data() calculate_result() display_result() Object s1 is a Student Object s2 is a Student

5.3 Differences Between Structured and OOP

Aspect Structured Programming Object-Oriented Programming
Basic Unit Function / Procedure Object and Class
Approach Top-down Bottom-up
Data Handling Data is often global or shared among functions Data is bundled inside objects (Encapsulation)
Reusability Limited reuse via functions High reuse via inheritance and composition
Examples C, Pascal Python, Java, C++, C#
Important: Python supports both:
  • You can write simple, structured programs using functions only.
  • You can also use full OOP with classes and objects (covered in later units).
