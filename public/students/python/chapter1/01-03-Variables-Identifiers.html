1.3.1 Variables & the Memory Concept

A variable in Python is a named reference to a value stored in memory. When you write a statement like: 

# Example: creating a variable
age = 18
name = "Rahul"
pi   = 3.14159

Python internally creates objects for the values 18, "Rahul" and 3.14159 in memory, and the variable names age, name and pi act like labels or tags attached to those objects.

Conceptual Picture ‚Äî Box & Label

Imagine memory as a cupboard full of small boxes. Each box can store one value. A variable is the label you stick on a box so that you can easily find the value later.

  • The box = memory location / object.
  • The label = variable name (identifier).
  • The thing inside the box = the actual value (data).

Some more examples:

roll_no = 25          # integer value
marks   = 87.5         # floating‚Äëpoint value
is_pass = True         # boolean value
city    = "Kolkata"    # string value

In Python, you do not need to declare the type separately (like int, float in C). The type is decided automatically based on the value you assign. This is called dynamic typing.

1.3.2 Identifiers & Naming Rules

An identifier is the name you give to variables, functions, classes, modules, etc. Every variable name is therefore an identifier.

A. Rules for a Valid Identifier

  • It can contain: letters (a‚Äìz, A‚ÄìZ), digits (0‚Äì9) and underscore (_).
  • It cannot start with a digit.
  • No special symbols: @, #, $, %, space, etc.
  • Python is case‚Äësensitive: Age and age are different.
  • It must not be a Python keyword (like if, for, class).
Valid vs Invalid Examples
# ‚úÖ Valid identifiers
student_name
_roll_no
marks2025
PI_VALUE

# ‚ùå Invalid identifiers
2ndStudent   # starts with digit
student-name # minus sign not allowed
full name    # space not allowed
class        # keyword, cannot use as name

B. Naming Conventions (Best Practices)

While Python allows many styles, following standard naming conventions makes your code easy to read and good for exams and real projects.

  • Use meaningful names: 
    # Better
average_marks = 86.5

# Poor
x = 86.5
  • For variables and functions, use snake_case: 
    total_amount = 1500
student_count = 42
  • For constants, many programmers use UPPER_SNAKE_CASE: 
    PI = 3.14159
MAX_STUDENTS = 60
  • Never use single‚Äëletter names like a, b, x in big programs unless it‚Äôs a small loop or maths formula.

1.3.3 Reserved Keywords in Python

Keywords are special words that Python reserves for its own use. You cannot use them as variable names or identifiers.

Some commonly used Python keywords are:

if else elif for while break continue True False and or not in is None def class return import from

You can see the complete list inside Python using:

import keyword
print(keyword.kwlist)
Keyword Error Example
# ‚ùå This will cause an error
class = "BCA"   # 'class' is a keyword

# ‚úÖ Correct way
class_name = "BCA"

If you try to use a keyword as a variable name, Python will show a syntax error while running or even while typing in many IDEs.

1.3.4 Comments in Python

Comments are notes that you write in your code for explanation. Python ignores comments while executing the program. They are only for humans.

A. Single‚ÄëLine Comments

A single‚Äëline comment starts with the hash symbol #. Anything after # on that line is treated as a comment. 

# This program calculates the total marks
physics = 78   # marks in Physics
chemistry = 82 # marks in Chemistry
maths = 91     # marks in Mathematics

total = physics + chemistry + maths  # adding all marks
print("Total =", total)
  • Use single‚Äëline comments to explain why you are doing something, not obvious things like # add 1 to i.

B. Multi‚ÄëLine Comments / Block Comments

Python does not have a special multi‚Äëline comment symbol like some languages. The common practice is to use triple‚Äëquoted strings ''' ... ''' or """ ... """ as block comments. 

""" 
This is a multi‚Äëline comment.
You can write explanations, author name,
creation date, or algorithm steps here.
"""

print("Welcome to Coder & AccoTax Python tutorial")

When such a triple‚Äëquoted string is placed at the beginning of a function / class / module, it is called a docstring (documentation string). Python can read docstrings at runtime using help(). 

def add(a, b):
    """Return the sum of two numbers a and b."""
    return a + b

help(add)   # shows the docstring as function help
Good Commenting Practice
  • Write comments for logic, formulas, and non‚Äëobvious decisions.
  • Don‚Äôt repeat exactly what the code already clearly says.
  • Keep comments updated if you change the code.
