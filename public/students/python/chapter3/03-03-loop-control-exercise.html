Section 1 ‚Äî MCQs (25 Questions)

Q1. The break statement in Python:
a) Skips current iteration
b) Exits the loop immediately
c) Does nothing
d) Exits the program
Answer: (b)
Explanation: break stops the loop at once and control moves to the first statement after the loop.
Q2. The continue statement:
a) Exits the loop
b) Skips the remaining body for current iteration
c) Exits the program
d) Restarts the program
Answer: (b)
Explanation: continue jumps to the next iteration of the loop.
Q3. The pass statement:
a) Terminates the loop
b) Skips an iteration
c) Is a placeholder that does nothing
d) Restarts the loop
Answer: (c)
Explanation: pass is used when a statement is required syntactically but no action is needed.
Q4. Which construct is most commonly used in menu-driven programs?
a) if only
b) while True with break
c) for with break
d) try-except
Answer: (b)
Explanation: Menus usually repeat until user chooses Exit, so an infinite while True with break is natural.
Q5. What happens if break is used inside a nested loop?
a) All loops exit
b) Only inner loop exits
c) Program ends
d) It is ignored
Answer: (b)
Explanation: break only exits the loop where it is written (inner loop if nested).
Q6. Which statement is useful to skip processing for some items in a loop?
a) break
b) continue
c) pass
d) return
Answer: (b)
Explanation: continue skips the rest of the body for that iteration and resumes with next iteration.
Q7. Which of these will create an infinite loop if no break is used?
a) for i in range(10)
b) while i > 0 with i not changing
c) while True
d) both (b) and (c)
Answer: (d)
Explanation: If the condition never becomes False, loop is infinite. while True is always True; if i never changes, condition remains True.
Q8. In a menu-driven program, which input usually causes loop to terminate?
a) 0

Explanation: We design the menu with a specific "Exit" option; when chosen, we call break.
Q9. Which statement is syntactically valid?
a) if x > 0: continue
b) if x > 0: break
c) if x > 0: pass
d) all of the above
Answer: (d)
Explanation: All three can appear inside a loop body, and are syntactically valid.
Q10. Which control statement is best to use as a reminder to write code later?

Explanation: pass is used as a placeholder where code will be added later.

Section 4 ‚Äî Short Conceptual Questions (15 Questions)

Q56. Explain the difference between break and continue.
Answer: break exits the loop completely; continue skips the rest of the current iteration and moves to the next.
Q57. Why is pass needed in Python?
Answer: Python doesn‚Äôt allow empty blocks; pass acts as a placeholder where no action is required yet.
Q58. Give one real-life example where break is useful in a loop.
Answer: Example: Searching a list of roll numbers; once found, use break to stop further searching.
Q59. Give one real-life situation where continue is useful.
Answer: Example: Looping through marks but skipping absent students (marked as -1) using continue.
Q60. Can break be used outside a loop? Why / Why not?
Answer: No. break is only valid inside loops; outside, it causes a syntax error.
Q61. What happens if we forget to update the loop variable in a while loop with continue?
Answer: The condition may never change, causing an infinite loop because execution keeps skipping the update code.
Q62. In a menu-driven program, why is while True often preferred?
Answer: It simplifies the condition logic; the loop always runs until an explicit break is executed on Exit choice.
Q63. Which statement is better for creating "empty function" definitions that will be implemented later?
Answer: Use pass inside the function body as a placeholder.
Q64. What is a ‚Äúsentinel value‚Äù in loops?
Answer: A special value (like -1 or 0) that indicates "stop now" when entered; often used with break.
Q65. Why should menus usually validate user input?
Answer: To handle invalid choices gracefully instead of crashing or behaving incorrectly.
Q66. Can we use continue in a for loop? Explain.
Answer: Yes. On continue, the loop immediately jumps to get the next item from the iterator.
Q67. What is the main difference between a ‚Äúcounter-controlled‚Äù loop and a ‚Äúmenu-driven‚Äù loop?
Answer: Counter-controlled loops run fixed number of times; menu-driven loops run until user chooses to exit.
Q68. Why is pass safer than leaving a block empty?
Answer: Python does not allow empty blocks; pass avoids syntax errors and clearly indicates intentional emptiness.
Q69. Does break affect code after the loop?
Answer: No. It only stops the loop; code after the loop executes normally.
Q70. When designing menu options, why is it good to keep ‚ÄúExit‚Äù as last option?
Answer: It‚Äôs intuitive for users; they naturally look at the end of menu to exit, improving usability.
