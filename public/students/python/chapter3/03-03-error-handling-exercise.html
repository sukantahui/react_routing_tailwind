2. Output-Based Questions

Q1 try: print(10 / 0) except: print("Error!") print("Done")
Q2 try: x = int("25") print(x + 5) except ValueError: print("Conversion error") print("End")
Q3 try: items = [1, 2, 3] print(items[5]) except IndexError: print("Index problem") except: print("Some other error")
Q4 try: num = int("abc") except ValueError: print("Value error!") else: print("No error")
Q5 try: a = 10 b = 0 c = a / b except ZeroDivisionError: print("Divide by zero") finally: print("Finished")

4. Coding Practice Questions

Easy

  1. Write a program to input a number and handle the case when the user enters text instead of number (using try‚Äìexcept).
  2. Write a program to divide two numbers entered by the user and handle division by zero.
  3. Write a program that opens a file and handles the case when the file does not exist.

Moderate

  3. Write a small calculator that handles: invalid numbers, division by zero, and any other unexpected errors.

Advanced

  1. Write a function that reads numbers from a text file (one per line), converts them to integers using try‚Äìexcept, and calculates the sum. Lines that are not valid numbers should be skipped with a warning.

  3. Implement a robust input function get_int(prompt) that repeatedly asks the user until a valid integer is entered, using while + try‚Äìexcept.

6. ASCII Diagrams & Flowcharts (Conceptual)

6.1 Flow of try‚Äìexcept

+----------------------------+ | START | +-------------+--------------+ | v [ Code in try ] | +-------+--------+ | Error occurs? | +---+-------+----+ | | Yes No | | v v [ Jump to [ Skip except ] except ] | | | v v [ Next statement ] | v END

6.2 try‚Äìexcept‚Äìelse

try block OK (no error) ---> else block runs | +-- if error ---> except block runs (else is skipped)

6.3 try‚Äìexcept‚Äìfinally

+-------------------+ | try | +---------+---------+ | +------+------+ | | OK Exception | | v v [Normal flow] [except] \ / \ / v v +-------+ | finally | +-------+ | v END
