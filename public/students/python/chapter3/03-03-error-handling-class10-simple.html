1. What is an Error?

When Python cannot understand or cannot run our code properly, it shows an error message. If we do not handle errors, our program stops in the middle.

In real life, if we make a mistake in a form, the teacher asks us to correct it. Similarly, Python shows errors so that we can correct the program.

Types of Errors (Basic)

  • Syntax Error ‚Äì we break Python‚Äôs grammar rules.
  • Runtime Error ‚Äì error while program is running (like divide by zero).

Example: Syntax Error

if x > 5 print("Hello")

Here we forgot : after if x > 5.

Example: Runtime Error

print(10 / 0) # division by zero

2. What is an Exception?

An exception is a special type of runtime error in Python. Example: ZeroDivisionError, ValueError, IndexError.

We can handle these exceptions using try and except so the program does not crash.

Simple Example (without handling)

a = int("abc") # ValueError print(a)

3. Using try‚Äìexcept (Simple)

Basic idea

We put risky code inside try. If error occurs, Python jumps to except.

try: # risky code a = int(input("Enter a number: ")) print("You entered:", a) except: print("Something went wrong!")

Flowchart of try‚Äìexcept (Simple)

START | v try block | +-- No error? --> go to NEXT LINE after except | +-- Error? ----> jump to except | v print message | v END

4. Small Examples

Example 1 ‚Äî Handle invalid input

try: age = int(input("Enter your age: ")) print("Your age is", age) except: print("Please enter a number only!")

Example 2 ‚Äî Handle division by zero

try: a = int(input("Enter numerator: ")) b = int(input("Enter denominator: ")) print("Result =", a / b) except ZeroDivisionError: print("You cannot divide by zero!") except ValueError: print("Enter numbers only!")

5. Quick Revision Points

  • Syntax error ‚Üí Python cannot read the code.
  • Runtime error / exception ‚Üí error while running.
  • try ‚Üí block where we put risky code.
  • except ‚Üí block that runs if error happens.
  • Using error handling makes our program more safe and user-friendly.
