1. Introduction to Errors & Exceptions

Errors stop the execution of a Python program. Python categorizes errors mainly as:

  • Syntax Errors ‚Äì violation of Python rules
  • Runtime Errors ‚Äì errors during execution
  • Exceptions ‚Äì specific runtime errors that can be handled
Error stops the program.
Exception Handling allows the program to continue.

2. Syntax Errors

These occur when Python cannot understand the code ‚Äî usually due to incorrect syntax.

Example ‚Äî Missing colon

if x > 10 print("Hello")

Example ‚Äî Wrong indentation

if True: print("Hi") # Error

3. Runtime Errors

These occur after the program starts running. They happen due to invalid operations (divide by zero, invalid type, missing file, etc.).

Example ‚Äî ZeroDivisionError

print(10 / 0)

Example ‚Äî TypeError

print("Age: " + 20)

Example ‚Äî NameError

print(value) # value not defined

4. Python Exception Types (Most Common)

Exception Description

NameError Variable not defined
FileNotFoundError File does not exist
AttributeError Invalid attribute access

5. try‚Äìexcept (Basic Exception Handling)

Syntax:

try:
    # code that may cause error
except:
    # code to handle error

Example ‚Äî Handling division error

try: print(10 / 0) except: print("Cannot divide by zero!")

Example ‚Äî Handling specific exception

try: x = int("hello") except ValueError: print("Conversion error!")

Example ‚Äî Catching multiple exceptions

try: a = int(input("Enter number: ")) print(10 / a) except ZeroDivisionError: print("Cannot divide by zero!") except ValueError: print("Invalid number!")

Example ‚Äî Try with Else

try: num = int(input("Enter number: ")) except ValueError: print("Invalid input") else: print("You entered:", num)

Example ‚Äî Try with Finally

try: file = open("data.txt") except FileNotFoundError: print("File not found") finally: print("Process completed")
finally block always executes ‚Äî even if an error occurs.

6. Best Practices for Error Handling

  • Handle only expected errors
  • Use specific exceptions instead of generic except:
  • Use finally for cleanup (files, DB connections)
  • Avoid hiding errors unnecessarily
  • Always validate user input
