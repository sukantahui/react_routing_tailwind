6.1 Modules

Importing modules, built‑in modules, and creating user‑defined modules.

What is a Module?

A module in Python is simply a file that contains Python code — functions, variables, classes, etc. Modules help keep code organized, reusable, and structured.

Example: A file named mymath.py with helpful math functions is considered a module.

Importing Modules

1. Using import

import math
print(math.sqrt(25))

This imports the entire module, so you must use module_name.function.

2. Using from ... import ...

from math import sqrt
print(sqrt(25))

This imports only the specific function directly.

3. Importing with Alias

import math as m
print(m.pi)

Common Built‑in Modules

1. math Module

  • Square root: math.sqrt()
  • Power: math.pow()
  • Pi value: math.pi
import math
print(math.sin(math.pi/2))

2. random Module

  • random.randint(a, b) → random integer
  • random.choice(list) → random element
  • random.random() → random float (0–1)
import random
print(random.randint(1, 10))
print(random.choice(["red","blue","green"]))

3. datetime Module

  • Dates and times
  • Current date/time
from datetime import datetime
now = datetime.now()
print(now)

Creating User‑Defined Modules

You can create your own Python module. Just create a file with functions.

Step 1 — Create file: mymodule.py

def greet(name):
    return f"Hello {name}!"

def add(a, b):
    return a + b

Step 2 — Import into another file

import mymodule
print(mymodule.greet("Sukanta"))
print(mymodule.add(5, 3))

Step 3 — Import only specific functions

from mymodule import greet
print(greet("Rahul"))
Why Use Modules?
  • Keeps programs organized
  • Makes code reusable
  • Easy to maintain and debug

Deep Concepts of Python Modules

A module is one of Python’s most powerful organisational features. Below we expand the tutorial with deeper explanations, diagrams, examples, and advanced use‑cases.

How Python Loads a Module (Internal Working)

When Python imports a module, the following happens internally:

  1. Python checks if the module is already loaded in sys.modules.
  2. If not found → Python searches for the module in the directories listed in sys.path.
  3. If found → Python compiles it to bytecode (.pyc).
  4. Python executes the module and loads its namespace.
import sys
print(sys.modules)
print(sys.path)

Types of Modules in Python

  • Built‑in Modules → Already compiled into Python (math, sys, time)
  • Standard Library Modules → Bundled but not preloaded (random, datetime, pathlib)
  • Third‑Party Modules → Installed using pip (numpy, pandas)
  • User‑Defined Modules → Your own .py files

Advanced Import Techniques

Import Everything (Not Recommended)

from math import *
print(sqrt(25))

Why not recommended? It pollutes the namespace and may overwrite names unintentionally.

Selective Import with Alias

from datetime import datetime as dt
print(dt.now())

Importing Multiple Functions

from math import sqrt, sin, tan

Conditional Import (Performance Optimisation)

if need_heavy_math:
    import numpy as np

1. math Module — Extended Examples

import math

2. random Module — Simulations

Creating a Package (Folder of Modules)

A package is a folder containing modules and an __init__.py file.

mypackage/
    __init__.py
    mathops.py
    stringops.py

Using a Package

from mypackage import mathops
print(mathops.add(3, 4))

Using __name__ == "__main__" in Modules

This ensures code runs only when the file is run directly, not when imported.

def greet():
    print("Hello")

if __name__ == "__main__":
    greet()

Module Execution Flow (Diagram)

Program → import module → Python searches path → loads → executes → stores in sys.modules

Common Errors When Working with Modules

  • ModuleNotFoundError → Path incorrect
  • ImportError → Name not found in the module
  • AttributeError → Misspelled function or attribute
