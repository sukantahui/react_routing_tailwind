1. Expressions in Python

In Python, an expression is any combination of values, variables, operators and function calls that Python can evaluate to produce a single result.

Examples of expressions
5 + 3 # arithmetic expression ‚Üí result 8 x * 10 + 5 # uses variable x "Py" + "thon" # string concatenation ‚Üí "Python" marks > 40 # relational ‚Üí True or False a and b # logical expression
Expression vs Statement: ‚Ä¢ Expression: produces a value. (e.g., 5 + 3) ‚Ä¢ Statement: performs an action. (e.g., print(5 + 3), if block, for loop)

2. Structure of an Expression

General form:

operand1 operator operand2 Example: 10 + 20, a * b, x > y

But expressions can be more complex with multiple operators and parentheses:

total = (basic + hra) - (tax * 2) + bonus

3. Types of Expressions

3.1 Arithmetic Expressions

Use arithmetic operators (+, -, *, /, //, %, **) and evaluate to a numeric result.

a = 10 b = 3 expr = a + b * 2 - 4 print(expr)

3.2 Relational (Comparison) Expressions

Use comparison operators (<, <=, >, >=, ==, !=) and result is True or False.

marks = 72 print(marks >= 40) # True print(marks < 60) # False

3.3 Logical Expressions

Combine relational expressions using logical operators (and, or, not).

age = 19 marks = 65 eligible = (age >= 18) and (marks >= 60) print(eligible) # True

3.4 Assignment Expressions

Use assignment operators (=, +=, -=, ...). These are usually statements, but they are built around expressions.

x = 10 + 5 # 10 + 5 is an expression, x = 15 is assignment statement x += 2 # x = x + 2

3.5 String Expressions

Expressions involving strings and string operators like concatenation (+) and repetition (*).

greet = "Hello " name = "Sukanta" msg = greet + name print(msg) line = "-" * 10 print(line)

3.6 Mixed-Type Expressions

When operands in an expression have different numeric types (int, float), Python automatically promotes the lower type.

x = 10 # int y = 3.5 # float result = x + y print(result) # 13.5 (float) print(type(result)) # <class 'float'>
Automatic promotion: int + float ‚Üí result is float. But string + int is not allowed without casting.

4. Evaluating Expressions Step-by-Step (Dry Run)

To understand an expression, we break it into steps following operator precedence and associativity.

4.1 Example 1

a = 10 b = 3 c = 2 expr = a + b * c - 4 print(expr)

Dry run:

  • Step 1: Identify highest precedence ‚Üí *
  • Step 2: b * c = 3 * 2 = 6
  • Step 3: Now expression becomes a + 6 - 4
  • Step 4: Evaluate from left to right for + and - ‚Üí 10 + 6 = 16
  • Step 5: 16 - 4 = 12
  • Final result: expr = 12

4.2 Example 2 ‚Äî With parentheses

x = 5 y = 2 z = 3 expr1 = x + y * z expr2 = (x + y) * z print(expr1) print(expr2)

Dry run:

  • expr1 ‚Üí 5 + 2 * 3 ‚Üí 2 * 3 = 6 ‚Üí 5 + 6 = 11
  • expr2 ‚Üí (5 + 2) * 3 ‚Üí 5 + 2 = 7 ‚Üí 7 * 3 = 21
Same numbers, different grouping ‚Üí different result. This is why understanding precedence and parentheses is very important.

5. Operator Precedence in Python

When an expression contains many different operators, Python uses a fixed precedence order to decide which parts to evaluate first.

5.1 Precedence Table (Common Operators)

Level Operators Description Associativity
Highest ( ) Parentheses (grouping) ‚Äì
** Exponent Right to Left
+x, -x, ~x Unary plus, unary minus, bitwise NOT Right to Left
*, /, //, % Multiplication, division, floor division, modulus Left to Right
+, - Addition, subtraction Left to Right
Comparison (<, <=, >, >=, ==, !=) Relational operators Left to Right
not Logical NOT Right to Left
and Logical AND Left to Right
or Logical OR Left to Right
Lowest =, +=, -=, ... Assignment operators Right to Left

5.2 Examples Based on Precedence

expr = 10 + 5 * 2 # * before + # 5 * 2 = 10, then 10 + 10 = 20 print(expr) # 20 expr2 = 10 + 5 * 2 ** 2 # ** first: 2 ** 2 = 4 # then *: 5 * 4 = 20 # then +: 10 + 20 = 30 print(expr2) # 30
expr3 = 100 / 5 * 2 # / and * same precedence, evaluated left to right: # 100 / 5 = 20.0 # 20.0 * 2 = 40.0 print(expr3) # 40.0

6. Associativity (Direction of Evaluation)

When multiple operators of the same precedence appear, Python uses associativity (left-to-right or right-to-left) to decide the evaluation order.

6.1 Left-to-Right Associativity

Operators like +, -, *, /, //, % are evaluated from left to right.

expr = 100 - 20 - 10 # Left to right: # Step 1: 100 - 20 = 80 # Step 2: 80 - 10 = 70 print(expr) # 70

6.2 Right-to-Left Associativity

The exponent operator ** is right-associative.

expr = 2 ** 3 ** 2 # Evaluated as 2 ** (3 ** 2) # 3 ** 2 = 9 # 2 ** 9 = 512 print(expr) # 512
If you write (2 ** 3) ** 2, result becomes 8 ** 2 = 64. So brackets change the result drastically.

7. Using Parentheses Effectively

Parentheses ( ) allow you to override default precedence and make your intention clear. This also improves readability.

a = 4 b = 3 c = 2 expr1 = a + b * c # 4 + 3 * 2 = 10 expr2 = (a + b) * c # (4 + 3) * 2 = 14 print(expr1) print(expr2)
Best practice: When in doubt, use parentheses. Even if you know precedence, parentheses make your code easier for others to read.

8. Mixed-Type and Boolean Expressions

8.1 Mixing Integers and Floats

x = 5 # int y = 2.0 # float result = x / y + 3 print(result) # 5 / 2.0 = 2.5 ‚Üí 2.5 + 3 = 5.5 print(type(result)) # <class 'float'>

8.2 Combining Relational and Logical Operators

a = 10 b = 5 c = 3 expr = a > b and b > c # a > b ‚Üí 10 > 5 ‚Üí True # b > c ‚Üí 5 > 3 ‚Üí True # True and True ‚Üí True print(expr)

8.3 Short-Circuit Evaluation (Recap in Expressions)

In and and or expressions, Python may stop early without checking all parts if the result is already decided.

def test(): print("test() called") return True print(False and test()) # test() is NOT called print(True or test()) # test() is NOT called

9. Common Tricky Expressions (Important for Exams)

9.1 Tricky Example 1

x = 2 y = 3 z = 4 expr = x + y * z ** 2 # z ** 2 = 4 ** 2 = 16 # y * 16 = 3 * 16 = 48 # x + 48 = 2 + 48 = 50 print(expr) # 50

9.2 Tricky Example 2

a = 10 b = 20 c = 5 print(a < b and b < c or a == 10) # Step 1: (a < b) ‚Üí True # Step 2: (b < c) ‚Üí 20 < 5 ‚Üí False # Step 3: True and False ‚Üí False # Step 4: a == 10 ‚Üí True # Step 5: False or True ‚Üí True

9.3 Tricky Example 3 ‚Äî Combining not, and, or

x = 5 y = 0 expr = not x > 3 or y != 0 and x == 5 # Step 1: x > 3 ‚Üí True # Step 2: not True ‚Üí False # Step 3: y != 0 ‚Üí 0 != 0 ‚Üí False # Step 4: x == 5 ‚Üí True # Step 5: False and True ‚Üí False # Step 6: False or False ‚Üí False print(expr)

10. Common Mistakes & Pitfalls

  • Using = instead of == in conditions.
  • Forgetting that input() returns string ‚Üí causes TypeError in numeric expressions.
  • Misunderstanding integer division vs float division.
  • Wrongly grouping conditions without parentheses.
Example ‚Äî input() Pitfall
a = input("Enter a number: ") b = input("Enter another number: ") print(a + b) # If a = "10", b = "5" # Output: "105" (string concatenation), NOT 15
Use int() or float() for numeric expressions:
a = int(input("..."))
