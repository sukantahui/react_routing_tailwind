Questions 26–50

26. Which statement about var is TRUE?

A. It is block-scoped
B. It is function-scoped
C. It cannot be redeclared
D. It throws error on hoisting

Answer: B
Explanation: var is scoped to the nearest function, not to blocks like if or for.

27. What is the output?

if (true) {
  var x = 5;
}
console.log(x);
Answer: 5
Explanation: var leaks out of block to the function/global scope.

28. What is the output?

if (true) {
  let y = 10;
}
console.log(y);
Answer: ReferenceError
Explanation: y is block-scoped; it doesn't exist outside the if block.

29. What is logged?

console.log(a);
var a = 10;
Answer: undefined
Explanation: var is hoisted and initialized with undefined before execution.

30. What happens here?

console.log(b);
let b = 20;
Answer: ReferenceError
Explanation: let is hoisted but not initialized; accessing it before the line creates a temporal dead zone error.

31. Which comparison is safest for equality?

A. ==   B. ===   C. =   D. !==

Answer: B
Explanation: === avoids type coercion and checks both type and value.

32. What is the result?

5 == "5"

A. true   B. false   C. TypeError   D. undefined

Answer: A (true)
Explanation: == converts the string to number then compares.

33. What is the result?

5 === "5"

A. true   B. false   C. TypeError   D. NaN

Answer: B
Explanation: Types are different (number vs string), so strict equality fails.

34. Evaluate:

"10" * 2

A. "102"   B. 20   C. NaN   D. "20"

Answer: B (20)
Explanation: * forces numeric conversion of the string.

35. Evaluate:

"10" + 2

A. "12"   B. "102"   C. 12   D. 102

Answer: B
Explanation: + with a string performs concatenation, not addition.

36. What is the output?

console.log(Boolean(0));
Answer: false
Explanation: In JS, 0, "", null, undefined, NaN, and false are falsy.

37. What does this print?

console.log(Boolean("0"));
Answer: true
Explanation: Non-empty strings are truthy, even "0".

38. Explicit conversion to number:

Number("50")

A. 50   B. "50"   C. NaN   D. Error

Answer: A
Explanation: Number("50") converts the string to a numeric value.

39. What is the output?

console.log(String(500));
Answer: "500"
Explanation: String() converts a value to string.

40. What is logged?

console.log(typeof Symbol("id"));
Answer: "symbol"
Explanation: A Symbol has primitive type symbol.

41. True or False: BigInt and number can be mixed directly in arithmetic.

Answer: False
Explanation: You cannot mix bigint with number directly; it throws a TypeError.

42. True or False: typeof NaN is "number".

Answer: True
Explanation: NaN is a special numeric value meaning “not a number”.

43. Evaluate:

true + 5
Answer: 6
Explanation: true becomes 1 in numeric context → 1 + 5 = 6.

44. Evaluate:

false + "5"
Answer: "false5"
Explanation: With + and a string, false becomes "false" and concatenates.

45. Which gives true?

A. 0 === false
B. "" === false
C. Boolean("") === false
D. null === undefined

Answer: C
Explanation: Boolean("") is false; strict equality is then true.

46. Output?

console.log(typeof 123n);
Answer: "bigint"
Explanation: Numeric literals with n suffix are BigInt.

47. Output?

let secret = Symbol("token");
let obj = {};
obj[secret] = 123;
console.log(obj[secret]);
Answer: 123
Explanation: Symbol keys work like hidden properties but still accessible via the same Symbol reference.

48. True or False: Two Symbols with the same description are equal.

Answer: False
Explanation: Symbol("id") === Symbol("id") is always false; each Symbol is unique.

49. Evaluate:

Number("hello")
Answer: NaN
Explanation: Non-numeric string cannot be converted to a valid number.

50. Evaluate:

Boolean("hello")
Answer: true
Explanation: Any non-empty string is truthy.
