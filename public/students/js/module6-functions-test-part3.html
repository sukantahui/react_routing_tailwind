Q1 ¬∑ Return & Unreachable Code
Concept

What is the output of the following code?

function test() {
  console.log("A");
  return "Done";
  console.log("B");
}

console.log(test());
  • A) A then B then Done
  • B) A then Done
  • C) Done only
  • D) A only

Correct Answer: B

The function logs "A", then hits return "Done". As soon as return executes, the function exits and console.log("B") is never reached (unreachable code). The outer console.log prints the returned value "Done". So the console shows: 

A
Done
Q2 ¬∑ Function Expression Timing
Expression & Hoisting

What happens when this code runs?

sayHi();

const sayHi = function () {
  console.log("Hello!");
};
  • A) Prints ‚ÄúHello!‚Äù
  • B) ReferenceError: sayHi is not defined
  • C) TypeError: sayHi is not a function
  • D) Nothing happens

Correct Answer: C

sayHi is a const function expression. The variable sayHi is hoisted but not initialized before its declaration line. When we call sayHi() above its definition, we are in the temporal dead zone of the const variable. The engine sees sayHi but it‚Äôs not yet a function value, which leads to: TypeError: sayHi is not a function at runtime in many environments.

Q3 ¬∑ Arrow Function Return
Arrow ¬∑ Syntax

What does the following code log?

const calc = (a, b) => {
  a + b;
};

console.log(calc(3, 4));
  • A) 7
  • B) undefined
  • C) NaN
  • D) Throws an error

Correct Answer: B

In arrow functions with a block body ({ }), you must explicitly use return. Here, a + b; is executed but its value is not returned. If a function doesn‚Äôt return anything explicitly, JavaScript returns undefined.

Correct version for 7 would be:

const calc = (a, b) => {
  return a + b;
};
Q4 ¬∑ Implicit Return
Arrow ¬∑ One-Liner

What is the output?

const double = n => n * 2;

console.log(double(5));
  • A) 10
  • B) 25
  • C) undefined
  • D) Error: missing return

Correct Answer: A

For single-expression arrow functions without curly braces, the expression value is implicitly returned. So n => n * 2 returns n * 2. With n = 5, the result is 10.

Q5 ¬∑ Default Parameter
Default Param

What is logged to the console?

function welcome(name = "Student") {
  console.log("Welcome,", name);
}

welcome("Sukanta");
welcome();
  • A) ‚ÄúWelcome, Sukanta‚Äù and then error
  • B) ‚ÄúWelcome, Sukanta‚Äù and ‚ÄúWelcome, Student‚Äù
  • C) ‚ÄúWelcome, Sukanta‚Äù and ‚ÄúWelcome, undefined‚Äù
  • D) Only ‚ÄúWelcome, Sukanta‚Äù

Correct Answer: B

When called with "Sukanta", the parameter name becomes "Sukanta". When called with no argument, the default value "Student" is used. So we get: 

Welcome, Sukanta
Welcome, Student
Q6 ¬∑ Rest Parameter Sum
Rest Param

What is the output?

function sumAll(...nums) {
  let total = 0;
  for (let n of nums) {
    total += n;
  }
  return total;
}

console.log(sumAll(2, 4, 6));
  • A) 2
  • B) 4
  • C) 6
  • D) 12

Correct Answer: D

The rest parameter ...nums collects all arguments into an array [2, 4, 6]. The loop adds them: 0 + 2 + 4 + 6 = 12. So the output is 12.

Q7 ¬∑ Scope Inside Function
Scope

What will happen when this code runs?

let x = 10;

function demo() {
  let y = 20;
  console.log(x, y);
}

demo();
console.log(y);
  • A) Prints ‚Äú10 20‚Äù then ‚Äú20‚Äù
  • B) Prints ‚Äú10 20‚Äù then ReferenceError
  • C) ReferenceError before anything prints
  • D) Prints ‚Äú10 undefined‚Äù then ‚Äúundefined‚Äù

Correct Answer: B

Inside the function, both x and y are accessible, so console.log(x, y) prints 10 20. Outside the function, y is not defined (it is local to demo), so console.log(y) causes a ReferenceError.

Q8 ¬∑ Block Scope with let
Scope ¬∑ Block

What is the output?

let a = 5;

if (true) {
  let a = 10;
  console.log("Inside:", a);
}

console.log("Outside:", a);
  • A) Inside: 10 and Outside: 10
  • B) Inside: 10 and Outside: 5
  • C) Inside: 5 and Outside: 10
  • D) Inside: 5 and Outside: 5

Correct Answer: B

let is block-scoped. The a inside the if block is a different variable than the outer a. So inside: a = 10, outside: a = 5.

Q9 ¬∑ Function Returning Object
Return ¬∑ Object

What does this code print?

function makeUser(name, age) {
  return {
    name: name,
    age: age,
    isAdult: age >= 18
  };
}

let u = makeUser("Rita", 17);
console.log(u.isAdult);
  • A) "true"
  • B) true
  • C) false
  • D) "false"

Correct Answer: C

The function returns an object with isAdult as age >= 18. For age = 17, this expression evaluates to boolean false. So u.isAdult is false, and the console prints false (a boolean, not a string).

Q10 ¬∑ Overwriting Functions
Declaration

What is the final output?

function greet() {
  console.log("Hello from first version");
}

function greet() {
  console.log("Hello from second version");
}

greet();
  • A) Hello from first version
  • B) Hello from second version
  • C) Both messages one after another
  • D) Error: duplicate function

Correct Answer: B

Function declarations with the same name result in the last one overwriting the previous ones in the same scope. After parsing, only the second greet exists. So calling greet() prints: "Hello from second version".

Q12 ¬∑ Early Return Guard
Pattern

What is printed?

function printGST(price) {
  if (price <= 0) {
    console.log("Invalid price");
    return;
  }
  console.log("GST =", price * 0.18);
}

printGST(-100);
printGST(1000);
  • A) "Invalid price" only
  • B) "GST = -18" and "GST = 180"
  • C) "Invalid price" and "GST = 180"
  • D) Nothing is printed

Correct Answer: C

For -100, the guard if (price <= 0) triggers, printing "Invalid price" and returning early. For 1000, it skips the guard and prints "GST = 180".

Q13 ¬∑ Function as Argument
Higher-Order

What is the output?

function applyTwice(fn, value) {
  return fn(fn(value));
}

function inc(x) {
  return x + 1;
}

console.log(applyTwice(inc, 5));
  • A) 5
  • B) 6
  • C) 7
  • D) 8

Correct Answer: C

First call: fn(value) ‚Üí inc(5) = 6. Second call: fn(6) ‚Üí inc(6) = 7. So applyTwice(inc, 5) returns 7.

Q14 ¬∑ Missing Arguments
Parameters

What is logged?

function area(width, height) {
  return width * height;
}

console.log(area(5));
  • A) 0
  • B) 5
  • C) NaN
  • D) undefined

Correct Answer: C

area(5) means width=5 and height=undefined. The multiplication 5 * undefined becomes NaN.

To avoid this, you could use a default parameter, e.g. function area(width, height = 1) { ... }.

Q15 ¬∑ Nested Functions & Scope
Scope ¬∑ Nested

What is the output?

function outer() {
  let message = "Outer";

  function inner() {
    let innerMsg = "Inner";
    console.log(message);
  }

  inner();
  console.log(innerMsg);
}

outer();
  • A) "Outer" then "Inner"
  • B) "Outer" then ReferenceError
  • C) ReferenceError before printing anything
  • D) Only "Inner"

Correct Answer: B

inner() can access message from outer, so it prints "Outer". But innerMsg is local to inner, not visible in outer, so console.log(innerMsg) throws a ReferenceError after the first print.

Q16 ¬∑ Arrow in Object Method
Arrow ¬∑ Method

In basic teaching, why is it usually not recommended to use arrow functions as object methods that rely on this?

  • A) Arrow functions are slower
  • B) Arrow functions cannot return values
  • C) Arrow functions do not have their own this
  • D) Arrow functions cannot be stored in objects

Correct Answer: C

Arrow functions do not have their own this binding; they use the this value from the surrounding scope (lexical this). For object methods where students expect this to refer to the object itself, a normal function is safer.

Call Flow

Q18 ¬∑ Function Returning Function
Higher-Order

What is printed?

function createMultiplier(factor) {
  return function (n) {
    return n * factor;
  };
}

const double = createMultiplier(2);
console.log(double(7));
  • A) 2
  • B) 7
  • C) 14
  • D) Error: factor not defined

Correct Answer: C

createMultiplier(2) returns a function that multiplies its input by 2. So double(7) returns 14. This is a classic closure example: the inner function remembers factor.

Q20 ¬∑ Function in Variable
Expression

What is printed?

let f = function (x) {
  return x * x;
};

let g = f;
f = null;

console.log(typeof g, g(4));
  • A) "function", 16
  • B) "object", 16
  • C) "function", error
  • D) "undefined", error

Correct Answer: A

f holds the function, then g = f copies the reference. Setting f = null does not affect g. So g is still a function. typeof g is "function" and g(4) is 16.

Q21 ¬∑ Missing return in Some Path
Return Trap

What does compute(5) log?

function compute(n) {
  if (n > 10) {
    return n * 2;
  }
  // no else
}

console.log(compute(5));
  • A) 10
  • B) 0
  • C) undefined
  • D) Error: missing return

Correct Answer: C

For n = 5, the condition n > 10 is false, so the if block is skipped and no return executes. In that case, JavaScript returns undefined by default.

Q24 ¬∑ Simple Real-Life Function
Scenario

You have a function used in fee calculation: 

function applyDiscount(amount, discountPercent = 10) {
  return amount - (amount * discountPercent / 100);
}

console.log(applyDiscount(10000));
console.log(applyDiscount(10000, 20));

What are the two printed values?

  • A) 9000 and 8000
  • B) 10000 and 8000
  • C) 9000 and 10000
  • D) 8000 and 9000

Correct Answer: A

First call uses default discountPercent = 10, so 10000 - 1000 = 9000. Second call uses 20%, so 10000 - 2000 = 8000.

Q25 ¬∑ Trick with return + newline
Syntax Trap

What does this function return?

function getUser() {
  return
  {
    name: "Amit"
  };
}

console.log(getUser());
  • A) { name: "Amit" }
  • B) undefined
  • C) Error: unexpected token
  • D) "Amit"

Correct Answer: B

Because of JavaScript‚Äôs automatic semicolon insertion, the line with return is treated as return; The object on the next line is never returned; it becomes a separate statement. So the function returns undefined.

Correct version:

function getUser() {
  return {
    name: "Amit"
  };
}
