Step 1 ¬∑ What Is Control Flow?

1. Control Flow: How JavaScript Makes Decisions

By default, JavaScript executes statements from top to bottom. Control flow statements like if, else, and switch allow you to choose different paths based on conditions.

1.1 A Real-World Analogy

  • If it is raining ‚Üí take an umbrella.
  • Else ‚Üí you don‚Äôt need an umbrella.

The same idea in JavaScript: 

let isRaining = true;

if (isRaining) {
  console.log("Take an umbrella.");
} else {
  console.log("No umbrella needed.");
}
A condition in JavaScript is any expression that results in true or false. JavaScript will treat many values as ‚Äútruthy‚Äù or ‚Äúfalsy‚Äù ‚Äî we explore that later in this module.
Step 2 ¬∑ Boolean Expressions

2. Boolean Expressions & Logical Building Blocks

A decision always starts with a question that returns yes/no, true/false. In JavaScript, this is called a boolean expression.

2.1 Relational (Comparison) Operators

10 > 5     // true
10 >= 10   // true
10 < 3     // false
5 == "5"   // true  (loose equality - not recommended)
5 === "5"  // false (strict equality - recommended)
Always use === and !==. Avoid == and != to prevent unexpected type coercion.

2.2 Logical Operators

// AND - all conditions must be true
age >= 18 && hasID

// OR - at least one condition must be true
isWeekend || isHoliday

// NOT - reverses a boolean
!isRaining

2.3 Example: Exam Eligibility

let attendance = 80;
let assignmentSubmitted = true;

let canSitForExam = attendance >= 75 && assignmentSubmitted;

console.log("Can sit for exam:", canSitForExam);
Step 3 ¬∑ Basic if

3. The if Statement

if executes a block of code only if the given condition is true.

3.1 Syntax

if (condition) {
  // code runs only when condition is true
}

3.2 Example: Positive Number Check

let number = 10;

if (number > 0) {
  console.log("Number is positive.");
}

3.3 Example: Simple Attendance Check

let attendance = 78;

if (attendance >= 75) {
  console.log("You meet the minimum attendance criteria.");
}
Teach students to read conditions in normal English: if (attendance >= 75) ‚Üí ‚ÄúIf attendance is greater than or equal to 75‚Äù.
Step 4 ¬∑ if‚Äìelse & Chains

4. if‚Ä¶else and else if Ladder

4.1 if‚Ä¶else (Two-Way Decision)

let age = 16;

if (age >= 18) {
  console.log("You are eligible to vote.");
} else {
  console.log("You are NOT eligible to vote yet.");
}

4.2 else if Ladder (Multi-Way Decision)

Used when you have more than two possible branches.

let marks = 82;
let grade;

if (marks >= 90) {
  grade = "A+";
} else if (marks >= 80) {
  grade = "A";
} else if (marks >= 70) {
  grade = "B";
} else if (marks >= 60) {
  grade = "C";
} else {
  grade = "D";
}

console.log("Grade:", grade);
The conditions are checked top to bottom. As soon as one condition becomes true, that block executes and the rest are ignored.

4.3 Example: Fee Discount Logic

let totalFee = 10000;
let category = "early-bird";  // "early-bird", "referral", "none"

let discount;

if (category === "early-bird") {
  discount = 0.20; // 20%
} else if (category === "referral") {
  discount = 0.10; // 10%
} else {
  discount = 0;
}

let finalAmount = totalFee - (totalFee * discount);
console.log("Payable Amount:", finalAmount);
Step 5 ¬∑ Nested Decisions

5. Nested if Statements

A nested if is an if statement written inside another if or else block. Use nesting when a second decision depends on the first.

5.1 Example: Scholarship Eligibility

let marks = 88;
let familyIncome = 250000;  // per year
let eligible = false;

if (marks >= 85) {
  if (familyIncome <= 300000) {
    eligible = true;
  }
}

if (eligible) {
  console.log("Scholarship approved.");
} else {
  console.log("Not eligible for scholarship.");
}
Nested if is powerful but can become confusing. If nesting gets too deep, consider restructuring with separate functions or a clearer ladder.
Step 6 ¬∑ Ternary Operator

6. Ternary Operator ‚Äî Compact Conditional

The ternary operator is a short way of writing simple if‚Ä¶else logic in a single expression.

6.1 Syntax

condition ? valueIfTrue : valueIfFalse;

6.2 Example: Even or Odd

let num = 7;
let result = (num % 2 === 0) ? "Even" : "Odd";

console.log(result); // "Odd"

6.3 Example: Short Message

let isLoggedIn = false;
let message = isLoggedIn ? "Welcome back!" : "Please log in.";

console.log(message);
Use ternary only for simple conditions. If logic is more complex, prefer full if‚Ä¶else statements for readability.
Step 7 ¬∑ switch

7. The switch Statement

switch is useful when you are comparing the same value against many fixed options (like menu choices, days, roles, etc.).

7.1 Basic Syntax

switch (expression) {
  case value1:
    // code
    break;
  case value2:
    // code
    break;
  default:
    // code
}

7.2 Example: Day Name from Number

let dayNumber = 3;
let dayName;

switch (dayNumber) {
  case 1:
    dayName = "Monday";
    break;
  case 2:
    dayName = "Tuesday";
    break;
  case 3:
    dayName = "Wednesday";
    break;
  case 4:
    dayName = "Thursday";
    break;
  case 5:
    dayName = "Friday";
    break;
  case 6:
    dayName = "Saturday";
    break;
  case 7:
    dayName = "Sunday";
    break;
  default:
    dayName = "Invalid day number";
}

console.log(dayName);
Important: Without break, execution ‚Äúfalls through‚Äù into the next case. Use that only when you intentionally want multiple cases to share logic.

7.3 Example: Fee Payment Mode

let paymentMode = "upi";

switch (paymentMode) {
  case "cash":
    console.log("Please pay at the front desk.");
    break;
  case "card":
    console.log("Card payment accepted at POS machine.");
    break;
  case "upi":
  case "net-banking":
    console.log("Online payment link has been sent to your phone.");
    break;
  default:
    console.log("Unknown payment mode.");
}
Step 8 ¬∑ Truthy, Falsy & Pitfalls

8. Truthy / Falsy Values & Common Mistakes

8.1 Falsy Values in JavaScript

These values behave like false in conditions:

  • false
  • 0
  • "" (empty string)
  • null
  • undefined
  • NaN
if ("") {
  console.log("This will NOT run");
}

if ("Hello") {
  console.log("This WILL run"); // "Hello" is truthy
}

8.2 Assignment (=) vs Comparison (== / ===)

// Common mistake:
let x = 5;

if (x = 10) {      // ‚ùå assigns 10 to x, condition becomes 10 (truthy)
  console.log("Runs always!");
}

// Correct:
if (x === 10) {    // ‚úî comparison
  console.log("Runs only if x is 10");
}
Always double-check you are using === inside conditions, not =. Many bugs come from this single character mistake.

8.3 Example: Safe Input Check

let username = ""; // suppose this comes from a form input

if (!username) {
  console.log("Username cannot be empty.");
} else {
  console.log("Welcome,", username);
}
