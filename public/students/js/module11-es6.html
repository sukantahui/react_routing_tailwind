Step 1 ¬∑ Why ES6+ Matters

1. Modern JavaScript ‚Äî Why It‚Äôs Important

ES6 (also called ES2015) and later versions introduced features that make code cleaner, safer, and easier to reason about. Modern projects, frameworks, and interviews expect you to be comfortable with these features.

  • Cleaner syntax ‚Üí less boilerplate, more readable code.
  • More expressive ‚Üí destructuring, spread, rest.
  • Safer ‚Üí let/const, optional chaining, ??.
  • Modern patterns ‚Üí used heavily in React, Node.js, etc.
For students: this module is a bridge from ‚Äútraditional JavaScript‚Äù to the way JavaScript is actually written in modern companies and projects.
Step 2 ¬∑ let & const vs var

2. let & const vs var (In Depth)

var is function-scoped and has tricky hoisting behavior. let and const are block-scoped and safer for modern code.

2.1 Scope Difference

if (true) {
  var x = 10;
  let y = 20;
}

console.log(x); // 10  (var is function-scoped, leaks out of block)
// console.log(y); // ReferenceError: y is not defined

2.2 Re-declaration & Re-assignment

var a = 1;
var a = 2; // allowed (but confusing!)

let b = 1;
// let b = 2; // ‚ùå not allowed, same scope re-declaration

const c = 1;
// c = 2; // ‚ùå not allowed, const cannot be re-assigned

2.3 Temporal Dead Zone (Concept)

// console.log(v); // undefined (due to hoisting)
var v = 5;

// console.log(l); // ReferenceError (temporal dead zone)
let l = 10;
Practical rule:
  • Use const by default.
  • Use let when you intentionally need to reassign.
  • Avoid var in modern code.
Step 3 ¬∑ Template Literals

3. Template Literals & String Methods

Template literals make string building easier and support embedded expressions and multi-line strings.

3.1 Basic Usage

const name = "Sukanta";
const course = "JavaScript";

const message = `Hello ${name}, welcome to the ${course} module!`;
console.log(message);

3.2 Multi-line Strings

const multiLine = `
Coder & AccoTax
JavaScript From Basic to Pro
Batch: Evening
`;

console.log(multiLine);

3.3 Useful String Methods with Template Literals

const title = "  modern javascript ES6+  ";

console.log(title.trim());        // remove spaces at both ends
console.log(title.toUpperCase()); // MODERN JAVASCRIPT ES6+
console.log(title.includes("ES6")); // true
console.log(title.replace("ES6+", "Features"));
Combine template literals with string methods for clean, readable UI messages and logs.
Step 4 ¬∑ Destructuring

4. Destructuring Arrays & Objects

Destructuring lets you ‚Äúunpack‚Äù values from arrays and objects into variables in a compact way.

4.1 Array Destructuring

const marks = [80, 85, 90];

const [math, science, english] = marks;

console.log(math, science, english); // 80 85 90

4.2 Skipping Values

const [first,, third] = [10, 20, 30];
console.log(first, third); // 10 30

4.3 Default Values

const [a = 1, b = 2] = [5];
console.log(a, b); // 5 2

4.4 Object Destructuring

const student = {
  name: "Amit",
  course: "JavaScript",
  city: "Barrackpore"
};

const { name, course } = student;
console.log(name, course); // Amit JavaScript

4.5 Renaming & Defaults

const { city: hometown = "Kolkata" } = student;

console.log(hometown); // Barrackpore

4.6 Nested Destructuring

const centre = {
  name: "Coder & AccoTax",
  address: {
    city: "Barrackpore",
    state: "West Bengal"
  }
};

const {
  address: { city, state }
} = centre;

console.log(city, state); // Barrackpore West Bengal
Destructuring is heavily used in React (e.g., const { useState } = React;) and when working with API responses.
Step 5 ¬∑ Spread & Rest

5. Spread ... & Rest ... Operators

The same ... syntax behaves as spread (expand) or rest (collect) depending on where it‚Äôs used.

5.1 Spread in Arrays

const arr1 = [1, 2, 3];
const arr2 = [4, 5];

const combined = [...arr1, ...arr2];
console.log(combined); // [1, 2, 3, 4, 5]

5.2 Copying Arrays (Shallow Copy)

const original = [10, 20, 30];
const copy = [...original];

copy.push(40);
console.log(original); // [10, 20, 30]
console.log(copy);     // [10, 20, 30, 40]

5.3 Spread in Objects

const baseStudent = { course: "JavaScript", centre: "Coder & AccoTax" };
const specific    = { name: "Neha", city: "Kolkata" };

const fullProfile = { ...baseStudent, ...specific };
console.log(fullProfile);
// { course: "JavaScript", centre: "...", name: "Neha", city: "Kolkata" }

5.4 Rest Parameters in Functions

function sum(...numbers) {
  return numbers.reduce((total, n) => total + n, 0);
}

console.log(sum(1, 2, 3));       // 6
console.log(sum(10, 20, 30, 40)); // 100

5.5 Rest in Destructuring

const [first, ...others] = [10, 20, 30, 40];
console.log(first);   // 10
console.log(others);  // [20, 30, 40]
Spread: ‚Äútake this array/object and spread its items/properties‚Äù. Rest: ‚Äúcollect the remaining items/properties into an array/object‚Äù.
Step 6 ¬∑ Defaults & Enhanced Objects

6. Default Parameters & Enhanced Object Literals

6.1 Default Parameters

function greet(name = "Student", course = "JavaScript") {
  console.log(`Hello ${name}, welcome to ${course}!`);
}

greet();                          // Student, JavaScript
greet("Amit");                    // Amit, JavaScript
greet("Neha", "ES6+ Features");   // Neha, ES6+ Features

6.2 Property Shorthand

const name = "Sukanta";
const topic = "ES6";

const courseInfo = {
  name,   // same as name: name
  topic,  // same as topic: topic
  level: "Intermediate"
};

console.log(courseInfo);

6.3 Method Shorthand

const utils = {
  log(message) {
    console.log("[LOG]", message);
  },
  error(message) {
    console.error("[ERROR]", message);
  }
};

utils.log("Hello");

6.4 Computed Property Names

const key = "courseName";

const obj = {
  [key]: "JavaScript From Basic to Pro",
  ["batch-" + 1]: "Evening"
};

console.log(obj.courseName); // JavaScript From Basic to Pro
console.log(obj["batch-1"]); // Evening
Enhanced object literals make configurations and data models more compact, especially in frameworks and API clients.
Step 7 ¬∑ Safer Access

7. Optional Chaining ?. & Nullish Coalescing ??

These features help you safely access deeply nested properties and provide default values only when something is null or undefined.

7.1 The Problem

const student = {
  name: "Amit",
  contact: {
    email: "amit@example.com"
  }
};

// console.log(student.contact.phone.number); // ‚ùå TypeError if phone is undefined

7.2 Optional Chaining ?.

console.log(student.contact?.phone?.number); // undefined (no error)

If anything before ?. is null or undefined, the whole expression safely becomes undefined instead of throwing an error.

7.3 Nullish Coalescing ??

const inputMarks = 0;

const defaultMarks1 = inputMarks || 50;
console.log(defaultMarks1); // 50 (0 is treated as falsy)

const defaultMarks2 = inputMarks ?? 50;
console.log(defaultMarks2); // 0 (0 is NOT null or undefined)

?? only falls back when the left side is null or undefined, not for 0, "", or false.

7.4 Combining Both

const centre = {
  name: "Coder & AccoTax",
  address: null
};

const city = centre.address?.city ?? "City not set";
console.log(city); // "City not set"
These two operators are extremely common in real-world codebases, especially when working with APIs where some properties may be missing.

ES6+ Quick Reference

  • Variables: let, const (block scoped)
  • Strings: backticks `...${}...` for templates
  • Destructuring: const {a,b} = obj;
  • Spread: [...arr], {...obj}
  • Rest: function f(...args) {}
  • Objects: shorthand props, method shorthand, computed keys
  • Safe access: obj?.prop?.nested
  • Defaults: a ?? b (only for null/undefined)
