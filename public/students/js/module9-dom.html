Step 1 · Understanding the DOM

1. What is the DOM?

The DOM (Document Object Model) is a tree-like representation of your HTML page that JavaScript can read and change. It turns elements like <h1>, <div>, and <button> into objects that you can access and modify.

1.1 Basic Idea

  • Browser reads your HTML file.
  • Browser builds a DOM tree in memory.
  • JavaScript talks to this tree: find nodes, change them, create/remove them.

1.2 Simple HTML → DOM Tree (Concept)

<body>
  <h1>Welcome</h1>
  <p>This is a paragraph.</p>
</body>

This becomes a tree:

  • document
  • → html
  • → body
  • → h1, p as children of body
Teach students to open DevTools → “Elements” tab and see the live DOM. It is the visual representation of the same tree.
Step 2 · Selecting Elements

2. Selecting Elements from the DOM

To change the page, first you must select an element. Think of this as “get a reference” to something on the page.

2.1 Example HTML

<h1 id="title">Welcome to DOM</h1>
<p class="tagline">Learn DOM step by step.</p>
<button id="change-btn">Change Title</button>

2.2 getElementById()

const title = document.getElementById("title");
console.log(title);

2.3 querySelector() & querySelectorAll()

const tagline = document.querySelector(".tagline"); // first match
const button  = document.querySelector("#change-btn"); // like getElementById

const allParagraphs = document.querySelectorAll("p"); // NodeList of all <p>
allParagraphs.forEach(p => console.log(p.textContent));
For modern JavaScript, querySelector and querySelectorAll are flexible and powerful. They accept any CSS selector (e.g., ".class", "#id", "div span").
Step 3 · Content & Styles

3. Changing Content & Styles Dynamically

Once you have a reference to an element, you can change its content and styling using properties like textContent, innerHTML, and style.

3.1 Changing Text

const title = document.getElementById("title");

title.textContent = "DOM Basics with Coder & AccoTax";

3.2 Changing HTML Inside an Element

const tagline = document.querySelector(".tagline");

tagline.innerHTML = "Learn <strong>DOM</strong> step by step.";
innerHTML can inject HTML and is powerful, but can be dangerous if used with user input (security: XSS). For simple text changes, prefer textContent.

3.3 Styling Elements

title.style.color = "#38bdf8";
title.style.backgroundColor = "#020617";
title.style.padding = "8px 12px";

3.4 Using classList (Cleaner for CSS)

title.classList.add("highlight");   // add a CSS class
title.classList.remove("old-title"); // remove a class
title.classList.toggle("hidden");    // add if not present, remove if present
Best practice: keep visual styling in CSS classes and use classList in JavaScript to switch classes on or off.
Step 4 · Creating & Removing

4. Creating, Appending & Removing Elements

4.1 Creating & Appending

const list = document.querySelector("#todo-list");

const li = document.createElement("li");
li.textContent = "Learn DOM basics";

list.appendChild(li);

4.2 append vs appendChild (Concept)

list.append("Text node", li); // append can take multiple nodes/strings

4.3 Removing Elements

// Modern way
li.remove();

// Older way
list.removeChild(list.firstElementChild);
Teaching idea: build a dynamic list where clicking a button adds a new <li> with text.
Step 5 · Attributes & Data

5. Working with Attributes, Classes & Data-Attributes

5.1 Attributes

<img id="logo" src="logo.png" alt="Coder & AccoTax logo">
const logo = document.getElementById("logo");

console.log(logo.getAttribute("src"));
logo.setAttribute("alt", "Coder & AccoTax JavaScript Course");

5.2 Classes with className & classList

logo.className = "logo-img rounded";
logo.classList.add("shadow");
logo.classList.remove("rounded");

5.3 Data Attributes & dataset

<button id="enroll-btn" data-course="javascript" data-level="basic">
  Enroll Now
</button>
const enrollBtn = document.getElementById("enroll-btn");

console.log(enrollBtn.dataset.course); // "javascript"
console.log(enrollBtn.dataset.level);  // "basic"
data-* attributes let you attach small bits of data to elements in a clean way, which JavaScript can easily read through dataset.
Step 6 · Mini-Project 1

6. DOM Mini-Project — Click Counter

Build a simple counter that increases or decreases when buttons are clicked. This shows real interaction between HTML, CSS, and JavaScript.

6.1 HTML Structure

<div id="counter-box">
  <h2>Counter</h2>
  <div id="count-display">0</div>
  <button id="decrement">-</button>
  <button id="increment">+</button>
</div>

6.2 JavaScript Logic

const display   = document.getElementById("count-display");
const incButton = document.getElementById("increment");
const decButton = document.getElementById("decrement");

let count = 0;

function updateDisplay() {
  display.textContent = count;
}

incButton.addEventListener("click", () => {
  count++;
  updateDisplay();
});

decButton.addEventListener("click", () => {
  count--;
  updateDisplay();
});
Extend this in class: • Add a “Reset” button • Prevent the count from going below 0 • Change color based on positive/negative value
Step 7 · Mini-Project 2

7. DOM Mini-Project — Show / Hide Details

A very common UI pattern: clicking a button to show or hide extra content (FAQ, “Read more”, etc.).

7.1 HTML Structure

<div class="faq-item">
  <button id="toggle-details">Show Details</button>
  <div id="details" style="display:none;">
    JavaScript DOM lets you control HTML structure, content, and styles dynamically.
  </div>
</div>

7.2 JavaScript Logic

const toggleBtn = document.getElementById("toggle-details");
const details   = document.getElementById("details");

toggleBtn.addEventListener("click", () => {
  const isHidden = details.style.display === "none";

  if (isHidden) {
    details.style.display = "block";
    toggleBtn.textContent = "Hide Details";
  } else {
    details.style.display = "none";
    toggleBtn.textContent = "Show Details";
  }
});
Professional improvement: instead of inline style="display:none;", hide via CSS class (e.g., .hidden { display:none; }) and toggle that class with classList.toggle("hidden").
Step 8 · Summary & Practice

8. Summary & Practice Tasks

8.1 Concept Checklist

  • Understand the DOM as a tree of nodes representing HTML.
  • Use getElementById, querySelector, querySelectorAll.
  • Change text, HTML, styles, and classes dynamically.
  • Create, append, and remove elements using JavaScript.
  • Work with attributes, classes, and data-attributes.
  • Build simple interactive features like counters and show/hide sections.

8.2 Practice Exercises

  1. Create an HTML list of tasks and use JavaScript to:
    • Add a new task when a button is clicked.
    • Remove the last task when another button is clicked.
  2. Build a “Change Theme” button that toggles a dark-mode class on the <body> element.
  3. Create a simple “character counter” for a text input: show how many characters the user has typed in real time under the input.
  4. Use data-* attributes on buttons (e.g., data-color) and change the background color of a box based on which button is clicked.
  5. Create an FAQ section with multiple questions. Clicking on a question should show/hide the answer below it.

Keep a dedicated practice file like module9-dom-basics.html + module9-dom.js and link them. Let students experiment directly in the browser.

DOM Method Quick Reference

  • Select: getElementById, querySelector, querySelectorAll
  • Content: textContent, innerHTML
  • Styles: style, classList.add/remove/toggle
  • Create: document.createElement
  • Insert: appendChild, append, prepend
  • Remove: element.remove(), parent.removeChild()
  • Attributes: getAttribute, setAttribute, dataset

Teacher Notes for Module 9

  • Always keep the HTML, CSS, and JS code side by side when teaching.
  • Ask students to open DevTools → “Elements” and “Console” panels together.
  • Show live edits in DevTools and compare with JavaScript-driven changes.
  • Let students modify text and styles from the console to build confidence.
  • Use lots of tiny experiments before full mini-projects.
