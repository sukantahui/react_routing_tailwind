Part 1 ‚Äî Object Basics & Property Access

Q1.
Which of the following best defines a JavaScript object?
  • A. A collection of indexed values only
  • B. A collection of key‚Äìvalue pairs, where keys are strings or Symbols
  • C. A collection of only functions
  • D. A special type of array
Correct Answer: B
A JavaScript object is a flexible collection of key‚Äìvalue pairs. Keys are typically strings (or Symbols) and values can be of any type including numbers, strings, arrays, or functions.
Q2.
Which is the correct way to define an object using literal syntax?
// Choose the valid object literal
  • A. const user = ( name: "Amit", age: 20 );
  • B. const user = [ name: "Amit", age: 20 ];
  • C. const user = { name: "Amit", age: 20 };
  • D. const user = object(name = "Amit", age = 20);
Correct Answer: C
Object literals are written using curly braces { } with key‚Äìvalue pairs inside. Option C is the correct syntax.
Q3.
Consider the object below. What will be logged?
const course = {
  title: "JavaScript",
  duration: "3 months",
  fee: 5000
};

console.log(course.title);
  • A. undefined
  • B. "title"
  • C. "JavaScript"
  • D. Error at runtime
Correct Answer: C
course.title accesses the title property of the object, whose value is the string "JavaScript".
Q4.
Which statement about dot vs bracket notation is correct?
  • A. Both notations are identical; there is never any difference.
  • B. Bracket notation is required for dynamic property names.
  • C. Dot notation is required for dynamic property names.
  • D. You cannot read properties using bracket notation.
Correct Answer: B
When the property name is stored in a variable (dynamic key), you must use bracket notation like obj[key]. Dot notation only works with fixed identifiers.
Q5.
What is the result of the following code?
const student = {
  name: "Neha",
  "course-name": "JavaScript Basic to Pro"
};

console.log(student.course-name);
  • A. "JavaScript Basic to Pro"
  • B. NaN
  • C. A ReferenceError
  • D. SyntaxError
Correct Answer: B (at runtime)
student.course-name is parsed as student.course - name (subtraction). student.course is undefined and name refers to the global name (often empty string), leading to NaN. The correct way is student["course-name"].
Q6.
How do you dynamically access a property whose name is stored in a variable?
const key = "fee";
const course = { title: "JS", fee: 5000 };
  • A. course.key
  • B. course["key"]
  • C. course(key)
  • D. course[key]
Correct Answer: D
Bracket notation allows using a variable: course[key] accesses the property whose name is stored in key, i.e. "fee".
Q7.
What will the following code print?
const centre = {
  name: "Coder & AccoTax",
  address: {
    city: "Barrackpore",
    state: "West Bengal"
  }
};

console.log(centre.address.city);
  • A. undefined
  • B. "address"
  • C. "Barrackpore"
  • D. An error
Correct Answer: C
centre.address is the nested object, and centre.address.city gives "Barrackpore".
Q8.
Which code correctly defines an array of student objects?
  • A. const students = { "Amit", "Neha" };
  • B. const students = [ "Amit" = 80, "Neha" = 90 ];
  • C. const students = [ { name: "Amit", marks: 80 }, { name: "Neha", marks: 90 } ];
  • D. const students = ( name: "Amit", marks: 80 );
Correct Answer: C
An array of objects is written with [ ] containing multiple { } object literals.
Q9.
Output of iterating over an array of objects using for...of:
const students = [
  { name: "Amit", marks: 85 },
  { name: "Neha", marks: 92 }
];

for (let s of students) {
  console.log(s.name);
}
  • A. Logs the whole objects twice
  • B. Logs "Amit" then "Neha"
  • C. Logs indexes 0 and 1
  • D. Throws an error
Correct Answer: B
for...of iterates over array elements. Each s is a student object, so s.name prints the names.
Q10.
What is the type of typeof {} in JavaScript?
  • A. "array"
  • B. "object"
  • C. "function"
  • D. "map"
Correct Answer: B
For a plain object literal {}, typeof returns the string "object".
Q11.
Which statement will correctly add a new property fee to course?
const course = { title: "JS" };
  • A. course.fee = 5000;
  • B. course("fee") = 5000;
  • C. course.add("fee", 5000);
  • D. course->fee = 5000;
Correct Answer: A
Properties can be added by assignment using dot notation: course.fee = 5000; or bracket notation.
Q12.
Which of the following deletes the age property from student?
const student = { name: "Amit", age: 18 };
  • A. remove student.age;
  • B. delete student.age;
  • C. student.age = null;
  • D. student.age.delete();
Correct Answer: B
The delete operator removes a property from an object. Setting to null only changes its value, not its existence.
Q13.
What will be logged in the following code?
const student = { name: "Neha", marks: 90 };
console.log("Result" in student);
  • A. true
  • B. false
  • C. "Result"
  • D. undefined
Correct Answer: B
The in operator checks if a property key exists. "Result" is not a property of student, so the result is false.
Q14.
Which loop is commonly used to iterate over an object's own enumerable properties?
  • A. for
  • B. for...of
  • C. for...in
  • D. while
Correct Answer: C
for...in iterates over enumerable property keys of an object. for...of is for iterable values (like arrays, strings).
Q15.
Predict the output of this snippet:
const student = { name: "Amit" };
student.course = "JavaScript";
student.name = "Rahul";

console.log(student.name, student.course);
  • A. "Amit" "JavaScript"
  • B. "Rahul" "JavaScript"
  • C. "Rahul" undefined
  • D. Error
Correct Answer: B
The name property is reassigned to "Rahul", and a new property course is added, so both values are available.
Q16.
What will console.log(Object.keys(student)) print?
const student = { name: "Neha", marks: 95, active: true };
console.log(Object.keys(student));
  • A. ["Neha", 95, true]
  • B. ["name", "marks", "active"]
  • C. ["student"]
  • D. []
Correct Answer: B
Object.keys returns an array of the object's own enumerable property names.
Q17.
Which expression safely reads a deeply nested property with bracket notation?
const centre = {
  address: {
    city: "Barrackpore"
  }
};
  • A. centre["address"]["city"]
  • B. centre["address"].city()
  • C. centre.city.address
  • D. centre(address, city)
Correct Answer: A
Both dot and bracket can be combined; here the fully bracketed centre["address"]["city"] is valid.
Q18.
Identify the error, if any:
const obj = {};
obj["full name"] = "Amit Kumar";

console.log(obj.full name);
  • A. No error, prints "Amit Kumar"
  • B. SyntaxError at obj.full name
  • C. ReferenceError: full is not defined
  • D. undefined
Correct Answer: B
obj.full name is invalid syntax; property names with spaces must be accessed using bracket notation: obj["full name"].
Q19.
What does Object.values(course) return?
const course = { title: "JS", fee: 5000, duration: "3 months" };
  • A. ["title","fee","duration"]
  • B. ["JS", 5000, "3 months"]
  • C. [["title","JS"],["fee",5000],["duration","3 months"]]
  • D. []
Correct Answer: B
Object.values returns an array of values of the object's own enumerable properties.
Q20.
What is logged here?
const student = { name: "Neha", marks: 90 };
student.marks = student.marks + 5;

console.log(student.marks);
  • A. 90
  • B. 95
  • C. undefined
  • D. NaN

Q21.
Which method converts an object into an array of [key, value] pairs?
  • A. Object.keys()
  • B. Object.values()
  • C. Object.entries()
  • D. Object.toArray()
Correct Answer: C
Object.entries() returns an array of [key, value] pairs.
Q22.
Which of these is NOT a good reason to group data into objects?
  • A. It keeps related data and behavior together.
  • B. It allows modelling real-world entities.
  • C. It makes code more organised and readable.
  • D. It guarantees faster execution time in all cases.
Correct Answer: D
Objects improve structure and readability but do not guarantee faster performance in every situation.
Q23.
Output?
const a = { value: 10 };
const b = a;
b.value = 20;

console.log(a.value);
  • A. 10
  • B. 20
  • C. undefined
  • D. Error
Correct Answer: B
Both a and b reference the same object in memory. Changing via b also affects a.
Q24.
Which spread syntax correctly clones a simple object?
const student = { name: "Amit", marks: 80 };
  • A. const copy = student;
  • B. const copy = clone(student);
  • C. const copy = { ...student };
  • D. const copy = new Object(student);
Correct Answer: C
The spread operator creates a shallow copy: { ...student }.
Q25.

  • A. Because JavaScript does not support any other type of array.
  • B. Because each object can represent a ‚Äúrecord‚Äù like a student, product, or invoice.
  • C. Because arrays cannot store primitive values.
  • D. Because objects cannot be stored individually.

Arrays of objects naturally model lists of entities (students, orders, courses), which is exactly how most real-world data is structured.
