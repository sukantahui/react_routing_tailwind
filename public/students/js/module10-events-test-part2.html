Section A — Input & Keyboard Events MCQs

Q1. Which event fires every time the value of an input changes?
A. change
B. input
C. keypress
D. keyup
Correct: B
Explanation: The input event fires immediately as the user types (or pastes/cuts), while change usually fires when the input loses focus or the user confirms the value.
Q2. Which event property gives the key pressed?
A. event.keyCode
B. event.key
C. event.code
D. event.which
Correct: B
Explanation: event.key returns the actual key value (e.g. "a", "Enter"). keyCode and which are deprecated in modern JavaScript.
Q3. Which keyboard event fires when the key is released?
A. keydown
B. keypress
C. keyup
D. input
Correct: C
Explanation: keyup fires after the user releases the key. keydown fires when the key is first pressed.
Q4. Which event is best for implementing “live search”?
A. keydown
B. keyup
C. click
D. submit
Correct: B
Explanation: With keyup, the input value has already been updated, so you can safely read the current text and filter results.
Q5. Which event triggers when you press Enter inside a text input?
A. keypress
B. input
C. keydown
D. both A and C
Correct: D
Explanation: Pressing Enter typically fires both keydown and keypress (though keypress is deprecated).
Q6. In modern JavaScript, which keyboard event is deprecated?
A. keydown
B. keyup
C. keypress
D. input
Correct: C
Explanation: keypress is deprecated and should not be used in new code. Prefer keydown / keyup.
Q7. event.target refers to:
A. The element listening to the event
B. The element which triggered the event
C. The window object
D. The parent element
Correct: B
Explanation: event.target is the element where the event actually occurred (e.g. the input that changed).
Q8. Which event fires repeatedly while a key is held down?
A. keypress
B. keyrepeat
C. keydown
D. keyup
Correct: C
Explanation: While a key is held, multiple keydown events fire repeatedly (auto-repeat behaviour).
Q9. Which method prevents default input behaviors?
A. event.prevent()
B. event.stop()
C. event.stopPropagation()
D. event.preventDefault()
Correct: D
Explanation: preventDefault() cancels the browser’s default action, such as form submit reload or character insertion.
Q10. Which input event fires when a user pastes text?
A. paste
B. input
C. change
D. both A and B
Correct: D
Explanation: Pasting triggers the paste event and then updates the field value, which also fires the input event.
Q11. Which property gives the physical key on the keyboard?
A. event.key
B. event.code
C. event.keyName
D. event.which
Correct: B
Explanation: event.code identifies the physical key (e.g. "KeyA"), independent of keyboard layout.
Q12. input vs change event — which statement is correct?
A. change fires instantly
B. input fires only on Enter key
C. input fires instantly, change fires on blur
D. Both are same
Correct: C
Explanation: input fires for every change (typing, paste, cut), while change fires when the element loses focus or the user commits the value.
Q13. Which event is best for a “character counter” under a textarea?
A. click
B. keyup
C. input
D. keydown
Correct: C
Explanation: input captures typing, backspace, paste and cut — perfect for live character counters.
Q14. Which event does NOT fire on non-character keys (e.g., Shift)?
A. keyup
B. keydown
C. keypress
D. input
Correct: C
Explanation: Historically, keypress fired only for character-producing keys, not for modifier keys like Shift.
Q15. event.target.value is available in:
A. input event
B. keyup event
C. keydown event
D. All of the above
Correct: D
Explanation: In all these handlers, event.target is still the input element, so event.target.value is accessible.
Q16. Which event is most reliable for detecting Backspace?
A. keypress
B. keydown
C. input
D. change
Correct: B
Explanation: keydown fires before the value is updated and works well for intercepting special keys like Backspace.
Q17. Which event is best for blocking numbers in a text-only field?
A. keydown
B. input
C. keyup
D. change
Correct: A
Explanation: Using keydown with event.preventDefault() allows you to stop the character before it appears in the field.
Q18. event.preventDefault() in a keydown handler:
A. blocks the key entirely
B. has no effect
C. only works on input event
D. only works on Enter key
Correct: A
Explanation: It cancels the default action, so the character is not inserted into the input.
Q19. Which event is best to detect paste from the right-click menu?
A. keydown
B. input
C. paste
D. keypress
Correct: C
Explanation: The paste event fires whenever content is pasted — even via context menu.
Q20. event.key === " " checks for:
A. Backspace
B. Tab
C. Spacebar
D. Delete
Correct: C
Explanation: A single space character literal corresponds to the Spacebar key.
Q21. Which input types do NOT fire the input event?
A. email
B. number
C. checkbox
D. All fire
Correct: D
Explanation: All these input types emit the input event when their value changes.
Q22. The fastest way to validate email live while typing is:
A. blur event
B. input event
C. change event
D. click event
Correct: B
Explanation: The input event lets you re-run validation on every keystroke for instant feedback.
Q23. Which keyboard event fires before the input value updates?
A. keydown
B. keyup
C. input
D. change
Correct: A
Explanation: keydown fires before the character is added to the input, so the value still reflects the previous state.
Q24. To detect Ctrl+S in a keyboard shortcut handler, which property must you check?
A. event.ctrlKey
B. event.shiftKey
C. event.altKey
D. event.metaKey
Correct: A
Explanation: event.ctrlKey is true when the Control key is held; combine it with event.key === "s".
Q25. Best event for updating “preview text” live as the user types:
A. keyup
B. input
C. change
D. submit
Correct: B
Explanation: input reacts to typing, pasting, deleting and cutting, giving the smoothest live preview experience.
