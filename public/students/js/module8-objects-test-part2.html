Part 2 ‚Äî Methods, this, and Constructor Functions

Click on each answer panel to reveal the explanation. Use this to deeply understand how methods, this, and constructor functions behave.

Q1.
In an object method, this usually refers to:
  • A. The global window object always
  • B. The function that is currently running
  • C. The object on which the method was called
  • D. undefined in all cases
Correct Answer: C
For a regular method call like obj.method(), this points to obj.
Q2.
What will be printed?
const student = {
  name: "Amit",
  greet() {
    console.log("Hi, I am " + this.name);
  }
};

student.greet();
  • A. "Hi, I am Amit"
  • B. "Hi, I am undefined"
  • C. "Hi, I am student"
  • D. Error
Correct Answer: A
Inside greet, this refers to student, so this.name is "Amit".
Q3.
What happens if you copy a method and lose the original object context?
const obj = {
  value: 10,
  show() {
    console.log(this.value);
  }
};

const fn = obj.show;
fn();
  • A. Prints 10
  • B. Prints undefined (or window.value in browsers)
  • C. Throws a TypeError
  • D. Always prints 0
Correct Answer: B
When fn() is called as a plain function, this no longer refers to obj. In strict mode it is undefined.
Q4.
Why are arrow functions usually not recommended for object methods using this?
  • A. Because arrow functions are slower
  • B. Because arrow functions do not have their own this
  • C. Because arrow functions cannot return values
  • D. Because arrow functions cannot be used inside objects
Correct Answer: B
Arrow functions capture this from the surrounding scope, so for methods where this should be the object, a normal function is better.
Q5.
Which is a valid constructor function by convention?
  • A. function student(name) { ... }
  • B. function Student(name) { ... }
  • C. constructor Student(name) { ... }
  • D. Student(name) => { ... }
Correct Answer: B
By convention constructor functions start with a capital letter, e.g. Student.
Q6.
What does the new keyword do with constructor functions?
  • A. Creates a completely new data type
  • B. Creates an object, sets its prototype, runs the constructor with this bound, and returns it
  • C. Copies methods from one object to another
  • D. Only allocates memory without calling the function
Correct Answer: B
new automates object creation, prototype linking, and binding this to the new instance.
Q7.
Output?
function Student(name) {
  this.name = name;
}

const s1 = new Student("Neha");
console.log(s1.name);
  • A. undefined
  • B. "Student"
  • C. "Neha"
  • D. Error
Correct Answer: C
new Student("Neha") creates a new object with name set to "Neha".
Q8.
What will be logged if Student is called without new?
function Student(name) {
  this.name = name;
}

const s = Student("Amit");
console.log(s);
  • A. A Student object
  • B. undefined
  • C. A runtime error
  • D. An empty object
Correct Answer: B (and this is wrong)
Without new, the function returns undefined (no return), and this refers to global / undefined depending on strict mode, which is not what we want.
Q9.
Which is the correct way to add a shared method via prototype?
function Student(name) {
  this.name = name;
}
  • A. Student.greet = function(){...}
  • B. Student.prototype.greet = function(){...}
  • C. Student().prototype.greet = ...
  • D. new Student().greet = ...
Correct Answer: B
Methods on Student.prototype are shared by all instances created using new Student().
Q10.
Predict the output:
function Student(name) {
  this.name = name;
}
Student.prototype.greet = function() {
  console.log("Hi " + this.name);
};

const s1 = new Student("Amit");
const s2 = new Student("Neha");

console.log(s1.greet === s2.greet);
  • A. false
  • B. true
  • C. Throws an error
  • D. undefined
Correct Answer: B
Both s1 and s2 share the same greet function from the prototype, so they are strictly equal.
