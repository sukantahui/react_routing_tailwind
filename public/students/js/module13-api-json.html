Step 1 · Foundations

1. What Are APIs & JSON?

In modern frontend development, your JavaScript rarely works alone. It usually talks to a backend or a public API over HTTP, and they exchange data using JSON.

1.1 API in Simple Words

  • API → Application Programming Interface.
  • In web context: a URL you call to get or send data (e.g. /api/students).
  • Your JavaScript is the client, server (or service) is the provider.
Example API endpoints
GET /api/students → returns list of students (JSON)
POST /api/students → create a new student (send JSON body)
GET /api/students/5 → details of student with id=5

1.2 What Is JSON?

  • JSON: JavaScript Object Notation.
  • Text format for structured data, based on JavaScript object syntax.
  • Easy to read for humans and easy to parse for machines.
{
  "name": "Coder & AccoTax",
  "city": "Barrackpore",
  "courses": ["JavaScript", "Python", "Accounts"],
  "active": true,
  "students": 120
}
Inside JS, JSON becomes normal objects and arrays. But when sent over network, it is plain text.
Step 2 · JSON in JavaScript

2. Reading & Writing JSON

Two main tools: JSON.parse() and JSON.stringify().

2.1 Converting JSON Text → JavaScript Object

const jsonText = '{"name":"Amit","course":"JavaScript","marks":90}';

const student = JSON.parse(jsonText); // now a real object
console.log(student.name);   // Amit
console.log(student.course); // JavaScript

2.2 Converting JS Object → JSON Text

const course = {
  title: "JavaScript from Basic to Pro",
  centre: "Coder & AccoTax",
  online: true
};

const json = JSON.stringify(course);
console.log(json);
// {"title":"JavaScript from Basic to Pro","centre":"Coder & AccoTax","online":true}

2.3 Pretty JSON (For Debugging)

const pretty = JSON.stringify(course, null, 2);
console.log(pretty);
/*
{
  "title": "JavaScript from Basic to Pro",
  "centre": "Coder & AccoTax",
  "online": true
}
*/
JSON limitations:
  • No functions, undefined, or comments allowed.
  • Keys and string values must use double quotes " ".
Step 3 · fetch() GET

3. fetch() Basics — GET Requests

fetch(url) is the main function to call APIs from the browser. It returns a Promise that resolves to a Response object.

3.1 First Example with .then()

fetch("https://jsonplaceholder.typicode.com/posts/1")
  .then(response => response.json())      // convert JSON text to JS object
  .then(data => {
    console.log("Post title:", data.title);
  })
  .catch(error => {
    console.error("Request failed:", error);
  });

3.2 Same Example with async/await

async function loadPost(id) {
  try {
    const response = await fetch(
      `https://jsonplaceholder.typicode.com/posts/${id}`
    );
    const data = await response.json();
    console.log("Post:", data);
  } catch (err) {
    console.error("Network error:", err);
  }
}

loadPost(1);
For your students: Start with .then(), then convert the same code to async/await to show how readable it becomes.
Step 4 · HTTP Methods

4. HTTP Methods: GET vs POST (With JSON)

The most common methods you use from frontend:

  • GET → retrieve data (no body).
  • POST → send data to create something.
  • (Later you’ll also see PUT, PATCH, DELETE.)

4.1 Sending a JSON Body with POST

async function createPost() {
  const newPost = {
    title: "My first API post",
    body: "Learning APIs & JSON at Coder & AccoTax",
    userId: 1
  };

  try {
    const response = await fetch(
      "https://jsonplaceholder.typicode.com/posts",
      {
        method: "POST",
        headers: {
          "Content-Type": "application/json"
        },
        body: JSON.stringify(newPost)
      }
    );

    const data = await response.json();
    console.log("Created:", data);
  } catch (err) {
    console.error("Error creating post:", err);
  }
}
Key points:
  • Always set "Content-Type": "application/json" when sending JSON.
  • Use JSON.stringify() for the request body.

4.2 Checking for HTTP Errors

async function safeFetch(url, options = {}) {
  const response = await fetch(url, options);
  if (!response.ok) {
    // response.status in 400–599 range
    throw new Error(`HTTP ${response.status}`);
  }
  return response.json();
}

Now you can use safeFetch everywhere and handle errors with try...catch.

Step 5 · UI Binding

5. Displaying API Data on the Page

Now we connect APIs to the DOM: convert JSON into HTML elements for users to see.

5.1 HTML Skeleton (List of Posts)

<div id="posts-app">
  <h2>Latest Posts</h2>
  <button id="load-posts">Load Posts</button>
  <div id="posts-status"></div>
  <ul id="posts-list"></ul>
</div>

5.2 JavaScript to Render Posts

const loadBtn    = document.getElementById("load-posts");
const statusBox  = document.getElementById("posts-status");
const postsList  = document.getElementById("posts-list");

function setStatus(message) {
  statusBox.textContent = message;
}

function renderPosts(posts) {
  postsList.innerHTML = ""; // clear previous
  posts.forEach(post => {
    const li = document.createElement("li");
    li.innerHTML = `<strong>${post.title}</strong><br>${post.body}`;
    postsList.appendChild(li);
  });
}

async function loadPosts() {
  try {
    setStatus("Loading...");
    postsList.innerHTML = "";

    const response = await fetch(
      "https://jsonplaceholder.typicode.com/posts?_limit=5"
    );
    if (!response.ok) throw new Error("HTTP " + response.status);

    const data = await response.json();
    renderPosts(data);
    setStatus("Loaded successfully");
  } catch (err) {
    setStatus("Error loading posts");
    postsList.innerHTML = 
      `<li style="color:#f97373">${err.message}</li>`;
  }
}

loadBtn.addEventListener("click", loadPosts);
Teaching point: separate renderPosts (UI logic) from data fetching logic for better structure.

6.3 JavaScript Logic (Medium-Level)

const searchInput = document.getElementById("search-input");
const reloadBtn   = document.getElementById("reload-btn");
const statusSpan  = document.getElementById("api-status");
const cardsBox    = document.getElementById("cards-container");

let allCourses = []; // will store full list from API

function setApiStatus(text) {
  statusSpan.textContent = text;
}

function createCourseCard(course) {
  // Assuming API returns fields like title, description, category
  return `
    <div class="course-card">
      <h4>${course.title}</h4>
      <p>${course.description || "No description provided."}</p>
      <p style="margin-top:4px;font-size:0.8rem;">
        <strong>Category:</strong> ${course.category || "General"}
      </p>
    </div>
  `;
}

function renderCourses(list) {
  if (!list.length) {
    cardsBox.innerHTML =
      `<p style="color:#9ca3af">No courses match your search.</p>`;
    return;
  }
  cardsBox.innerHTML = list.map(createCourseCard).join("");
}

function applyFilter() {
  const query = searchInput.value.toLowerCase().trim();
  if (!query) {
    renderCourses(allCourses);
    return;
  }
  const filtered = allCourses.filter(course =>
    course.title.toLowerCase().includes(query)
  );
  renderCourses(filtered);
}

async function loadCourses() {
  try {
    setApiStatus("Loading...");
    cardsBox.innerHTML = "";

    // Example endpoint (replace with your own API if needed)
    const response = await fetch(
      "https://jsonplaceholder.typicode.com/posts?_limit=12"
    );
    if (!response.ok) {
      throw new Error("HTTP " + response.status);
    }

    const data = await response.json();

    // Adapt JSONPlaceholder fields to our course structure
    allCourses = data.map(item => ({
      title: item.title,
      description: item.body,
      category: "API Demo"
    }));

    renderCourses(allCourses);
    setApiStatus(`Loaded ${allCourses.length} items`);
  } catch (err) {
    setApiStatus("Error while loading");
    cardsBox.innerHTML = 
      `<p style="color:#f97373">${err.message}</p>`;
  }
}

// EVENT HANDLERS
reloadBtn.addEventListener("click", loadCourses);
searchInput.addEventListener("input", applyFilter);

// Initial load on page open
loadCourses();
Extension ideas:
  • Add a dropdown filter by category (if API provides categories).
  • Add pagination buttons or “Load more”.
  • Show a “No results” message clearly when filter matches 0 items.

APIs & JSON Quick Reference

  • JSON → JS: JSON.parse(text)
  • JS → JSON: JSON.stringify(obj)
  • GET: fetch(url)
  • POST JSON:
    fetch(url, { method:"POST", headers:{ "Content-Type":"application/json" }, body: JSON.stringify(data) })
  • Check error: if (!response.ok) throw ...
  • Parse JSON: const data = await response.json();
  • Pattern: “Fetch → Parse → Render in DOM → Handle errors”.
