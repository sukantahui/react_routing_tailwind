Section A — Output Questions (Spread, Rest, Object Literals)

Q1. Predict the output: 
const arr1 = [1, 2];
const arr2 = [3, 4];
const result = [...arr1, ...arr2];
console.log(result);
Answer: [1, 2, 3, 4]
Spread expands arrays into individual elements.
Q2. Output? 
const base = { a: 1, b: 2 };
const extra = { b: 5, c: 10 };
const combined = { ...base, ...extra };
console.log(combined);
Answer: { a: 1, b: 5, c: 10 }
Properties in extra override those in base.
Q3. What does this log? 
function sum(...nums) {
  return nums.reduce((a, c) => a + c, 0);
}
console.log(sum(1, 2, 3));
Answer: 6
Rest collects all parameters into an array.
Q4. Output? 
const [first, ...others] = [10, 20, 30, 40];
console.log(first);
console.log(others);
10
[20, 30, 40]
Q5. Output? 
const obj = { x: 1, y: 2 };
const clone = { ...obj };
clone.x = 99;
console.log(obj.x);
Answer: 1
Spread creates a shallow copy—modifying clone does not affect original.

Section D — Conceptual Questions

Q16. Why is spread safer than Object.assign() for copying objects?
Because spread syntax is cleaner and avoids mutating the target object. Object.assign() mutates the first argument.
Q17. Explain the difference between spread and rest in one sentence.
Spread expands values; rest collects values.
Q18. Why use method shorthand in objects?
It makes code shorter, cleaner, and is the modern ES6 standard for object methods.
Q19. What happens if two objects merged via spread have the same key?
The value from the later object overrides the earlier one.
Q20. Why is rest parameter useful in functions?
It allows functions to accept unlimited arguments without using arguments object.
Q21. Can rest be used in object destructuring? 
const {a, ...rest} = obj;
Yes, it collects remaining properties.
Q22. Why is spread often used in React?
It helps easily clone state, pass props, and update nested objects immutably.
Q23. Example of computed property name? 
const key="name";
const obj={ [key]:"Amit" };
Q24. Give a real-life use of spread in arrays.
Copying arrays before modifying them to maintain immutability.
Q25. Why are ES6 spread/rest essential for modern JS developers?
They simplify merging, cloning, parameter handling, and make code more expressive, especially in frameworks like React, Node.js, and modern API design.
