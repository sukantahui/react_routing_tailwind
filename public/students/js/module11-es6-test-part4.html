Section A — Output Questions (Optional Chaining & ??)

Q1. Predict the output: 
const student = {
  name: "Amit",
  contact: {
    email: "amit@example.com"
  }
};

console.log(student.contact?.email);
console.log(student.address?.city);
Answer:
"amit@example.com"
undefined
Explanation: contact exists, so email is read normally. address is undefined, so address?.city becomes undefined without throwing an error.
Q2. What will this log? 
const marks = null;
const finalMarks = marks ?? 50;
console.log(finalMarks);
Answer: 50
Explanation: ?? uses right side only if left is null or undefined. Here it is null, so 50 is used.
Q3. Predict the output: 
const input = 0;

const a = input || 100;
const b = input ?? 100;

console.log(a, b);
Answer: 100 0
Explanation: || treats 0 as falsy and chooses 100. ?? only treats null/undefined as “empty”, so it keeps 0.
Q4. What is logged? 
const centre = {
  name: "Coder & AccoTax",
  address: {
    city: "Barrackpore"
  }
};

console.log(centre.address?.city ?? "City NA");
console.log(centre.branch?.city ?? "City NA");
Answer:
"Barrackpore"
"City NA"
Explanation: First time, city exists. Second time, branch is undefined, optional chaining returns undefined, then ?? uses the fallback.
Q5. What is printed? 
const config = {
  theme: null
};

const theme = config.theme ?? "dark";
console.log(theme);
Answer: "dark"
Explanation: theme is explicitly null, so ?? uses default "dark".

Section D — Conceptual & Short Answer (Exam Style)

Q16. Question: Why is optional chaining very helpful when working with API responses in real projects?
Sample Answer:
API responses often have missing or nested fields. Optional chaining lets you safely access deep properties without checking each level manually. It avoids TypeError: Cannot read properties of undefined and keeps the code cleaner, especially when dealing with dynamic or partially available data.
Q17. Question: Explain with an example why a ?? b is safer than a || b when 0 is a valid value.
Sample Answer:
|| treats 0 as falsy. So 0 || 10 becomes 10, which may be wrong if 0 is a valid value.
?? only treats null and undefined as “empty”. Example: const marks = 0; const result = marks ?? 50; Here result is 0, which is what we want.
Q18. Question: For a dashboard at Coder & AccoTax, you receive: 
response.data.user.profile.city
Sometimes user or profile is missing. Show how you’d safely read city with a default `"Unknown City"` using ES6+. 
const city = response?.data?.user?.profile?.city ?? "Unknown City";
Q19. Question: Briefly describe how destructuring, spread and rest together make your JavaScript code more expressive.
Sample Answer:
Destructuring lets us unpack values directly into variables, making code shorter. Spread lets us easily copy and merge arrays/objects without mutation. Rest lets us collect “the rest” of the values into an array or object, which is great for flexible function parameters. Together they replace a lot of manual loops and if-conditions with clean, declarative code.
Q20. Question: In the JavaScript course at Coder & AccoTax, why is it important for students to understand ?. and ?? before they move to React or Node.js?
Sample Answer:
Modern frameworks and Node.js codebases use these operators everywhere for safe property access and defaults. If a student doesn’t understand ?. and ??, they will struggle to read and debug real-world code. Mastering them early makes the transition to React, APIs and backend projects much smoother.
Q21. Question: Write a small code snippet that uses template literals, destructuring and optional chaining together. 
const student = {
  name: "Neha",
  course: { title: "ES6+ Features" }
};

const { name, course } = student;
const title = course?.title ?? "Unknown Course";

console.log(`Student: ${name}, Course: ${title}`);
Q22. Question: What is one danger of “over-using” optional chaining?
Sample Answer:
Over-using optional chaining can hide real bugs. If you always write obj?.prop?.subProp, you might silently get undefined instead of noticing that some part of your data is never set correctly. Use it thoughtfully where missing data is expected, not to hide all errors.
Q23. Question: Compare default parameters and ?? for providing default values. When would you use each?
Sample Answer:
Default parameters provide defaults when arguments are undefined or not passed. ?? provides defaults at expression level when a value is null or undefined. Use default parameters in function signatures to set normal defaults; use ?? inside function bodies or other expressions when data comes from variables, APIs or calculations.
Q24. Question: Give a short example where || gives a wrong default, but ?? is correct for a marks field. 
const marks = 0;

// Wrong: 0 replaced with 50
const m1 = marks || 50;   // 50

// Correct: 0 is kept
const m2 = marks ?? 50;   // 0
Explanation: A student can actually get 0 marks, so 0 is valid and should not be replaced by the default.
Q25. Question: Imagine you are preparing students at Coder & AccoTax for interviews. How would you summarise Module 11 (ES6+) in 3–4 lines to show its importance?
Sample Answer:
Module 11 teaches the modern JavaScript that companies actually use: let/const, template literals, destructuring, spread/rest, enhanced objects, optional chaining and ??. These features appear in almost every React, Node.js and frontend project. Knowing them well makes your code shorter, safer, easier to read, and helps you clear technical interviews confidently.
