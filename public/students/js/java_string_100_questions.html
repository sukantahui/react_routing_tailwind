Part A ‚Äî Basics of String

Fundamental definition, creation and core properties.

  1. Q1 What is a String in Java?
    Answer: A String is an object of the java.lang.String class that represents an immutable sequence of characters.
    In Java, strings are not primitive; they are reference types stored as objects and provide many built‚Äëin methods for text processing.
  2. Q2 Which package contains the String class?
    Answer: The java.lang package.
    Core classes like String, Math, System etc. are stored in java.lang, which is imported automatically.
  3. Q3 How do you write a string literal in Java?
    Answer: By enclosing characters in double quotes, for example "Hello".
    Single quotes are used for char, while double quotes create a String literal.
  4. Q4 Is String a primitive type or a reference type?
    Answer: String is a reference type (class type), not a primitive.
    Although we use it very often like a basic type, it is internally an object of the String class.
  5. Q5 What does ‚Äúimmutable‚Äù mean for a String?
    Answer: Immutable means that once a String object is created, its content cannot be changed.
    Any operation that seems to modify a string (like concatenation) actually creates a new String object with the updated value.
  6. Q6 Write two common ways to create a String object.
    Answer: (1) Using a literal: String s = "Hello"; (2) Using new: String s = new String("Hello");.
    Both create String objects but behave differently in terms of memory (SCP vs Heap).
  7. Q7 Where are string literals stored?
    Answer: In a special memory area called the String Constant Pool (SCP).
    The SCP allows Java to reuse common string literals instead of creating duplicates, which saves memory.
  8. Q8 Does new String("Hello") always create a new object?
    Answer: Yes, it creates a new String object in heap memory even if the same literal exists in the pool.
    The literal "Hello" may also be in the SCP, but the new keyword ensures a separate object is created.
  9. Q9 Can a String hold the value null?
    Answer: Yes, a String reference can be null, meaning it does not point to any object.
    Using any method on a null String reference will cause a NullPointerException.
  10. Q10 Write one advantage of making String immutable.
    Answer: It enables safe sharing and caching of string objects, which improves performance and security.
    Since strings cannot change, the same instance can be reused by many references without risk of accidental modification.

Part B ‚Äî Core String Methods

Frequently used methods and their purposes.

  1. Q11 Which method returns the length of a String?
    Answer: length() method.
    For example, "Hello".length() returns 5 because the string has 5 characters.
  2. Q12 What does charAt(int index) do?
    Answer: It returns the character present at the given index (0‚Äëbased).
    For example, "Java".charAt(1) returns 'a'.
  3. Q13 What does substring(2,5) return for String s = "Programming"?
    Answer: It returns "ogr".
    Index 2 is 'o' and index 4 is 'r'; substring excludes the character at end index 5.
  4. Q14 Which method is used to compare two strings by content (case‚Äësensitive)?
    Answer: equals(String other).
    equals() returns true only when the sequences of characters in both strings are exactly the same.
  5. Q15 How do you compare two strings without considering case?
    Answer: Use equalsIgnoreCase(String other).
    For example, "JAVA".equalsIgnoreCase("java") returns true.
  6. Q16 What is returned by compareTo() when two strings are equal?
    Answer: It returns 0.
    A zero result indicates that both strings are lexicographically equal.
  7. Q17 Name methods that convert a string to upper‚Äëcase and lower‚Äëcase.
    Answer: toUpperCase() and toLowerCase().
    They return new String objects; the original string remains unchanged due to immutability.
  8. Q18 What does the trim() method do?
    Answer: It removes leading and trailing white spaces from a string.
    It does not remove spaces present in the middle of the string.
  9. Q19 Which method finds the first index of a given character in a string?
    Answer: indexOf(char ch).
    If the character is absent, it returns -1, which is useful for search logic.
  10. Q20 Which method replaces all occurrences of a character in a string?
    Answer: replace(char oldChar, char newChar).
    For example "banana".replace('a','*') returns "b*n*n*".

Part C ‚Äî Operations & Common Patterns

More methods, searching, splitting and joining.

  1. Q21 How can you check if a string starts with a particular prefix?
    Answer: Use startsWith(String prefix).
    Example: "Welcome".startsWith("We") returns true.
  2. Q22 Which method checks whether a string ends with a particular suffix?
    Answer: endsWith(String suffix).
    Example: "file.txt".endsWith(".txt") returns true.
  3. Q23 How do you split a sentence into words using space as delimiter?
    Answer: String[] words = s.split(" ");
    split() returns an array of strings obtained by breaking the original string at each occurrence of the delimiter.
  4. Q24 What is returned by indexOf("is") for "This is Java"?
    Answer: It returns 2.
    Indexing starts from 0; 'T' is 0, 'h' is 1, first 'i' of "is" is at index 2.
  5. Q25 How do you obtain a character array from a string?
    Answer: Use toCharArray() method.
    Example: char[] ch = s.toCharArray(); creates an array of all characters in the string.
  6. Q26 Which method checks whether a string is empty?
    Answer: isEmpty().
    It returns true if the string length is 0; note that null is different from empty string.
  7. Q27 Write the output of "Java".substring(1).
    Answer: "ava".
    Calling substring(begin) returns all characters from index begin to the end of the string.
  8. Q28 How can you remove all spaces from a string?
    Answer: Use replace(" ", "").
    Example: "a b c".replace(" ","") results in "abc".
  9. Q29 Which method can convert a value of another type (like int) into a string?
    Answer: String.valueOf().
    Example: String s = String.valueOf(123); gives "123" as a string.
  10. Q30 What does lastIndexOf('a') return for "banana"?
    Answer: It returns 5.
    The last 'a' in "banana" appears at index positions 1, 3 and 5; so lastIndexOf returns 5.

Part D ‚Äî equals(), == and Immutability

Conceptual understanding and small code‚Äëbased questions.

  1. Q31 What does equals() compare in a String?
    Answer: It compares the content (sequence of characters) of two strings.
    If both strings have identical characters in the same order, equals() returns true.
  2. Q32 What does the == operator compare for String references?
    Answer: It compares whether two String references point to the same object in memory.
    Even if contents are same, == may be false when the references refer to different objects.
  3. Q33 Consider String a = "Hello"; String b = "Hello"; Is a == b true?
    Answer: Yes, it is true.
    Both a and b refer to the same literal in the String Constant Pool, so their references are equal.
  4. Q34 Consider String a = "Hello"; String b = new String("Hello"); Is a == b true?
    Answer: No, it is false.
    a refers to the literal in SCP, while b refers to a heap object created by new; they are different references.
  5. Q35 In the same example, is a.equals(b) true?
    Answer: Yes, a.equals(b) is true.
    Both strings have the same sequence of characters "Hello", so their contents are equal.
  6. Q36 What will be the output of System.out.println("Java" + 10 + 20);?
    Answer: The output is Java1020.
    Left‚Äëto‚Äëright evaluation causes "Java" + 10 to form "Java10", then "Java10" + 20 gives "Java1020".
  7. Q37 What will be the output of System.out.println(10 + 20 + "Java");?
    Answer: The output is 30Java.
    First 10 + 20 is evaluated as integer addition (30), then 30 is concatenated with "Java".
  8. Q38 Why does s.toUpperCase() not change the original string s?
    Answer: Because String is immutable; toUpperCase() returns a new String object instead.
    The original reference still points to the old content; to keep changes, we must assign the result to a variable.
  9. Q39 What concept allows the same string literal to be reused in many places without extra memory?
    Answer: String Constant Pool (SCP) together with immutability.
    Because strings cannot change, a single instance can be safely reused across the program.
  10. Q40 Is String s = ""; the same as String s = null;?
    Answer: No, they are not the same.
    "" is an empty string object with length 0, while null means the reference does not point to any object at all.

  9. Q49 What is the output of:

Part F ‚Äî Logic & Short Programs Using Strings

Common exam‚Äëstyle coding tasks using String methods.

  1. Q51 How would you count the number of vowels in a given string?
    Answer: Loop through the string with length() and charAt(), and count characters that belong to "aeiouAEIOU".
    This uses character checking inside a loop, a very standard board‚Äëlevel String program.
  2. Q52 How can you reverse a string without using built‚Äëin reverse methods?
    Answer: Start from the last index length()-1, append each character to a new string in a loop until index 0.
    This builds the reverse order manually; it is also a good practice question on loops plus strings.
  3. Q53 How do you check if a string is a palindrome?
    Answer: Reverse the string and compare original and reversed using equals() or equalsIgnoreCase().
    If both are equal, the string reads the same forwards and backwards, so it is a palindrome.
  4. Q54 How can you count words in a sentence using String methods?
    Answer: Use split(" ") to split on spaces and then use length of the resulting array.
    Example: String[] w = s.split(" "); int count = w.length;.
  5. Q55 How would you count the frequency of a given character inside a string?
    Answer: Traverse the string with a loop, compare each character with the target, and increment a counter when they match.
    This demonstrates basic traversal using charAt() and conditional checking.
  6. Q56 How can you extract the first letter of each word from a sentence?
    Answer: Split the sentence using split(" ") and take word.charAt(0) from each word.
    You can concatenate these initials to form an acronym or short code for the sentence.
  7. Q57 Write a logic to change all vowels in a string to '*'.
    Answer: Use a loop, build a new string: if character is vowel, append '*', else append the original character.
    Because strings are immutable, you constantly build a new result string instead of modifying the original.
  8. Q58 How can you check if a string only contains digits?
    Answer: Traverse each character and check with Character.isDigit(ch); if all are digits, the string is numeric.
    This is often used before converting a string to an integer to avoid exceptions.
  9. Q59 How do you find the longest word in a sentence?
    Answer: Split the sentence into words, then loop through the array and track the word with maximum length().
    This question combines split() and basic loop logic, very common in school projects.
  10. Q60 How can you swap the case of each letter (upper to lower, lower to upper) in a string?
    Answer: Loop through the string; if Character.isUpperCase(ch), convert to lower, else if lower, convert to upper and append to a new string.
    This shows use of character‚Äëwise methods with strings, common in loop exercises.

Part G ‚Äî StringBuffer, StringBuilder & Advanced Points

Related classes for mutable strings and conceptual comparisons.

  1. Q61 What is the main limitation of using String for repeated modifications?
    Answer: Because String is immutable, each modification creates a new object, which can be slow and memory‚Äëconsuming.
    In loops or heavy concatenation, using String may lead to performance issues and unnecessary garbage objects.
  2. Q62 Which two classes are provided by Java for mutable character sequences?
    Answer: StringBuffer and StringBuilder.
    They allow modification of the underlying character sequence without creating new objects every time.
  3. Q63 Which is thread‚Äësafe: StringBuffer or StringBuilder?
    Answer: StringBuffer is thread‚Äësafe.
    Its methods are synchronized, making it safe for use in multi‚Äëthreaded programs, but slightly slower than StringBuilder.
  4. Q64 Which is faster in single‚Äëthreaded programs: StringBuffer or StringBuilder?
    Answer: StringBuilder is generally faster.
    Because it is not synchronized, there is less overhead, making it suitable for single‚Äëthreaded use cases.
  5. Q65 How do you create a StringBuilder with initial text "Java"?
    Answer: StringBuilder sb = new StringBuilder("Java");
    Similarly, StringBuffer can be created by new StringBuffer("Java").
  6. Q66 Which methods of StringBuilder are commonly used for modification?
    Answer: append(), insert(), delete(), reverse(), replace().
    These methods operate on the same underlying buffer instead of creating new objects each time.
  7. Q67 How can you convert a StringBuilder back to a String?
    Answer: Use the toString() method.
    Example: String res = sb.toString(); returns an immutable String with the builder‚Äôs content.
  8. Q68 In which situation would you prefer String over StringBuilder?
    Answer: When the text is not going to change, such as messages, labels, constant values.
    String is simpler, safe, and optimised for constant or rarely‚Äëchanging values.
  9. Q69 What does the capacity() method of StringBuilder return?
    Answer: It returns the current capacity of the builder‚Äôs internal character buffer.
    Capacity may be greater than the current length; it grows automatically as more characters are added.
  10. Q70 Does the String class have methods like append() and insert()?
    Answer: No, these are methods of StringBuffer and StringBuilder, not of String.
    String is immutable, so it focuses on returning new values instead of direct in‚Äëplace modification methods.

Part H ‚Äî Mixed Conceptual, MCQ & True / False

Quick‚Äëfire questions for revision and exams.

  1. Q71 True or False: String objects can be changed after they are created.
    Answer: False.
    Strings are immutable; modifications always create new objects instead of changing the old one.
  2. Q72 True or False: String is present in java.util package.
    Answer: False.
    The correct package is java.lang.
  3. Q73 True or False: "abc" == "abc" is always true in Java.
    Answer: True, for literals in the same class.
    Both refer to the same literal in the SCP; however, avoid == for general content comparison.
  4. Q74 True or False: String s = null; means an empty string.
    Answer: False.
    null means no object; an empty string would be "" with length 0.
  5. Q75 MCQ: Which of the following is NOT a valid way to declare a String?
    A) String s = "Java";
    B) String s = new String("Java");
    C) String s = 'J';
    D) String s = null;
    Answer: Option C.
    Single quotes denote a char, not a String; the other options are valid declarations.
  6. Q76 MCQ: Which method would you use to check if two strings are equal ignoring case?
    A) equals()
    B) equalsIgnoreCase()
    C) compareTo()
    D) match()
    Answer: Option B.
    equalsIgnoreCase() is specifically designed for case‚Äëinsensitive comparisons.
  7. Q77 MCQ: The expression "Java" + 5 results in:
    A) Compilation error
    B) String "Java5"
    C) Integer 5
    D) Runtime error
    Answer: Option B.
    Number 5 is converted to string and concatenated, giving "Java5".
  8. Q78 MCQ: Which of these is a valid String method?
    A) length (without brackets)
    B) size()
    C) length()
    D) getLength()
    Answer: Option C.
    In Java, length() (with parentheses) is the method that returns the length of a string.
  9. Q79 MCQ: Which expression correctly concatenates two strings s1 and s2?
    A) s1.concat(s2)
    B) s1 + s2
    C) Both A and B
    D) None of these
    Answer: Option C.
    Both concat() and + create a new String containing the combined text.
  10. Q80 True or False: After executing s.concat("abc"); without assignment, s is changed.
    Answer: False.
    Because String is immutable, the result of concat() must be assigned back (e.g. s = s.concat("abc");) to see the change.

Part I ‚Äî Short Theory / Exam‚ÄëStyle Questions

Direct answer questions often asked in written papers.

  1. Q81 Define a String in one sentence suitable for an exam.
    Answer: A String is an immutable object in Java that represents a sequence of characters and is an instance of the java.lang.String class.
    This definition highlights both the object nature and the immutability of strings, which examiners expect.
  2. Q82 Write any two differences between String and StringBuilder.
    Answer: String is immutable while StringBuilder is mutable; String is thread‚Äësafe by design, whereas StringBuilder is not synchronized and faster in single‚Äëthreaded scenarios.
    Mentioning immutability and performance/thread‚Äësafety differences usually fetches full marks.
  3. Q83 Why is it recommended to use equals() instead of == for comparing strings?
    Answer: Because equals() compares the content, whereas == only compares memory addresses and may give false even when strings look same.
    This is a classic conceptual doubt; using equals() avoids bugs related to different string objects.
  4. Q84 What is the return type of length() method of String?
    Answer: int.
    The number of characters in a string is always represented as an integer in Java.
  5. Q85 Mention one situation where trim() is very useful.
    Answer: When reading input from the user or file, to remove accidental spaces at the beginning and end before comparison.
    Without trimming, extra spaces could cause equals() to return false even though the main text is same.
  6. Q86 Write the statement to extract the first three characters of the string str.
    Answer: String sub = str.substring(0, 3);
    This takes characters at indices 0, 1 and 2 (end index 3 is excluded).
  7. Q87 How do you check if the string s contains the word "Java"?
    Answer: Use s.contains("Java").
    contains() returns true if the given sequence is found anywhere inside the string.
  8. Q88 Write a single statement that converts a string str to upper‚Äëcase and stores it back in str.
    Answer: str = str.toUpperCase();
    Because toUpperCase() returns a new String, we must assign the result back to the variable to keep the change.
  9. Q89 Explain the role of the String Constant Pool in one or two lines.
    Answer: The String Constant Pool is a special memory area where Java stores string literals so that identical strings can be reused instead of creating multiple copies.
    It improves memory efficiency and works correctly because strings are immutable and cannot be modified later.
  10. Q90 Why is String often used as a key in maps (like HashMap)?
    Answer: Because String is immutable, its hash code never changes, which makes it safe and reliable to use as a key in hash‚Äëbased collections.
    If keys were mutable, changing them after insertion could break the internal hashing logic.

    Strings appear everywhere‚Äîfrom simple console I/O to web applications‚Äîso they form the backbone of everyday coding tasks.
