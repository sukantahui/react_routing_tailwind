Step 1 ¬∑ JavaScript Syntax Essentials

1. Understanding JavaScript Syntax (The Language Rules)

Syntax refers to the set of rules that define how JavaScript code must be written for the engine to understand it. Mastering syntax is the first step to writing clean, predictable programs.

1.1 JavaScript Is Case-Sensitive

// These are all different identifiers
let Name = "A";
let name = "B";
let NAME = "C";

Strict rule: Never mix uppercase/lowercase by accident.

1.2 Statements End with Semicolons (Optional but Recommended)

JavaScript allows missing semicolons, but professionals always use them.

let a = 10;
let b = 5;
console.log(a + b);
Encourage all students to write semicolons consistently ‚Äî this avoids auto-insertion bugs.

1.3 Whitespace Doesn't Matter, But Formatting Does

// All valid JS
let x=10;
let y = 10;
let z     =     10;

But professionals follow spacing conventions (one space around operators, indentation inside blocks).

1.4 Blocks Are Written Inside Curly Braces { }

if (true) {
  console.log("Inside block");
}

Step 3 ¬∑ Variables

3. Variables ‚Äî var, let & const in Depth

A variable stores data. Choosing the right keyword (var, let, const) is fundamental to avoiding bugs related to scope and unexpected changes.

3.1 Declaring with let

Use let when the value will change later.

let score = 10;
score = 20;  // allowed

3.2 Declaring with const

Use const when the value will NOT change.

const pi = 3.1416;
// pi = 10; ‚ùå ERROR

3.3 Declaring with var (Old & Avoided)

var a = 10;
var a = 20; // allowed ‚ùå (dangerous)
Never teach beginners to use var. It causes hoisting & scope confusion.

3.4 Redeclaration vs Reassignment

// let
let x = 5;
// let x = 10; ‚ùå redeclare not allowed
x = 10;      // ‚úî reassignment allowed

// const
const y = 5;
// y = 10; ‚ùå cannot reassign

3.5 Example ‚Äî Real-Life Variable Use

let age = 16;
let passingMarks = 35;
let studentMarks = 60;

if (studentMarks >= passingMarks) {
  console.log("Passed");
} else {
  console.log("Try Again");
}
Step 4 ¬∑ Data Types

4. JavaScript Data Types (Primitive + Reference)

4.1 Primitive Types

  • number ‚Äî numeric values
  • string ‚Äî text
  • boolean ‚Äî true/false
  • null ‚Äî empty on purpose
  • undefined ‚Äî not yet assigned
  • symbol ‚Äî unique identifiers
  • bigint ‚Äî extremely large numbers

4.2 Examples of Primitive Types

let age = 20;               // number
let name = "Sukanta";       // string
let isStudent = true;       // boolean
let phone = null;           // null
let address;                // undefined
let uniqueID = Symbol();    // symbol
let big = 12345678901234567890n;  // bigint

4.3 Check Type Using typeof

console.log(typeof age);       // "number"
console.log(typeof name);      // "string"
console.log(typeof phone);     // "object"  (special behavior)
console.log(typeof big);       // "bigint"
Teach students early: typeof null returns ‚Äúobject‚Äù ‚Äî this is a known historical bug in JS.
Step 5 ¬∑ Expressions

5. Expressions & Operators

An expression is anything that returns a value.

5.1 Arithmetic Operators

10 + 5    // 15  
10 - 5    // 5  
10 * 5    // 50  
10 / 5    // 2  
10 % 3    // 1  
2 ** 3    // 8  (power)

5.2 String Expressions

"Hello" + " World"        // "Hello World"
let a = 10;
"Score: " + a               // "Score: 10"

5.3 Template Literals (Modern Best Practice)

let name = "Sukanta";
console.log(`Hello, ${name}!`);

5.4 Comparison

5 == "5"     // true  (loose equality)
5 === "5"    // false (strict equality)
ALWAYS use === and !== to avoid bugs caused by type coercion.
Step 6 ¬∑ Type Conversion

6. Type Conversion (Explicit & Implicit)

6.1 Explicit Conversion

Number("10")    // 10
String(500)      // "500"
Boolean(0)       // false

6.2 Implicit Conversion

"10" * 2     // 20  
"10" + 2     // "102" (string concatenation)
true + 5     // 6
Teach students that JavaScript tries to ‚Äúguess‚Äù types ‚Äî this is powerful but dangerous.
