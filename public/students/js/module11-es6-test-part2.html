25 high-quality questions covering array & object destructuring, nested unpacking, default values, rest operator, and practical ES6 usage exactly as used in React, APIs, Node, and modern JavaScript.

Section A — Output Questions (Destructuring)

Q1. What is the output? 
const nums = [10, 20, 30];
const [a, b] = nums;
console.log(a, b);
Answer: 10 20
The first two items are unpacked into variables.
Q2. Output? 
const marks = [50];
const [x = 10, y = 20] = marks;
console.log(x, y);
Answer: 50 20
x gets the value 50; y uses its default value 20.
Q3. Output? 
const arr = [1, 2, 3, 4];
const [first, , third] = arr;
console.log(first, third);
Answer: 1 3
The second value is skipped.
Q4. Output? 
const student = { name:"Amit", city:"Kolkata" };
const { city } = student;
console.log(city);
Answer: Kolkata
Q5. Output? 
const user = { name:"Neha" };
const { city = "Unknown" } = user;
console.log(city);
Answer: Unknown Property missing → default applied.

Section B — MCQs (Destructuring Concepts)

Q6. Which syntax renames a destructured property?

A. { name → n }
B. { name: n }
C. { name = n }
D. { rename(name, n) }
Correct: B { name: n } stores name’s value in variable n.
Q7. What does the rest operator ... do in destructuring?

A. Spreads values
B. Collects remaining values into an array/object
C. Deletes unused values
D. Freezes the variable
Correct: B Rest collects remaining values (e.g., const [...others]).
Q8. Destructuring is mainly used in:

A. React props
B. API responses
C. Function parameters
D. All of the above
Correct: D Widely used in modern JS workflows.
Q9. What happens? 
const person = null;
const { a } = person;
❌ Error: Cannot destructure null You must check object existence.
Q10. Which syntax extracts nested properties?

A. { x.y.z }
B. { x:{y:{z}} }
C. { x > y > z }
D. { z(y(x)) }
Correct: B Nested destructuring syntax.

Section C — Practical Coding Questions

Q11. Extract the first two values and the rest: 
const data = [10, 20, 30, 40, 50];
// a = ?, b = ?, others = ?

const [a, b, ...others] = data;
a = 10, b = 20, others = [30, 40, 50]
Q12. Extract city from nested object: 
const centre = {
  name:"Coder & AccoTax",
  address:{ city:"Barrackpore", pin:700120 }
};
const { address:{ city } } = centre;
Q13. Swap two variables using destructuring. 
let a = 5, b = 10;
[a, b] = [b, a];
Q14. Fill code to extract only the course and city: 
const profile = {
  name:"Amit",
  course:"ES6",
  city:"Kolkata"
};

const { course, city } = profile;
Q15. Assign default value: 
const { marks = 80 } = {};  
console.log(marks);
Answer: 80 Default value used since property missing.

Section D — Short Explanations

Q16. Why is destructuring heavily used with API responses?
It allows quick extraction of required properties from large objects, making code shorter, clearer, and easier to maintain.
Q17. Explain array rest operator in one line.
It collects remaining elements of the array into a new array.
Q18. Why is renaming useful in object destructuring?
It avoids naming conflicts and gives more meaningful variable names.
Q19. What problem does default destructuring solve?
It prevents undefined values when the property is missing.
Q20. How do you safely destructure a nested property?
Check object existence first, or use optional chaining before destructuring.
Q21. Give an example of destructuring inside function parameters. 
function show({name, city}) {
  console.log(name, city);
}
Q22. What’s the difference between spread and rest?
Spread expands values; rest collects values.
Q23. Why is destructuring essential in React?
It simplifies extracting props and state variables, making components cleaner.
Q24. Explain nested destructuring in one sentence.
It allows extracting values from inside nested objects or arrays in one step.
Q25. A student asks: “Why use destructuring when I can manually access values?” Give a short mentor-style reply.
Destructuring makes code cleaner, reduces repetition, avoids mistakes in property access, and matches modern coding patterns used in frameworks like React.
