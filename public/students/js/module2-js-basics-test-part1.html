Questions 1–25

1. Which of the following is a valid JavaScript identifier?

A. 2name   B. first-name   C. _totalMarks   D. let

Answer: C
Explanation: Identifiers cannot start with a digit or use - as minus. Keywords like let are reserved. _totalMarks is valid.

2. JavaScript is __________.

A. Case-insensitive   B. Case-sensitive   C. Whitespace-insensitive   D. Two of the above

Answer: D (B and C)
Explanation: JS is case-sensitive, but mostly ignores extra whitespace (except inside strings).

3. Which line is the best practice for ending statements?

A. let a = 10
B. let a = 10;
C. let; a = 10
D. let a = 10,

Answer: B
Explanation: Semicolons are optional but recommended to avoid automatic semicolon insertion bugs.

4. Which is a correct single-line comment?

A. // This is JS
B. # This is JS
C. -- This is JS
D. ** This is JS

Answer: A
Explanation: // starts a single-line comment in JavaScript.

5. Which is a valid multi-line comment in JS?

A. /* comment */
B. <!-- comment -->
C. // comment //
D. # comment

Answer: A
Explanation: /* ... */ is the multi-line comment syntax.

6. What does this print?

let Name = "A";
let name = "B";
console.log(Name, name);
Answer: A B
Explanation: Name and name are different identifiers in a case-sensitive language.

7. Which keyword is recommended for a value that will change?

A. const   B. let   C. var   D. fixed

Answer: B
Explanation: let is block-scoped and designed for reassignable variables.

8. Which keyword is best for a constant like PI?

A. let   B. var   C. const   D. static

Answer: C
Explanation: const prevents reassignment, ideal for fixed values.

9. Which variable declaration style should beginners avoid?

A. let   B. const   C. var   D. all

Answer: C
Explanation: var is function-scoped and hoisted with undefined, causing confusion.

10. What is the output?

let a;
console.log(a);
Answer: undefined
Explanation: Declared but not assigned variables have value undefined.

11. Which of these is NOT a primitive type in JS?

A. number   B. object   C. boolean   D. symbol

Answer: B
Explanation: object is a reference type, not primitive.

12. Which primitive represents intentional “empty”?

A. undefined   B. null   C. 0   D. empty string

Answer: B
Explanation: null means “no value on purpose”.

13. What does typeof null return?

A. "null"   B. "object"   C. "undefined"   D. error

Answer: "object"
Explanation: A historical bug in JS, but relied on in practice.

14. Which line creates a unique Symbol?

A. let s = Symbol("id");
B. let s = new Symbol("id");
C. let s = symbol("id");
D. let s = Symbol;

Answer: A
Explanation: Symbol() is called as a function, not with new.

15. Which type best describes 12345678901234567890n?

A. number   B. bigint   C. string   D. NaN

Answer: B
Explanation: Trailing n makes it a bigint literal.

16. What is an expression?

A. A single number only
B. Anything that returns a value
C. Only arithmetic operations
D. Only function calls

Answer: B
Explanation: Any construct that produces a value is an expression.

17. What is the value of this expression?

10 + 5 * 2

A. 30   B. 25   C. 20   D. 15

Answer: B (25)
Explanation: Multiplication first → 5 * 2 = 10, then 10 + 10 = 20 — wait, check: correct is 10 + (5*2) = 20. So answer is C, 20.

18. What will be printed?

let a = 10;
let b = 5;
console.log(a + b, a - b);
Answer: 15 5
Explanation: 10+5=15, 10-5=5; both values are logged.

19. Which operator is used for exponentiation?

A. ^   B. **   C. pow   D. ^^

Answer: B
Explanation: JS uses ** for power: 2 ** 3 = 8.

20. What is the output?

console.log("Hello" + " " + "World");
Answer: "Hello World"
Explanation: + concatenates strings.

21. Predict the output:

let a = "Score: ";
let b = 40;
console.log(a + b);
Answer: "Score: 40"
Explanation: Number b is converted to string and concatenated.

22. What does this log?

let age = 20;
console.log(typeof age);
Answer: "number"
Explanation: Integers and floats share the number type.

23. What is logged?

let name = "Sukanta";
console.log(`Hello, ${name}!`);
Answer: "Hello, Sukanta!"
Explanation: Template literals inject variable values with ${...}.

24. Which string literal style supports interpolation?

A. " "   B. ' '   C. ` `   D. All of them

Answer: C
Explanation: Only backticks (template literals) support ${...} interpolation directly.

25. Which best describes whitespace in JS?

A. Always significant
B. Mostly ignored except in strings
C. Not allowed
D. Only tabs are allowed

Answer: B
Explanation: Extra spaces/newlines are usually ignored, but not inside quoted text.
