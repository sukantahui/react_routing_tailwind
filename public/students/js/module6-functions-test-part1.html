Q1 · Concept Check
Definition
What is the main purpose of a function in JavaScript?
  • A. To store multiple values in a single variable
  • B. To repeat a block of code for a fixed number of times
  • C. To group reusable logic into a named block that can be called whenever needed
  • D. To style the web page using JavaScript
Answer: C
Explanation:

A function represents a reusable block of code that performs a specific task. You define it once and then call it from multiple places. Options A and B describe arrays and loops, not functions. Styling is done using CSS, not primarily via functions (option D).

Q2 · Syntax
Function Declaration
Which of the following is a correct function declaration?
  • A. function greet { console.log("Hi"); }
  • B. function greet() console.log("Hi");
  • C. function greet() { console.log("Hi"); }
  • D. greet function() { console.log("Hi"); }
Answer: C
Explanation:

The correct syntax is function name() { ... }. Option C follows this. Option A misses parentheses, B misses curly braces, and D has incorrect keyword order.

Q3 · Output Prediction
Call vs Definition
What will be the output of the following code?
function show() {
  console.log("Coder & AccoTax");
}

show();
Answer:

Coder & AccoTax

Explanation:

The function show logs a message. It is then called with show();, so the message is printed once. Defining alone doesn’t output; calling does.

Q4 · Parameters
Concept
In a function definition, what are parameters?
  • A. The values we pass when calling the function
  • B. The variables listed in the function’s parentheses during definition
  • C. The return type of a function
  • D. The number of times a function is executed
Answer: B
Explanation:

Parameters are placeholders declared in the function definition (inside parentheses). The actual values passed during function call are called arguments (option A).

Q5 · Arguments
Basics
Given the function below, which part represents the arguments?
function add(a, b) {
  return a + b;
}

add(3, 7);
Answer:

The values 3 and 7 in the call add(3, 7) are the arguments.

Explanation:

a and b are parameters; the actual values you pass to the function — 3 and 7 — are called arguments.

Q6 · Return Value
Output
What will be logged to the console?
function multiply(a, b) {
  return a * b;
}

let result = multiply(4, 5);
console.log(result);
Answer:

20

Explanation:

multiply(4, 5) returns 4 * 5 = 20, which is stored in result and then printed.

Q7 · MCQ
Return
What happens if a function does not have a return statement?
  • A. It returns 0
  • B. It returns undefined
  • C. It throws an error
  • D. It always returns true
Answer: B
Explanation:

In JavaScript, if a function finishes without encountering a return, the result of the call is undefined.

Q8 · Output Prediction
Return vs Console
What will be the output?
function test() {
  console.log("Inside");
}

let x = test();
console.log(x);
Answer:

First line: Inside
Second line: undefined

Explanation:

test() logs "Inside" but doesn’t return anything, so x becomes undefined. Printing x shows undefined.

Q9 · Concept
Hoisting
Which type of function can be safely called before its definition in the code?
  • A. Function expressions stored in const
  • B. Arrow functions stored in let
  • C. Function declarations
  • D. None of the above
Answer: C
Explanation:

Function declarations are hoisted with their definitions, so you can call them before they appear in the source. Function expressions and arrow functions in const/let are not fully hoisted in the same way.

Q10 · Code Reading
Easy
What does this function do?
function greetStudent(name) {
  return "Welcome, " + name + "!";
}
Answer:

It returns a greeting message like "Welcome, Rahul!" for the given name.

Explanation:

The function concatenates "Welcome, ", the passed name, and an exclamation mark, then returns the resulting string.

Q11 · Function Expression
Medium
Which of the following is a valid function expression?
  • A. const f = function(a, b) { return a - b; };
  • B. function f = (a, b) { return a - b; };
  • C. let function = (a, b) => a - b;
  • D. f(a, b) => { return a - b; }
Answer: A
Explanation:

Option A stores an anonymous function into a constant variable — this is a valid function expression. Options B, C, and D have incorrect syntax or use reserved keywords wrongly.

Q12 · Arrow Function
Syntax
Convert this function into an equivalent arrow function:
function square(n) {
  return n * n;
}
Answer (one possible):
const square = n => n * n;
Explanation:

For a single parameter, parentheses are optional, and when the body is a single expression, you can omit return and braces to use implicit return.

Q13 · Default Parameter
Medium
What will this code print?
function welcome(name = "Student") {
  console.log("Welcome,", name);
}

welcome("Sukanta");
welcome();
Answer:

Welcome, Sukanta
Welcome, Student

Explanation:

First call uses the passed argument. Second call uses the default parameter value "Student" because no argument is provided.

Q14 · Rest Parameter
Concept
Which syntax correctly uses the rest parameter?
  • A. function sum(...a, b) { }
  • B. function sum(a, ...b) { }
  • C. function sum(a, b...) { }
  • D. function ...sum(a, b) { }
Answer: B
Explanation:

The rest parameter ...b must be the last parameter in the list. Option B is correct; A puts rest in front, which is invalid.

Q15 · Short Program
Hands-on
Write a function isEven(n) that returns true if a number is even, else false.
Sample Answer:
function isEven(n) {
  return n % 2 === 0;
}
Explanation:

A number is even if it leaves remainder 0 when divided by 2. Using === 0 ensures strict equality check.

Q16 · Scope
Concept
What is the scope of a variable declared with let inside a function?
  • A. Global scope
  • B. Block scope (inside current block only)
  • C. Function scope (entire function)
  • D. It has no scope
Answer: C (within the function body blocks)
Explanation:

A variable declared with let inside a function is not accessible outside the function. Inside, it still respects block scope, but from outside point of view, it is limited to the function.

Q17 · Output Prediction
Medium
What will be printed?
let message = "Global";

function show() {
  let message = "Local";
  console.log(message);
}

show();
console.log(message);
Answer:

First: Local
Second: Global

Explanation:

The inner message shadows the global one inside the function. Outside, the global variable is printed.

Q18 · MCQ
Return
Which statement about return is true?
  • A. A function can have only one return statement.
  • B. The return statement must always be the last line of a function.
  • C. Once return is executed, the function stops immediately.
  • D. return is optional in JavaScript functions and has no effect.
Answer: C
Explanation:

A function can have multiple return statements in different branches. But whenever one is executed, control leaves the function immediately.

Q19 · Code Reading
Medium
What is logged?
function calc(p, r = 0.1) {
  return p * r;
}

console.log(calc(1000));
console.log(calc(1000, 0.2));
Answer:

100
200

Explanation:

First call uses default rate 0.1 → 1000 × 0.1 = 100. Second call overrides default with 0.2 → 1000 × 0.2 = 200.

Q20 · Short Program
Hands-on
Write an arrow function addGST(amount) that returns amount * 1.18.
Sample Answer:
const addGST = amount => amount * 1.18;
Explanation:

Single parameter → can omit parentheses. Single expression → implicit return. This arrow function returns the amount including 18% GST.

Q21 · Output Prediction
Tricky
What will this code print?
console.log(typeof add);

function add(a, b) {
  return a + b;
}
Answer:

"function"

Explanation:

Function declarations are hoisted along with their definitions. So when typeof add runs, add is already known as a function.

Q22 · MCQ
Naming
Which is a good naming convention for a function in JavaScript?
  • A. DoCalculation()
  • B. do_calculation()
  • C. doCalculation()
  • D. DOCALCULATION()
Answer: C
Explanation:

JavaScript convention is camelCase for functions and variables. Option C follows camelCase: doCalculation.

Q23 · Short Program
Practice
Write a function greetCourse(courseName) that returns:
"Welcome to " + courseName + " course!"
Sample Answer:
function greetCourse(courseName) {
  return "Welcome to " + courseName + " course!";

Explanation:

Simple string concatenation. Could also be done with template literals: `Welcome to ${courseName} course!`.

Q24 · Output Prediction

console.log(x);

Q25 · Concept Mix
Revision
Which of the following statements are true about functions in JavaScript?Multi-select (Think)
  • A. A function can be stored in a variable.
  • B. A function can be passed as an argument to another function.
  • C. A function can be returned from another function.
  • D. A function must always be named.
Answers: A, B, C
Explanation:

In JavaScript, functions are first-class citizens: they can be stored in variables, passed as arguments, and returned from other functions. Anonymous functions (without names) are also allowed, so D is false.
