1. Introduction to Arrays

An array in Java is a fixed-size, indexed, contiguous collection of elements of the same type. Arrays are objects in Java and stored in heap memory.

1.1 Array Characteristics

  • Zero-based indexing
  • Fixed length (cannot be resized)
  • Homogeneous data
  • Stored in contiguous memory blocks
  • Runtime length checking — avoids overflow

1.2 Declaring and Initializing 1D Arrays

int[] arr1 = new int[5];           // default values = 0
int[] arr2 = {10, 20, 30, 40};      // direct initialization
String[] names = new String[]{"A","B","C"};

Memory Model

The array variable stores a reference, while actual values are stored in heap memory.

2. Multidimensional Arrays

3. Arrays Utility Class (java.util.Arrays)

The Arrays class provides ready-made operations for arrays.

3.1 Printing Arrays

int[] a = {10, 20, 30};
System.out.println(Arrays.toString(a));

3.2 Sorting Arrays

Arrays.sort(a);  // QuickSort/Timsort internally

3.3 Searching (Binary Search)

int index = Arrays.binarySearch(a, 20);

3.4 Filling Arrays

Arrays.fill(a, 100);

3.5 Copying Arrays

int[] b = Arrays.copyOf(a, 5);

3.6 Deep Printing (Multidimensional)

System.out.println(Arrays.deepToString(matrix));

4. String Handling in Java

Strings are immutable objects stored in the String Constant Pool (SCP) for memory efficiency.

4.1 String Creation

String s1 = "Hello";         // stored in SCP
String s2 = new String("Hi"); // stored in heap

4.2 Why Strings are Immutable?

  • Thread safety
  • Security — cannot modify credentials accidentally
  • HashCode caching improves performance

4.3 Important String Methods

  • length()
  • substring()
  • indexOf()
  • equals() vs ==
  • toUpperCase(), toLowerCase()
  • split(), trim(), replace()

4.4 String Pool Optimization

String a = "Hello";
String b = "Hello";
System.out.println(a == b); // true (same SCP reference)

5. StringBuilder & StringBuffer

5.1 Why Not Use String for Modifications?

Strings are immutable → every modification creates a new object → memory + performance overhead.

5.2 StringBuffer (Thread-Safe)

StringBuffer sb = new StringBuffer("Hello");
sb.append(" World");

Methods are synchronized → safe for multithreading but slower.

5.3 StringBuilder (Fast, Not Thread-Safe)

StringBuilder sb = new StringBuilder("Hello");
sb.append(" Java");

5.4 Which One to Use?

  • Use StringBuilder for high-performance concatenation
  • Use StringBuffer for multi-threaded environments
  • Use String for small and constant text

5.5 Performance Comparison

String      → slowest (creates many objects)
StringBuffer → medium (synchronized)
StringBuilder → fastest (no locking)

byte[] data = s.getBytes();

String s = new String(ch);

Splitting Strings

7. Best Practices (Intermediate → Expert)

  • Always prefer StringBuilder inside loops
  • Use Arrays.sort() instead of writing custom sorting
  • Use Arrays.deepEquals() for multidimensional comparison
  • Check array.length before accessing indexes
  • Use enhanced-for loop for readability
  • For very large data: prefer ArrayList over arrays
