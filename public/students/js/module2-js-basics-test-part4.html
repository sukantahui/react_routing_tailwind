Questions 76 – 100

76. Output?

let x = 10;
let y = 3;
console.log(x % y);

A. 1   B. 3   C. 0   D. NaN

Answer: A
Explanation: The remainder of 10 ÷ 3 is 1.

77. Output?

let x = 2;
let y = 3;
console.log(x ** y * 2);

A. 8   B. 16   C. 12   D. 10

Answer: B
Explanation: 2 ** 3 = 8, then 8 * 2 = 16.

78. Output?

let a = 10;
let b = 5;
console.log(a / b + a * b);

A. 12   B. 52   C. 50   D. 2

Answer: B
Explanation: 10 / 5 = 2, 10 * 5 = 50, total 52.

79. Output?

console.log(5 == "5" && 5 === "5");

A. true   B. false   C. "true"   D. Error

Answer: B
Explanation: First part is true, second is false; true && false → false.

80. Output?

console.log(5 == "5" || 5 === "5");

A. true   B. false   C. "true"   D. Error

Answer: A
Explanation: || is OR; at least one side is true.

81. Output?

let a = 0;
console.log(!a);

A. 0   B. 1   C. true   D. false

Answer: C
Explanation: 0 is falsy; !0 becomes true.

82. Write a short snippet that logs "Passed" if marks ≥ 35, otherwise "Failed".

let marks = 40; // example
if (marks >= 35) {
  console.log("Passed");
} else {
  console.log("Failed");
}
Explanation: Uses a simple if/else decision based on comparison.

83. Write a single expression (no if) that evaluates to "Passed" or "Failed" using the ternary operator.

let marks = 40;
let result = marks >= 35 ? "Passed" : "Failed";
Explanation: Ternary operator: condition ? valueIfTrue : valueIfFalse.

84. Output?

let language = "JS";
console.log(`I am learning ${language}.`);

A. I am learning language.   B. I am learning JS.   C. `I am learning JS.`   D. Error

Answer: B
Explanation: Template literal inserts the variable’s value.

85. Correct the error:

console.log("Today is ${day}");

Assume let day = "Sunday";

console.log(`Today is ${day}`);
Explanation: Template literals must use backticks, not double quotes, for interpolation.

86. Write a piece of code that declares price and quantity and logs the total bill.

let price = 100;
let quantity = 3;
let total = price * quantity;
console.log(total);
Explanation: Basic arithmetic expression using *.

87. Output?

let text = "Score: " + 50;
console.log(text);

A. 50   B. "Score: 50"   C. NaN   D. Error

Answer: B
Explanation: Concatenation of string and number yields a string.

88. Create a const for DISCOUNT = 10 and compute the final price from base price 500 after discount (simple percentage).

const DISCOUNT = 10;
let base = 500;
let finalPrice = base - (base * DISCOUNT / 100);
console.log(finalPrice);
Explanation: Applies percentage discount: price - price × rate / 100.

89. Output?

let value = "100";
value = Number(value);
console.log(typeof value);

A. "string"   B. "number"   C. "object"   D. "bigint"

Answer: B
Explanation: Number() converts the string to a number.

90. Output?

let isMember = false;
console.log(isMember ? "Discount" : "No Discount");

A. "Discount"   B. "No Discount"   C. true   D. Error

Answer: B
Explanation: The condition is false, so ternary returns the second value.

91. Define in one line: What is a JavaScript variable?

Answer (Sample): A variable is a named container used to store data that can be used and changed in a program.

92. Define in one line: What is a primitive data type?

Answer (Sample): A primitive data type is a basic, indivisible value such as number, string, boolean, null, undefined, symbol, or bigint.

93. Define in one line: What is type coercion?

Answer (Sample): Type coercion is the automatic conversion of one data type to another during expression evaluation.

94. Give an example of implicit type coercion in JavaScript.

"5" * 2   // becomes 10
"5" + 2   // becomes "52"
Explanation: In the first case, strings are converted to numbers; in the second, number is converted to string.

95. Write a line of code that safely compares value and type of 5 and "5".

console.log(5 === "5");
Explanation: === compares both type and value without coercion.

96. Create a simple greeting function using expressions only (no function keyword) and assign it to a const variable.

const greet = (name) => `Hello, ${name}!`;
Explanation: This is a function expression using an arrow function that returns a string.

97. True or False: Every valid JavaScript program is made of statements and expressions.

Answer: True
Explanation: JS code is composed of statements (actions) and expressions (value-producing pieces).

98. Which of the following is a pure expression?

A. let x = 10;
B. x = x + 1;
C. 2 + 3
D. if (x > 5) { x++; }

Answer: C
Explanation: 2 + 3 produces a value without side effects; others modify state or declare variables.

99. Why is it good practice to use meaningful variable names?

Answer (Sample): Meaningful names make code self-explanatory, easier to read, maintain and debug.

100. In one or two lines, summarize what a beginner must master from this module before learning conditions and loops.

Answer (Sample):
They should understand how to declare variables with let and const, know all basic data types, write and read simple expressions, use console.log(), and be comfortable with type conversion and template literals.
Explanation: These are the foundations on which decision-making and looping concepts are built.
