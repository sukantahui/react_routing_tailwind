Step 1 · Native ES Modules

1. ES Modules: import & export

ES Modules allow JavaScript files to import and export functions, variables, and classes. This is the recommended way to structure modern JavaScript applications.

1.1 Named Exports

// math.js
export const add = (a, b) => a + b;
export const sub = (a, b) => a - b;
// main.js
import { add, sub } from "./math.js";
console.log(add(10, 5));

1.2 Default Export

// logger.js
export default function log(msg) {
  console.log("LOG:", msg);
}
// app.js
import log from "./logger.js";
log("Hello");

1.3 Mixing Exports

// config.js
export const API_URL = "https://api.example.com";
export default { version: "1.0", mode: "dark" };

1.4 Importing Everything

import * as utils from "./utils.js";
utils.sayHello();
ES modules only run via HTTP (Live Server / real server). They do NOT load directly via file://.
Step 2 · Folder Structure

2. Folder Structure for Small-Medium Projects

project/
  index.html
  css/
    styles.css
  js/
    main.js
    utils.js
    api/
      auth.js
      posts.js
  assets/
    images/
    icons/
  • Single Responsibility: each file has one purpose.
  • Group by feature: posts, auth, users, UI components.
  • Use modules: keep functions small, reusable, clean.
Naming rule: keep file names meaningful — dom.js, storage.js, api.js.
Step 3 · Node + NPM

3. What Is Node.js & NPM? (Conceptual Overview)

Node.js lets us run JavaScript outside the browser. NPM is the world’s largest library registry for JavaScript packages.

3.1 Node.js — Why It Matters

  • Allows JavaScript to run in terminal/server.
  • Used for tooling, bundlers, backend development.
  • Enables frameworks like React, Express, Vue.

3.2 NPM — Node Package Manager

  • Install libraries like axios, lodash, dayjs.
  • Manages dependencies for your project.
  • Runs scripts (dev, build, test).

3.3 Initializing NPM

npm init -y
This creates package.json → a file that stores project dependencies and scripts.
Step 4 · Installing Packages

4. Installing & Using Simple NPM Packages

4.1 Install a Package

npm install axios

4.2 Use It in an ES Module

import axios from "axios";

async function loadPosts() {
  const res = await axios.get("https://jsonplaceholder.typicode.com/posts");
  console.log(res.data);
}

4.3 Installing Dev Dependencies

npm install -D prettier eslint
Use devDependencies for tools that don't run in final production code.
Step 5 · Bundlers

5. Intro to Bundlers: Vite & Webpack

Bundlers take your modular JavaScript and assets, optimize them, and prepare them for deployment.

5.1 Why Bundlers?

  • Combine many JS files into a single optimized bundle.
  • Minify code and remove unused parts (tree shaking).
  • Handle CSS, images, fonts, JSON, and more.
  • Provide fast dev environment with hot-reload.

5.2 Vite (Recommended for Beginners)

npm create vite@latest myapp

Ultra-fast dev server, simple config, ES module based.

5.3 Webpack (Traditional, Powerful)

npm install webpack webpack-cli

Highly customizable, used in enterprise environments.

In your course, introduce Vite first → then show Webpack conceptually.
