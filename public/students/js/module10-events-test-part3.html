Module 10 — Events Test Part 3 · Mouse Events & Event Object

25 premium-level questions focusing on click events, mousemove, mouseenter, mouseleave, buttons, coordinates, preventDefault and event bubbling, capturing, delegation.

Section A — Output & Behavior Prediction

Q1. What will happen when the button is clicked? 
<button id="btn">Click</button>

document.getElementById("btn").addEventListener("click", function(){
  console.log("Clicked!");
});
Answer: Logs "Clicked!" Explanation: A simple click event listener on the button triggers the callback.
Q2. Predict the output: 
<div id="box">Hello</div>

document.getElementById("box").onclick = () => console.log("Div clicked");
Answer: "Div clicked"
Q3. What logs on double-click? 
document.addEventListener("dblclick", () => console.log("Double!"));
Answer: Logs "Double!" when user double-clicks anywhere.
Q4. What will console.log print? 
<div id="a">
  <button id="b">Click</button>
</div>

document.getElementById("a").addEventListener("click", () => {
  console.log("A");
});
document.getElementById("b").addEventListener("click", () => {
  console.log("B");
});
Answer: B
A
Explanation: Click on button → button handler fires → event bubbles to parent div.
Q5. Output? 
document.addEventListener("contextmenu", e => {
  e.preventDefault();
  console.log("Right click blocked");
});
Answer: Right-click menu is blocked + logs message.

Section B — MCQs on Mouse Events

Q6. Which event fires when the mouse pointer enters an element?

A. mouseover
B. mouseenter
C. mousein
D. hover

Correct: B Explanation: mouseenter does not bubble; mouseover does.
Q7. Which mouse event gives continuous values while moving?

A. mousemove
B. mouseover
C. mouseenter
D. mousewheel

Correct: A
Q8. Which property returns mouse position relative to the viewport?

A. event.clientX
B. event.pageX
C. event.screenX
D. event.offsetX

Correct: A
Q9. event.button value for left-click?

A. 0 B. 1 C. 2 D. None

Correct: A
Q10. Which event fires when mouse leaves element?

A. mouseleave B. mouseexit C. mouseout D. both A and C

Correct: D

Section C — Event Bubbling, Delegation, Capturing

Q11. What is event bubbling?

Event goes from target → parent → document.
Q12. Which option correctly attaches listener during capturing? 
element.addEventListener("click", handler, ???)
A. true B. false C. "capture" D. "bubble"

Correct: A (or {capture:true})
Q13. Event delegation benefits?

A. Fewer listeners B. More performance C. Handles future elements D. All of these

Correct: D
Q14. event.stopPropagation() does what?

Stops event from bubbling further up.
Q15. event.target refers to:

The actual element clicked.

Section D — Mini Logical Questions

Q16. What prints? 
<div id="d">
  <button>X</button>
</div>

document.getElementById("d").addEventListener("click", (e) => {
  console.log(e.target.tagName);
});
Answer: "BUTTON"
Q17. Button is inside a link. What happens? 
<a href="https://google.com">
  <button id="b">Click</button>
</a>

document.getElementById("b").addEventListener("click", e => {
  e.preventDefault();
});
Clicking button will NOT open link.
Q18. What does event.clientY mean?

Vertical mouse position relative to viewport.
Q19. Which is true?

A. mouseenter bubbles B. mouseover doesn't bubble C. mouseenter doesn't bubble D. mouseout never fires

Correct: C
Q20. event.currentTarget refers to:

The element whose listener is currently running.

Section E — Conceptual Short Answers

Q21. Explain difference between event.target and event.currentTarget.

event.target: where the event originated. event.currentTarget: the element whose listener is executing.
Q22. What is preventDefault used for?

Stops browser’s default action like opening links or submitting forms.
Q23. Why is event delegation useful in dynamic lists?

Parent handles clicks even for newly added child elements.
Q24. Explain mouseover vs mouseenter.

mouseover: bubbles + fires repeatedly. mouseenter: does NOT bubble + fires once on entry.
Q25. Why use passive event listeners for scroll/touch?

They improve performance by signalling the browser that event won't be blocked.
