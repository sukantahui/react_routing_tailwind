Step 1 ¬∑ What is an Event?

1. Events ‚Äî The Bridge Between User & JavaScript

An event is anything that happens in the browser: a mouse click, a key press, a form submission, page load, scrolling, etc. Event handling is how your JavaScript reacts to those actions.

1.1 Common Event Types

  • click ‚Äî when the user clicks a button, link, etc.
  • input ‚Äî when the content of an input field changes.
  • submit ‚Äî when a form is submitted.
  • keyup / keydown ‚Äî keyboard interactions.
  • Mouse events: mouseenter, mouseleave, mousemove.
Almost every interactive feature on a webpage starts with: ‚ÄúWhen the user does X (event), run Y (JavaScript function).‚Äù
Step 2 ¬∑ addEventListener

2. Listening to Events with addEventListener

addEventListener is the modern standard way to attach handlers to events.

2.1 Basic Pattern

element.addEventListener("event-name", handlerFunction);

2.2 Example: Click Event

<button id="click-me">Click Me</button>

<script>
  const btn = document.getElementById("click-me");

  function handleClick() {
    console.log("Button was clicked!");
  }

  btn.addEventListener("click", handleClick);
</script>

2.3 Removing a Listener

To remove a listener, the handler must be a named function (not an anonymous arrow directly inside).

btn.removeEventListener("click", handleClick);
Avoid inline HTML events like onclick="..." in modern code. Prefer addEventListener for cleaner separation of HTML & JS.
Step 3 ¬∑ Different Event Types

3. Input, Keyboard & Mouse Events

3.1 Live Input Feedback

<input id="name-input" placeholder="Type your name" />
<p id="preview">Preview will appear here</p>

<script>
  const nameInput = document.getElementById("name-input");
  const preview   = document.getElementById("preview");

  nameInput.addEventListener("input", () => {
    preview.textContent = "Hello, " + nameInput.value;
  });
</script>

3.2 Keyup (e.g., search as you type)

const searchBox = document.getElementById("search");

searchBox.addEventListener("keyup", (event) => {
  console.log("Key pressed:", event.key);
});

3.3 Mouse Events

const box = document.getElementById("hover-box");

box.addEventListener("mouseenter", () => {
  box.textContent = "Mouse inside";
});

box.addEventListener("mouseleave", () => {
  box.textContent = "Mouse left";
});
These examples help students ‚Äúsee‚Äù events: show logs in the console and change simple text on screen.
Step 4 ¬∑ Event Object & Propagation

4. The Event Object & Propagation (Bubbling & Capturing)

When an event happens, the browser passes an event object to your handler. It contains details like: which key, which mouse button, which element, etc.

4.1 Using the Event Object

const btn = document.getElementById("log-event");

btn.addEventListener("click", (event) => {
  console.log("Event type:", event.type);
  console.log("Target element:", event.target);
});

4.2 Event Bubbling Basics

<div id="outer">
  <button id="inner">Click me</button>
</div>

<script>
  const outer = document.getElementById("outer");
  const inner = document.getElementById("inner");

  outer.addEventListener("click", () => {
    console.log("Outer DIV clicked (bubbling)");
  });

  inner.addEventListener("click", () => {
    console.log("Inner BUTTON clicked");
  });
</script>

When you click the button, the click event first fires on the button, then ‚Äúbubbles‚Äù up to its parent (the div), and then further up to the document.

4.3 Stopping Propagation (Intro)

inner.addEventListener("click", (event) => {
  console.log("Inner BUTTON clicked only");
  event.stopPropagation(); // prevent event from reaching outer
});
For beginners, focus on understanding that events move from the target element outward. Explain capturing only at a very conceptual level.
Step 5 ¬∑ Form Handling

5. Form Handling & Simple Validation

Forms are the main way users send data: login, signup, contact forms, etc. JavaScript can validate data before sending it to the server.

5.1 Example Form

<form id="signup-form">
  <input id="email" type="email" placeholder="Email" />
  <input id="password" type="password" placeholder="Password" />
  <button type="submit">Sign Up</button>
  <p id="error-msg" style="color:#f87171;"></p>
</form>

5.2 Handling submit Event

const form     = document.getElementById("signup-form");
const email    = document.getElementById("email");
const password = document.getElementById("password");
const errorMsg = document.getElementById("error-msg");

form.addEventListener("submit", (event) => {
  event.preventDefault(); // stop default page refresh

  if (!email.value.includes("@")) {
    errorMsg.textContent = "Please enter a valid email address.";
    return;
  }

  if (password.value.length < 6) {
    errorMsg.textContent = "Password must be at least 6 characters.";
    return;
  }

  errorMsg.textContent = "";
  console.log("Form is valid. Ready to send to server.");
});
Always call event.preventDefault() if you want to handle form submission manually in JavaScript.
Step 6 ¬∑ Component 1: Modal

6. Building an Interactive Modal

A modal is a dialog box that appears on top of the page. It‚Äôs commonly used for alerts, forms, and confirmations.

6.1 HTML Structure

<button id="open-modal">Open Modal</button>

<div id="overlay" class="hidden">
  <div id="modal">
    <h3>Welcome</h3>
    <p>This is a simple modal window.</p>
    <button id="close-modal">Close</button>
  </div>
</div>

In CSS (concept):

  • #overlay ‚Üí fixed position, full screen, dark background.
  • .hidden ‚Üí display:none;

6.2 JavaScript Logic

const openModalBtn  = document.getElementById("open-modal");
const closeModalBtn = document.getElementById("close-modal");
const overlay       = document.getElementById("overlay");

openModalBtn.addEventListener("click", () => {
  overlay.classList.remove("hidden");
});

closeModalBtn.addEventListener("click", () => {
  overlay.classList.add("hidden");
});

// Optional: close when clicking outside the modal
overlay.addEventListener("click", (event) => {
  if (event.target === overlay) {
    overlay.classList.add("hidden");
  }
});
This introduces students to a common pattern: ‚ÄúAdd a class to show something, remove a class to hide it.‚Äù
Step 7 ¬∑ Component 2: Accordion

7. Building an Accordion (Expandable Sections)

7.1 HTML Structure

<div class="accordion-item">
  <button class="accordion-header">What is JavaScript?</button>
  <div class="accordion-body hidden">
    JavaScript is a programming language for the web...
  </div>
</div>

<div class="accordion-item">
  <button class="accordion-header">What is DOM?</button>
  <div class="accordion-body hidden">
    DOM stands for Document Object Model...
  </div>
</div>

7.2 JavaScript Logic

const headers = document.querySelectorAll(".accordion-header");

headers.forEach(header => {
  header.addEventListener("click", () => {
    const body = header.nextElementSibling;
    body.classList.toggle("hidden");
  });
});
Extend this to:
  • Close other sections when one opens.
  • Change the icon (‚ûï / ‚ûñ) based on open/close state.
Step 8 ¬∑ Component 3: Tabs

8. Building Tabs for Switching Content

8.1 HTML Structure

<div class="tabs">
  <button class="tab-btn active" data-tab="overview">Overview</button>
  <button class="tab-btn" data-tab="syllabus">Syllabus</button>
  <button class="tab-btn" data-tab="faq">FAQ</button>
</div>

<div class="tab-content" id="overview">
  JavaScript course overview content...
</div>
<div class="tab-content hidden" id="syllabus">
  Detailed syllabus content...
</div>
<div class="tab-content hidden" id="faq">
  Frequently asked questions...
</div>

8.2 JavaScript Logic

const tabButtons = document.querySelectorAll(".tab-btn");
const tabPanels  = document.querySelectorAll(".tab-content");

tabButtons.forEach(btn => {
  btn.addEventListener("click", () => {
    const targetId = btn.dataset.tab;

    // 1. Update active button
    tabButtons.forEach(b => b.classList.remove("active"));
    btn.classList.add("active");

    // 2. Show correct panel
    tabPanels.forEach(panel => {
      if (panel.id === targetId) {
        panel.classList.remove("hidden");
      } else {
        panel.classList.add("hidden");
      }
    });
  });
});
Tabs teach:
  • Using data-* attributes for configuration.
  • Working with groups of elements.
  • Adding/removing classes for UI state.

Events Quick Reference

  • Attach: element.addEventListener("click", handler)
  • Remove: element.removeEventListener("click", handler)
  • Mouse: click, dblclick, mouseenter, mouseleave
  • Keyboard: keydown, keyup
  • Form: submit, input, change
  • Event object: event.type, event.target, event.key
  • Control flow: event.preventDefault(), event.stopPropagation()
