1. Introduction to Threading

A thread is a lightweight, independent path of execution within a program. Java supports multithreading using:

  • ‚úî Thread class
  • ‚úî Runnable interface
  • ‚úî Executor framework
  • ‚úî Concurrency utilities (Locks, Semaphores, Futures)
üéØ Personal Trainer Tip: A thread is like a separate worker inside your program, doing tasks in parallel.

2. Creating Threads ‚Äî The Two Classic Ways

2.1 Extending Thread Class

class MyThread extends Thread {
  public void run() {
    System.out.println("Thread running...");
  }
}

public class Demo {
  public static void main(String[] args) {
    MyThread t = new MyThread();
    t.start();
  }
}
‚úî start() starts a new thread ‚úî run() contains the task logic

2.2 Implementing Runnable Interface

class MyTask implements Runnable {
  public void run() {
    System.out.println("Task running...");
  }
}

public class Demo {
  public static void main(String[] args) {
    Thread t = new Thread(new MyTask());
    t.start();
  }
}
üí° Use Runnable when you want to extend another class.

3. Thread Lifecycle

A Java thread passes through the following states:

  • NEW
  • RUNNABLE
  • RUNNING
  • BLOCKED / WAITING
  • TERMINATED
Thread t = new Thread();
t.start();  // NEW ‚Üí RUNNABLE ‚Üí RUNNING

4. Thread Methods (Most Important)

  • start() ‚Üí starts thread
  • run() ‚Üí actual task
  • sleep(ms) ‚Üí pause thread
  • join() ‚Üí wait for thread to finish
  • isAlive() ‚Üí thread status

Example ‚Äî Using sleep() and join()

class Worker extends Thread {
  public void run() {
    try {
      for(int i=1; i<=5; i++){
        System.out.println(i);
        Thread.sleep(500);
      }
    } catch(Exception e){}
  }
}

public class Demo {
  public static void main(String[] args) throws Exception {
    Worker w = new Worker();
    w.start();
    w.join();  // main waits until w finishes
    System.out.println("Done!");
  }
}

5. Synchronization ‚Äî Avoiding Data Damage

When multiple threads access shared data, race conditions may occur.

5.1 Using synchronized Method

class Counter {
  int count = 0;

  synchronized void increment() {
    count++;
  }
}

5.2 Using synchronized Block

void increment() {
  synchronized(this) {
    count++;
  }
}
‚ö†Ô∏è Without synchronization, shared data gets corrupted!

6. Deadlock ‚Äî The Classic Trap

A deadlock occurs when two threads wait forever for each other. 

Thread-1 locks A, waits for B  
Thread-2 locks B, waits for A  
‚Üí DEADLOCK

7. Executors ‚Äî Professional Thread Pooling

import java.util.concurrent.*;

class Demo {
  public static void main(String[] a) {
    ExecutorService pool = Executors.newFixedThreadPool(3);

    pool.submit(() -> System.out.println("Task executed"));
    pool.shutdown();
  }
}
üöÄ Thread pools are the modern, recommended way to run threads in Java.

8. Callable & Future ‚Äî Getting Results from Threads

Callable task = () -> 10 * 10;

Future result = pool.submit(task);

System.out.println(result.get());  // prints 100
