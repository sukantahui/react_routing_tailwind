100 Practice Questions — Object-Oriented Programming in Java

Use these questions for self-study, classroom discussion, assignments, and exam or interview preparation. They are conceptual, scenario-based, and code-oriented, focusing on access modifiers, encapsulation, abstraction, classes, objects, initialization, and constructor chaining.

Access Modifiers
10 Questions
  1. Explain the four access modifiers in Java and their visibility in terms of class, package, subclass, and outside world.
  2. What is the default (package-private) access level in Java and when would you intentionally use it?
  3. Can a top-level class be declared as private or protected? Justify your answer.
  4. What is the difference between protected and default access modifiers in terms of inheritance across packages?
  5. Why is it considered good practice to make fields private and provide public getters and setters?
  6. Can a constructor be declared private? Give a practical use case where this is beneficial.
  7. How would you design a class so that it can only be instantiated from within the same package?
  8. Before Java 9, could interface methods be declared protected or private? Explain with reasons.
  9. In what scenarios would you mark a method as public but keep the class itself package-private?
  10. What happens if you declare a method in a subclass with a more restrictive access level than in its superclass?
Encapsulation
10 Questions
  1. Define encapsulation in the context of object-oriented programming and explain its benefits.
  2. How does Java support encapsulation using classes, fields, and methods?
  3. Rewrite a class that exposes all fields as public into a properly encapsulated version using getters and setters.
  4. Explain how encapsulation helps in maintaining and evolving large codebases over time.
  5. How can encapsulation help in enforcing business rules or input validation? Give a concrete example.
  6. What is the relationship between encapsulation and data hiding? Are they exactly the same? Discuss.
  7. How can misuse or overuse of setters break encapsulation? Provide an example and then correct it.
  8. Discuss how immutable classes (such as String) demonstrate encapsulation principles.
  9. Design an encapsulated class Student with private fields name, roll, and marks along with appropriate methods.
  10. Explain how encapsulation interacts with IDE features like automatic refactoring and code generation.
Abstraction
10 Questions
  1. Define abstraction in Java and distinguish it clearly from encapsulation with examples.
  2. How do abstract classes provide abstraction in Java? Give a code example.
  3. How do interfaces provide abstraction in Java? Contrast with abstract classes.
  4. Write an example of an abstract class with at least one abstract method and one concrete method.
  5. Explain why we cannot instantiate an abstract class directly. What is the typical usage pattern instead?
  6. What happens if a subclass of an abstract class does not implement all of its abstract methods?
  7. Compare abstraction using abstract classes versus interfaces in terms of design flexibility and reuse.
  8. How did the role of interfaces change after the introduction of default and static methods in Java 8?
  9. Design an abstract class Shape with an abstract method area() and implement Circle and Rectangle classes.
  10. Explain how abstraction enables plug-and-play or replaceable modules in large software systems.
Classes, Objects & Methods
10 Questions
  1. Explain the difference between a class and an object in Java using a real-life analogy.
  2. What is an instance method and how does it differ from a static method?
  3. Can we access instance variables from a static method? Justify your answer with an example.
  4. How is memory allocated for objects created with the new keyword in Java?
  5. Write a class representing a Book with fields, a constructor, and a displayDetails() method.
  6. Explain method overloading with a suitable Java example showing different parameter lists.
  7. Why is the main method in Java declared as public static void main(String[] args)?
  8. What is a utility class? How would you design one using only static methods and a private constructor?
  9. Explain the role of the this keyword when referring to fields inside methods and constructors.
  10. How can you prevent a class from being instantiated while still allowing access to its methods?
Initialization Types
10 Questions
  1. List and briefly describe the different ways of initializing fields in a Java class.
  2. What is an instance initializer block and when exactly is it executed in the object lifecycle?
  3. What is a static initializer block and when is it executed relative to object creation?
  4. Compare initialization via constructors and via instance initializer blocks in terms of readability and flexibility.
  5. Show how to initialize a static final constant that requires complex logic or error handling.
  6. Write a class that demonstrates field initialization at declaration, in an instance block, and in a constructor.
  7. Explain the order of initialization for static fields, instance fields, instance initializer blocks, and constructors.
  8. What is meant by “textual order” when executing static initializers and static blocks?
  9. Under what circumstances would you prefer an instance initializer block over overloaded constructors?
  10. Explain how default values are assigned to instance and static fields before explicit initialization in Java.
Constructor Basics
10 Questions
  1. Define a constructor in Java and explain how it differs from a regular method.
  2. What are the key rules for creating constructors in Java (name, return type, modifiers)?
  3. Can a class have multiple constructors? Explain with an example.
  4. What is a no-argument constructor and when is it automatically generated by the compiler?
  5. Explain the concept of constructor overloading with a code example.
  6. Can a constructor be abstract, static, final, or synchronized? Justify your answer for each modifier.
  7. What happens if you explicitly define a parameterized constructor but no default (no-arg) constructor?
  8. How can you prevent external code from creating objects of a class using constructors alone?
  9. Describe a scenario where using overloaded constructors leads to code duplication and potential bugs.
  10. Why is it considered good practice to centralize initialization logic in one primary constructor?
Constructor Chaining with this()
10 Questions
  1. Explain the idea of constructor chaining within the same class using the this() call.
  2. What is the main benefit of using this() inside a constructor instead of duplicating code?
  3. Why must a call to this() be the first statement in a constructor?
  4. Can there be more than one this() call inside a single constructor? Why or why not?
  5. Write a class that has three constructors where each less-detailed constructor calls a more detailed one using this().
  6. What happens if two constructors call each other indirectly using this()? Analyze the effect on compilation or runtime.
  7. How does constructor chaining improve maintainability and reduce errors in initialization logic?
  8. Refactor a hypothetical class with duplicated code in multiple constructors into a version that uses constructor chaining.
  9. Can this() and super() be called within the same constructor body? Explain the reason behind the rule.
  10. Describe how you would design constructors for a Rectangle class (length, breadth, default values) using constructor chaining.
Constructor Chaining with super()
10 Questions
  1. Explain how constructor chaining works between a subclass and its superclass using super().
  2. What happens if you do not explicitly call super() in a subclass constructor?
  3. Why must a call to super() (if present) be the first statement in a constructor?
  4. Can a subclass constructor call a specific overloaded constructor of the superclass? Provide a code example.
  5. Write a class hierarchy where the subclass passes parameters to a specific superclass constructor using super().
  6. Explain the order of execution when both super() and this() are involved in a multi-level inheritance chain.
  7. How does constructor chaining ensure proper initialization of inherited fields and superclass state?
  8. What is the difference in behavior between calling super() explicitly and relying on the compiler to insert it implicitly?
  9. Can we call super() from a static context? Explain why or why not.
  10. Discuss the impact of constructor chaining on debugging and stack traces when exceptions occur during object creation.
Mixed Conceptual & Code-Based Questions
10 Questions
  1. Given a class with private fields and public getter/setter methods, identify how encapsulation is implemented and suggest any improvements.
  2. Write a small class hierarchy using an abstract base class and two concrete subclasses to model employees (e.g., FullTimeEmployee, PartTimeEmployee).
  3. Explain with code how you would use protected access to allow subclasses in other packages to access certain fields.
  4. Write a code snippet to show a static initialization block initializing a log file path and a static logger instance.
  5. Consider a class with multiple constructors where some perform validation and some do not. Refactor it to ensure validation happens consistently.
  6. Given a code fragment where fields are initialized in multiple places (declaration, instance block, constructor), explain the final values when an object is created.
  7. Write Java code that demonstrates the difference between calling overridden methods from constructors and from normal methods.
  8. Given an abstract class with one abstract method and two concrete methods, write a subclass that overrides the abstract method and optionally overrides one concrete method.
  9. Analyze a code snippet where a subclass constructor fails to call the correct superclass constructor, resulting in a compilation error. Explain the fix.
  10. Design a Java class for a configuration settings object that uses private fields, public getters, no setters, and constructor-based initialization for immutability.
Advanced Scenario-Based OOP Questions
10 Questions
  1. You are designing an API for a payment module. How would you use abstraction and encapsulation together to hide the payment gateway details from client code?
  2. A junior developer has made all fields in a class public “for convenience”. List the concrete risks this creates in a real project and how you would fix it.
  3. Design a class hierarchy for a simple learning management system with abstract base classes and interfaces representing courses, instructors, and students.
  4. Given an existing class that is heavily used, how can you introduce better encapsulation without breaking existing client code?
  5. In a multi-module application, how can you use package-private classes and public interfaces to create a clean layered architecture?
  6. Explain how constructor chaining can help when you add a new field to a class that already has several overloaded constructors.
  7. Consider a logging framework class where only one instance must ever exist. How would access modifiers and constructors be used to implement this (Singleton pattern)?
  8. When designing a base abstract class for database entities, what members would you likely make protected vs private vs public, and why?
  9. How can misusing inheritance and constructor chaining make unit testing harder? Propose a design improvement.
  10. Describe a realistic scenario in which changing an access modifier from public to protected or private improves the robustness and maintainability of the system.
