Question 1
Which function signature is best suited for calculating simple interest in a reusable way?
Easy
  • A. function simpleInterest()
  • B. function simpleInterest(p, r, t)
  • C. function simpleInterest(amount)
  • D. function simpleInterest(p)
Answer & Explanation

Correct Option: B

Simple interest depends on principal (p), rate (r), and time (t). A reusable function should accept all three as parameters: simpleInterest(p, r, t).

Question 2
What will the following code print in the console?
Easy 
function greet(name) {
  console.log("Hello " + name);
}

let result = greet("Sukanta");
console.log(result);
  • A. Hello Sukanta then Hello Sukanta
  • B. Hello Sukanta then undefined
  • C. undefined then Hello Sukanta
  • D. Only undefined
Answer & Explanation

Correct Option: B

greet("Sukanta") prints Hello Sukanta but does not return anything, so result becomes undefined. The second console.log(result) prints undefined.

Question 3
Which of the following is a valid arrow function that returns the square of a number?
Easy
  • A. const square = n => { n * n };
  • B. const square = n => return n * n;
  • C. const square = (n) => n * n;
  • D. square => (n) { return n * n; }
Answer & Explanation

Correct Option: C

For a single-expression arrow function with implicit return, you write (n) => n * n. Option A misses return inside { }, Option B is invalid syntax, Option D is not a valid arrow function structure.

Question 8

  • B. No value

Question 14
Which of the following will throw an error?
Hard 
// A
hello();
function hello() { console.log("Hi"); }

// B
greet();
const greet = function(){ console.log("Hello"); };
  • A. Only A throws error
  • B. Only B throws error
  • C. Both A and B throw error
  • D. Neither A nor B throw error
Answer & Explanation

Correct Option: B

Function declarations (A) are hoisted, so calling hello() before its definition works. const greet is in the temporal dead zone before its definition, so calling greet() before that causes a runtime error.

Question 18
Choose the best function to check if a string is empty or only spaces.
Hard
  • A. function isEmpty(str){ return !str; }
  • B. function isEmpty(str){ return str === ""; }
  • C. function isEmpty(str){ return str.trim() === ""; }
  • D. function isEmpty(str){ return str == " "; }
Answer & Explanation

Correct Option: C

str.trim() removes spaces from both ends, so "" after trim means the string was empty or only spaces. A & B fail for strings like " ", and D only checks a single space.
