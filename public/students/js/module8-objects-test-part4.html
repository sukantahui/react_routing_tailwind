Section A — Output & Encapsulation Questions

Q1. What is printed? 
class Bank {
  #balance = 1000;
  getBalance() {
    return this.#balance;
  }
}

const acc = new Bank();
console.log(acc.getBalance());
Answer: 1000
Explanation: #balance is a private field. It is accessible only inside class methods.
Q2. Output? 
class Account {
  #pin = 4444;
  reveal() {
    return this.pin;
  }
}

const a = new Account();
console.log(a.reveal());
Answer: undefined
Explanation: this.pin is different from private #pin. Only this.#pin would work.
Q3. Predict the output: 
class User {
  constructor(name) {
    this._name = name;
  }
  get name() {
    return this._name.toUpperCase();
  }
}

const u = new User("amit");
console.log(u.name);
Answer: AMIT
Getter transforms the stored value.
Q4. Output? 
class Data {
  set value(v) {
    this._v = v * 2;
  }
  get value() {
    return this._v;
  }
}

const d = new Data();
d.value = 6;
console.log(d.value);
Answer: 12
Setter doubles the input before storing.
Q5. What is printed? 
class Person {
  constructor(n) {
    this.name = n;
  }
  introduce() {
    console.log("I am " + this.name);
  }
}

const obj = { name: "Neha" };
Person.prototype.introduce.call(obj);
Answer: I am Neha
call() forces this to refer to obj.

Section B — MCQs on Private Fields, call/apply/bind, Factory & Mixins

Q6. Private fields start with:

A. @ B. # C. _ D. $

Correct: B
Private fields always begin with #.
Q7. Which statement is TRUE?

A. Private fields are inherited by subclasses. B. Private fields cannot be accessed outside the class body. C. Private fields can be accessed using this["#field"]. D. Private fields are stored on prototype.

Correct: B
Q8. call() differs from apply() because:

A. call() executes later B. apply() takes arguments as an array C. call() cannot bind this D. Both are identical

Correct: B
Q9. bind() :

A. Immediately calls the function B. Returns a new function with fixed this C. Deletes prototype chain D. Is same as super()

Correct: B
Q10. A factory function:

A. Must use new B. Returns a new object C. Does not return anything D. Replaces classes completely

Correct: B

Section D — Conceptual & Real-World OOP Problems

Q16. Why are private fields useful in real-world applications?

Private fields prevent unauthorized code from modifying internal values. This is crucial for banking apps, billing systems, authentication tokens, etc.
Q17. Give one real example of using bind().

When passing object methods as callbacks (e.g. event handlers), bind() ensures the method retains its correct this.
Q18. Explain why getters/setters help in validation.

Setters allow you to intercept values before storing them, so you can validate, sanitize, or transform user input.
Q19. Why are factory functions sometimes better than classes?

They offer flexibility, simpler syntax, and can close over variables without needing this or new.
Q20. When would you use a mixin?

When you want to add capabilities to multiple unrelated classes, without changing their inheritance structure.
Q21. What does encapsulation help achieve?

It hides internal details and exposes a clean public interface, reducing accidental misuse and making maintenance easier.
Q22. Why is directly modifying object properties usually discouraged?

It breaks encapsulation and may lead to inconsistent or invalid state.
Q23. In an OOP student management system for Coder & AccoTax, what advantage does using classes provide?

It organizes student data + behavior together, supports reusability, inheritance, and easier long-term maintenance.
Q24. What is the difference between composition and inheritance?

Inheritance creates an "is-a" relationship (Student is a Person). Composition creates a "has-a" relationship (Car has an Engine).
Q25. Which OOP feature prevents external modification of sensitive data?

Encapsulation (via private fields, getters/setters).
