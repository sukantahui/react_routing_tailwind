Module 13 — Test Part 1

JSON Basics · JSON.parse · JSON.stringify · Rules of JSON
1. What does JSON stand for?
a) Java Source Object Notation
b) JavaScript Oriented Network
c) JavaScript Object Notation
d) Java Syntax Object Notation
✔ Correct: c
JSON = JavaScript Object Notation, a lightweight format for data exchange.
2. JSON is primarily used for:
a) Styling web pages
b) Data storage & data exchange
c) Server hosting
d) Creating animations
✔ Correct: b
JSON is mainly used to send and receive structured data across systems.
3. Which is valid JSON?
a) {name:'Amit', age:22}
b) {"name":"Amit","age":22}
c) ['Amit',22]
d) (name:"Amit")
✔ Correct: b
JSON requires double quotes around keys and string values.
4. Which function converts JSON text into a JavaScript object?
a) JSON.object()
b) parse.JSON()
c) JSON.parse()
d) JSON.toJS()
✔ Correct: c
5. What does JSON.stringify() do?
a) Converts JavaScript object → JSON string
b) Converts JSON → JS object
c) Saves JSON to the server
d) Encrypts JSON
✔ Correct: a
6. Which of these is NOT allowed in JSON?
a) Strings
b) Numbers
c) Booleans
d) Functions
✔ Correct: d
JSON cannot contain functions or undefined values.
7. What is the output of JSON.stringify({x:10, y:20})?
a) {x:10, y:20}
b) '{"x":10,"y":20}'
c) '[x:10,y:20]'
d) ["x","y"]
✔ Correct: b
8. JSON keys must always be:
a) Single quoted
b) Double quoted
c) Without quotes
d) Template literals
✔ Correct: b
9. What will JSON.parse('true') return?
a) "true"
b) true
c) {true}
d) Error
✔ Correct: b
JSON booleans convert to JS booleans.
10. Which is valid JSON array?
a) {1,2,3}
b) ["A","B","C"]
c) ["A",'B',"C"]
d) (1,2,3)
✔ Correct: b
JSON requires all strings to use double quotes only.
11. What is wrong in: {"name":"Amit", age:20}?
a) name uses double quotes
b) age is a number
c) age key is not in double quotes
d) JSON cannot have numbers
✔ Correct: c
12. JSON is mainly used in:
a) Network communication
b) Browser rendering
c) CSS styling
d) Audio processing
✔ Correct: a
13. JSON supports comments.
a) True
b) False
✔ Correct: b
JSON never allows comments.
14. What will JSON.parse('"Hello"') return?
a) Hello
b) "Hello"
c) ["Hello"]
d) Error
✔ Correct: a
15. JSON can store:
a) Objects
b) Arrays
c) Nested structures
d) All of the above
✔ Correct: d
16. What is the output?
{"a":1,"b":2}.a
a) 1
b) Syntax error
c) undefined
d) "a"
✔ Correct: b
JSON is not JS code; you cannot access properties inside JSON text.
17. JSON data type NOT supported:
a) null
b) array
c) undefined
d) boolean
✔ Correct: c
18. Pretty JSON uses:
a) JSON.pretty()
b) JSON.stringify(obj, null, 2)
c) JSON.format()
d) print.JSON()
✔ Correct: b
19. What is the output of JSON.parse('123')?
a) "123"
b) 123
c) NaN
d) Error
✔ Correct: b
20. JSON keys must be:
a) numbers only
b) always strings
c) can be functions
d) can be booleans
✔ Correct: b
21. What is invalid JSON?
a) {"name":"Amit"}
b) {"age":20}
c) {"active":true}
d) {"name": "Amit",}
✔ Correct: d
Trailing commas are not allowed.
22. JSON cannot contain:
a) nested arrays
b) nested objects
c) undefined
d) null
✔ Correct: c
23. What will JSON.parse('{}') produce?
a) null
b) empty object {}
c) []
d) Error
✔ Correct: b
24. JSON strings must use:
a) double quotes
b) single quotes
c) either
d) backticks
✔ Correct: a
25. Which method is used to safely send data in APIs?
a) JSON.write()
b) JSON.stringify()
c) parse.JSON()
d) jsonConvert()
✔ Correct: b
JSON.stringify sends robust, valid JSON through APIs.
