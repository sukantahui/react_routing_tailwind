Step 1 · What is an Object?

1. Objects — The Heart of JavaScript

An object is a collection of key–value pairs. Keys are called properties, and when a property holds a function, it’s called a method.

1.1 Object Literal Syntax

const student = {
  name: "Amit",
  age: 18,
  course: "JavaScript",
  isActive: true,
  greet: function () {
    console.log("Hello, I am " + this.name);
  }
};

console.log(student.name); // "Amit"
student.greet();           // "Hello, I am Amit"
Mentally read an object as: “student has a name, age, course, isActive flag, and a greet behavior”.
Step 2 · Accessing Properties

2. Dot Notation vs Bracket Notation

2.1 Dot Notation (Most Common)

console.log(student.name);
student.age = 19;
console.log(student.age);

2.2 Bracket Notation (Dynamic or Special Keys)

console.log(student["course"]);

const key = "isActive";
console.log(student[key]); // same as student.isActive

2.3 When Brackets Are Required

const fees = {
  "course-fee": 5000,
  "gst%": 18
};

// fees.course-fee ❌ (invalid)
console.log(fees["course-fee"]); // ✅
Rule of thumb: use dot notation generally; switch to bracket notation for dynamic keys or keys with spaces/special characters.
Step 3 · Nested Structures

3. Nested Objects & Arrays of Objects

3.1 Nested Object

const centre = {
  name: "Coder & AccoTax",
  address: {
    line1: "25(10/A) Shibtala Road",
    city: "Barrackpore",
    state: "West Bengal"
  },
  courses: ["JavaScript", "Python", "Accounts"]
};

console.log(centre.address.city);      // "Barrackpore"
console.log(centre.courses[0]);        // "JavaScript"

3.2 Array of Objects

const students = [
  { name: "Amit",  marks: 85 },
  { name: "Neha",  marks: 92 },
  { name: "Rahul", marks: 76 }
];

for (let s of students) {
  console.log(`${s.name} scored ${s.marks}`);
}
Nested objects + arrays are the basis of real data structures (API responses, database records, etc.).
Step 4 · this Keyword (Basics)

4. this Keyword — Basic Understanding

this usually refers to “the object that is calling the method”. In an object method, this is a very natural way to refer to that object.

4.1 this Inside a Method

const student1 = {
  name: "Neha",
  course: "JavaScript",
  greet() {
    console.log("Hello, I am " + this.name);
  }
};

student1.greet(); // "Hello, I am Neha"

4.2 Arrow Functions & this (Basic Warning)

const student2 = {
  name: "Rahul",
  greet: () => {
    console.log("Hello, I am " + this.name);
  }
};

student2.greet(); // "Hello, I am undefined" (in most cases)
Arrow functions do not have their own this. For object methods, prefer normal function syntax (greet() { ... }).
Step 5 · Constructor Functions

5. Constructor Functions & new Keyword

Constructor functions are regular functions used with new to create multiple similar objects.

5.1 Defining a Constructor

function Student(name, course) {
  this.name = name;
  this.course = course;
  this.greet = function () {
    console.log("Hi, I am " + this.name);
  };
}

const s1 = new Student("Amit", "JavaScript");
const s2 = new Student("Neha", "Python");

s1.greet(); // "Hi, I am Amit"
s2.greet(); // "Hi, I am Neha"

5.2 What new Does (Conceptually)

  • Creates a new empty object: {}
  • Sets its internal prototype to Student.prototype
  • Calls Student with this pointing to that new object
  • Returns the new object (if you don’t return anything manually)
Constructor function names usually start with a capital letter (e.g., Student, Course).
Step 6 · Prototype Chain

6. Prototypes & the Prototype Chain

Every object in JavaScript has an internal link to another object known as its prototype. Methods are usually stored on the prototype so that all instances can share them.

6.1 Moving Methods to the Prototype

function Student(name, course) {
  this.name = name;
  this.course = course;
}

// one shared method for all students
Student.prototype.greet = function () {
  console.log("Hi, I am " + this.name);
};

const s1 = new Student("Amit", "JavaScript");
const s2 = new Student("Neha", "Python");

s1.greet();
s2.greet();

6.2 Inspecting Prototypes

console.log(Object.getPrototypeOf(s1) === Student.prototype); // true
console.log(s1.__proto__ === Student.prototype);              // true (legacy but useful to see)
When a property is not found on the object itself, JavaScript looks up the prototype chain: object → object.__proto__ → ... → Object.prototype → null.
Step 7 · ES6 Classes

7. ES6 Classes — Modern OOP Syntax

ES6 class syntax is a cleaner way to write constructor + prototype patterns. Under the hood, it still uses prototypes.

7.1 Basic Class

class Student {
  constructor(name, course) {
    this.name = name;
    this.course = course;
  }

  greet() {
    console.log(`Hi, I am ${this.name} from ${this.course}`);
  }
}

const s1 = new Student("Amit", "JavaScript");
s1.greet();

7.2 Static Methods

class MathUtils {
  static add(a, b) {
    return a + b;
  }
}

console.log(MathUtils.add(2, 3)); // 5
Static methods belong to the class itself, not to individual instances. You call them as ClassName.method(), not on an object.
Step 8 · Inheritance

8. Inheritance with extends & super

Inheritance allows one class to reuse and extend the behavior of another.

8.1 Base & Derived Class

class Person {
  constructor(name) {
    this.name = name;
  }

  introduce() {
    console.log(`I am ${this.name}.`);
  }
}

class Student extends Person {
  constructor(name, course) {
    super(name); // call Person constructor
    this.course = course;
  }

  introduce() {
    // override method
    console.log(`I am ${this.name}, studying ${this.course}.`);
  }
}

const p = new Person("Sourav");
const s = new Student("Neha", "JavaScript");

p.introduce(); // I am Sourav.
s.introduce(); // I am Neha, studying JavaScript.
super() must be called in the subclass constructor before using this. Overriding lets you change or extend behavior in derived classes.
Step 9 · Encapsulation

9. Getters, Setters & Private Fields

Encapsulation means hiding internal details and exposing a clean interface. ES6+ classes support getters, setters, and private fields for this.

9.1 Getters & Setters

class Account {
  constructor(owner, balance) {
    this.owner = owner;
    this._balance = balance;
  }

  get balance() {
    return this._balance;
  }

  set balance(amount) {
    if (amount < 0) {
      console.log("Balance cannot be negative.");
      return;
    }
    this._balance = amount;
  }
}

const acc = new Account("Amit", 1000);
console.log(acc.balance);  // uses getter
acc.balance = 1500;        // uses setter
console.log(acc.balance);

9.2 Private Fields (Modern JS)

class SecureAccount {
  #balance; // private field

  constructor(owner, balance) {
    this.owner = owner;
    this.#balance = balance;
  }

  deposit(amount) {
    this.#balance += amount;
  }

  getBalance() {
    return this.#balance;
  }
}

const sa = new SecureAccount("Neha", 2000);
sa.deposit(500);
console.log(sa.getBalance()); // 2500
// sa.#balance ❌ Syntax error: cannot access private field
Private fields (#field) are not supported in very old browsers, but are standard in modern JavaScript.
Step 10 · Patterns

10. Factory Functions & Mixins (Patterns)

10.1 Factory Function

A function that returns a new object, without using new.

function createStudent(name, course) {
  return {
    name,
    course,
    greet() {
      console.log(`Hi, I am ${this.name}`);
    }
  };
}

const st1 = createStudent("Amit", "JS");
st1.greet();

10.2 Simple Mixin

Mixin = object that contains methods which can be copied into other objects.

const canLog = {
  log() {
    console.log(`[LOG]`, this);
  }
};

const obj = { name: "Test" };
Object.assign(obj, canLog);

obj.log(); // [LOG] { name: "Test", log: [Function] }
In real projects, you see a mix of classes, factory functions, and composition patterns. Teach this gradually — focus on understanding, not memorising patterns.
Step 11 · this Binding

11. this Binding, call, apply, bind

Sometimes you want to control which object this refers to when calling a function. For that, JavaScript provides call, apply, and bind.

11.1 Example Setup

function introduce(city) {
  console.log(`I am ${this.name} from ${city}.`);
}

const person = { name: "Amit" };

11.2 call — Pass Arguments One by One

introduce.call(person, "Kolkata");

11.3 apply — Pass Arguments as Array

introduce.apply(person, ["Barrackpore"]);

11.4 bind — Create a New Function with Fixed this

const introductionsForAmit = introduce.bind(person);
introductionsForAmit("Delhi");
introductionsForAmit("Mumbai");
call and apply execute immediately with a chosen this. bind returns a new function that remembers the chosen this.
Step 12 · Summary & Practice

12. Summary & Practice Tasks

12.1 Concept Checklist

  • Create and use object literals with properties and methods.
  • Access and update data using dot and bracket notation.
  • Model real data with nested objects and arrays of objects.
  • Explain what this refers to inside an object method.
  • Use constructor functions with new.
  • Understand how prototypes share methods across instances.
  • Use ES6 classes, static methods, and inheritance.
  • Apply getters, setters, and private fields for encapsulation.
  • Understand factory functions and mixin basics.
  • Control this using call, apply, and bind.

12.2 Practice Exercises

  1. Create an object course with properties: title, duration, fee, and a method describe() that prints a nice sentence.
  2. Create an array of 5 student objects (name, marks). Use a loop to print the names of students who scored >= 80.
  3. Write a constructor function Book(title, author, price) with:
    • properties: title, author, price
    • a prototype method getDetails()
  4. Convert the Book constructor into an ES6 class with a method and one static helper (e.g., comparePrice(book1, book2)).
  5. Create a base class Person and a subclass Teacher with additional property subject. Override an introduce() method.
  6. Implement a BankAccount class with private balance and public methods: deposit(), withdraw(), getBalance().
  7. Make a simple factory function createUser(name, role) that returns user objects with a describe() method.
  8. Create a standalone function and use call / apply / bind to make it work with different objects as this.

Quick Reference — Object & OOP

  • Literal: { ... } for simple structs
  • Constructor: function + new for many similar objects
  • Class: modern syntax for constructor + prototype
  • Prototype: shared methods & inheritance chain
  • this: usually “who called the method”
  • Encapsulation: getters, setters, private fields

Teacher Notes for Module 8

  • Start from simple objects (name, marks, course).
  • Use diagrams to show prototype chains and class relationships.
  • Focus on intuition: “objects hold data + behavior together”.
  • Show both constructor/prototype and class syntax; emphasise that classes are syntax sugar.
  • Revisit this module later during project building to reinforce OOP concepts.
