Step 1 ¬∑ Why Functions?

1. What Is a Function?

A function is a reusable block of code designed to perform a specific task. Instead of copying the same logic many times, you define it once and call it whenever needed.

1.1 Real-Life Analogy

  • A ‚Äúprint invoice‚Äù function in billing software
  • A ‚Äúcalculate tax‚Äù function in accounting
  • A ‚Äúsend OTP‚Äù function in a login system

In JavaScript, a simple function looks like this:

function greet() {
  console.log("Hello from Coder & AccoTax!");
}

greet(); // calling/invoking the function
Defining a function does nothing by itself until you call it. Think of the definition as writing a recipe, and the call as cooking the dish.
Step 2 ¬∑ Function Declarations

2. Function Declarations (Named Functions)

2.1 Syntax

function functionName() {
  // function body
}

2.2 Example: Simple Greeting

function sayHello() {
  console.log("Hello, student!");
}

sayHello();
sayHello();

2.3 Functions with Parameters

Parameters act like variables inside the function. You pass arguments when calling the function. 

function greetStudent(name) {
  console.log("Welcome, " + name + "!");
}

greetStudent("Sukanta");
greetStudent("Rahul");

2.4 Multiple Parameters

function add(a, b) {
  console.log("Sum =", a + b);
}

add(10, 5);
add(100, 250);
Train students to read function signatures aloud: add(a, b) ‚Üí ‚Äúadd takes two parameters, a and b‚Äù.
Step 3 ¬∑ Return Values

3. Returning Values from Functions

A function can return a value using the return keyword. The returned value can be stored in a variable or used in expressions.

3.1 Basic Return

function multiply(a, b) {
  return a * b;
}

let result = multiply(4, 5);
console.log("Result =", result);

3.2 Multiple Returns via Objects

JavaScript functions return only one value, but that value can be an object.

function calculateMarks(m1, m2, m3) {
  let total = m1 + m2 + m3;
  let average = total / 3;
  return {
    total: total,
    average: average
  };
}

let report = calculateMarks(80, 90, 85);
console.log("Total =", report.total);
console.log("Average =", report.average);
Important: As soon as JavaScript encounters return, it exits the function immediately ‚Äî any code after return will not run.

3.3 Early Return Pattern

function divide(a, b) {
  if (b === 0) {
    console.log("Cannot divide by zero.");
    return; // stop here
  }
  console.log("Result =", a / b);
}

divide(10, 0);
divide(10, 2);
Step 4 ¬∑ Function Expressions

4. Function Expressions

A function expression is a function stored inside a variable. It does not require a name of its own.

4.1 Syntax

const add = function (a, b) {
  return a + b;
};

console.log(add(3, 4));

4.2 Anonymous Functions

The function above has no name (anonymous); it‚Äôs accessed through the variable add.

4.3 Difference vs Declarations (Concept)

  • Function declarations are hoisted ‚Äî you can call them before their definition.
  • Function expressions are not available before the line where they are defined.
// Declaration
hello(); // works

function hello() {
  console.log("Hi!");
}

// Expression
greet(); // ‚ùå error: cannot access before initialization

const greet = function () {
  console.log("Hello!");
};
For teaching, you can prefer function declarations at first, then gradually introduce expressions and hoisting.
Step 5 ¬∑ Arrow Functions

5. Arrow Functions (Modern JavaScript)

Arrow functions are a shorter syntax introduced in ES6. They are very common in modern JavaScript, especially with arrays and callbacks.

5.1 Basic Syntax

const square = (n) => {
  return n * n;
};

console.log(square(5));

5.2 Short Form (Implicit Return)

const cube = n => n * n * n;

console.log(cube(3));

5.3 Arrow with Multiple Parameters

const add = (a, b) => a + b;
console.log(add(10, 20));

5.4 When Not to Use Arrow Functions (Basic Idea)

  • As methods inside objects when you need this (covered later in OOP module).
  • When using arguments object (arrow functions don‚Äôt have their own arguments).
For beginners, treat arrow functions as a shorter way to write simple functions. Deep this behavior can be explained in advanced modules.
Step 6 ¬∑ Parameters Variations

6. Default Parameters & Rest Parameters

6.1 Default Parameters

function welcome(name = "Student") {
  console.log("Welcome,", name);
}

welcome("Sukanta"); // Welcome, Sukanta
welcome();          // Welcome, Student

6.2 Rest Parameters (Collect Many Arguments)

function sumAll(...numbers) {
  let total = 0;
  for (let n of numbers) {
    total += n;
  }
  return total;
}

console.log(sumAll(1, 2, 3));        // 6
console.log(sumAll(5, 10, 15, 20));  // 50
Use rest parameters when you don‚Äôt know in advance how many arguments will be passed to the function.
Step 7 ¬∑ Scope

7. Variable Scope with Functions

Scope defines where a variable is accessible. Functions create their own scope.

7.1 Local vs Global Scope

let globalMessage = "I am global";

function showMessages() {
  let localMessage = "I am local";
  console.log(globalMessage); // accessible
  console.log(localMessage);  // accessible
}

showMessages();

console.log(globalMessage);   // accessible
// console.log(localMessage); // ‚ùå error: not defined

7.2 Block Scope with let and const

if (true) {
  let a = 10;
  const b = 20;
  console.log(a, b); // works
}

// console.log(a, b); // ‚ùå not accessible outside block
Teach students: let and const are block-scoped (between { }), while functions themselves create function scope.
Step 8 ¬∑ Functions + Arrays

8. Functions with Arrays (Preview of Higher-Order Thinking)

Functions are often passed into other functions ‚Äî this idea is the base of higher-order functions and functional programming.

8.1 Example: Apply a Discount

const prices = [100, 200, 300];

function applyGST(price) {
  return price * 1.18;
}

for (let p of prices) {
  console.log("With GST:", applyGST(p));
}

8.2 Example: Simple Callback with forEach

const students = ["Amit", "Neha", "Sourav"];

students.forEach(function (name) {
  console.log("Hello,", name);
});

You will explore array methods (map, filter, reduce) in the Arrays module. Here, just introduce the idea that functions can be passed around like values.

Step 9 ¬∑ Summary & Practice

9. Summary & Practice Tasks

After this module, students should be able to write and use functions comfortably, reason about parameters, return values, and understand basic scope.

9.1 Concept Checklist

  • Explain what a function is and why it is useful.
  • Write function declarations with parameters and return values.
  • Use function expressions and understand the difference from declarations.
  • Write arrow functions with and without implicit returns.
  • Use default parameters and rest parameters.
  • Understand the difference between global, local, and block scope.

9.2 Practice Exercises

  1. Write a function square(n) that returns the square of a number. Call it for 2, 5, and 10 and print results.
  2. Write a function isEligibleForVote(age) that returns true if age ‚â• 18, else false. Test with different ages.
  3. Write a function calculateSimpleInterest(p, r, t) and return the interest.
  4. Use an arrow function max(a, b) that returns the larger of the two numbers.
  5. Create a function printTable(n) that prints the multiplication table of n from 1 to 10 using a loop.
  6. Create a function sumAll(...nums) using rest parameters and test with different sets of numbers.
  7. Demonstrate scope: create a global variable, a local variable, and a block-scoped variable, and try to access them from different places.
