Step 1 · Understand What JavaScript Is

1. What Exactly Is JavaScript?

JavaScript is a high-level, interpreted programming language that runs primarily in the browser and controls the behavior of web pages. Think of HTML as the skeleton, CSS as the clothes & colors, and JavaScript as the brain & muscles that make everything move and respond.

1.1 The “Big Three” of Frontend

  • HTML — defines what is on the page (headings, paragraphs, forms, buttons).
  • CSS — defines how those things look (colors, spacing, fonts, layout).
  • JavaScript — defines how the page behaves (interactions, animations, logic).

When a browser loads a webpage:

  1. It downloads the HTML file.
  2. It downloads and applies CSS files.
  3. It downloads and executes JavaScript files step by step.
Expert note: JavaScript is single-threaded and uses an event loop to manage tasks. We will explore the event loop in the Asynchronous JavaScript module. For now, remember: the browser executes JavaScript code one line at a time, from top to bottom in each script.

1.2 Where Can JavaScript Run?

  • In the browser — this is our starting point. Every modern browser has a built-in JS engine.
  • On the server using Node.js — this allows JavaScript to build backend APIs.
  • In tools & scripts — bundlers, CLIs, automation tools also use JavaScript.

In this course, we start with browser-based JavaScript and later introduce the idea of Node.js.

Step 2 · See How Browsers Treat HTML, CSS, JS

2. How Does the Browser Use JavaScript?

The browser converts your HTML into a tree structure called the DOM (Document Object Model), and JavaScript can read and modify this tree in real time.

2.1 A Mental Model of a Web Page

Consider this simple HTML page:

<!DOCTYPE html>
<html>
  <head>
    <title>My First Page</title>
  </head>
  <body>
    <h1>Welcome!</h1>
    <p>Learning JavaScript with Coder &amp; AccoTax.</p>
  </body>
</html>

When the browser loads this file, internally it builds a structure like:

  • document (the entire page)
  • inside it: html element
  • inside it: head and body elements
  • inside body: h1 and p elements

JavaScript can access this structure using the global document object and change text, styles, or even add/remove elements dynamically.

Teaching tip: When explaining DOM, always relate it to a “family tree” diagram — parent, child, and sibling relationships between elements.
Step 3 · Set Up Your Environment

3. Tools You Need to Start Coding

You do not need any heavy tools to start JavaScript. A browser and a code editor are enough.

3.1 Browser (JavaScript Engine)

Use any modern browser (Chrome, Edge, Firefox, Brave, etc.). They all have:

  • A powerful JavaScript engine (e.g., V8 in Chrome).
  • Built-in Developer Tools to see logs and debug code.

3.2 Code Editor

We will use VS Code, but any text editor works. VS Code advantages:

  • Syntax highlighting for HTML, CSS, JavaScript.
  • Extensions for auto-format, snippets, live server, etc.
  • Integrated terminal (later useful for Node.js).

3.3 Folder Structure for This Module

Create a folder for your JavaScript course, for example:

javascript-course/
  module-1-getting-started/
    index.html
    script.js          (we will create this in Step 7)
Keep all files for each module in separate folders. This habit prepares students for larger project structures later.
Step 4 · Write Your First JavaScript Program

4. Creating the First HTML + JavaScript File

In this step, we will write a complete HTML file with JavaScript inside it and open it in the browser.

4.1 Create index.html

Inside module-1-getting-started, create a new file named index.html with this content:

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Getting Started with JavaScript</title>
</head>
<body>
  <h1>JavaScript – Getting Started</h1>
  <p>This page is created for students of Coder &amp; AccoTax.</p>

  <script>
    // This is our first JavaScript code
    alert("Welcome to JavaScript!");
    console.log("This message is from JavaScript.");
  </script>
</body>
</html>

4.2 Understand Each Part

  • <script> ... </script> tells the browser: “Now execute JavaScript code.”
  • alert(...) shows a popup message on the page.
  • console.log(...) prints a message to the browser’s console (for developers).
  • // starts a single-line comment in JavaScript.
If you see nothing happening, check:
  • Did you save the file as index.html (not index.html.txt)?
  • Did you open it in a real browser (not in VS Code’s preview only)?

4.3 Run the Page in the Browser

  1. Open the folder in your file manager.
  2. Double-click index.html (or right-click → Open with → your browser).
  3. You should see the webpage with the heading and paragraph.
  4. An alert box “Welcome to JavaScript!” should appear.

After closing the alert, open the Developer Tools console and you will see the console.log("This message is from JavaScript.") output.

Step 5 · Use the Browser Console Like a Pro

5. Exploring JavaScript in the Browser Console

The console is a safe playground where you can experiment with JavaScript without editing files.

5.1 Opening Developer Tools

  • In Chrome/Edge: Press F12 or Ctrl + Shift + I.
  • Click the Console tab.

5.2 Try These Commands Step by Step

Type each line one by one in the console and press Enter:

> 2 + 3
5

> "Hello" + " " + "World"
"Hello World"

> let name = "Sukanta";
> name
"Sukanta"

> console.log("Learning JS with Coder & AccoTax");
Learning JS with Coder & AccoTax

> alert("Console can also call alert boxes!");

Every line starting with > is what you type. The next line is the output from JavaScript.

Expert practice: Ask students to try simple math, string concatenation, and small expressions in the console before writing them inside a <script> tag.
Step 6 · Move JavaScript into a Separate File

6. External JavaScript File (Best Practice)

For professional projects, we almost never keep all JavaScript directly inside HTML. We separate it into .js files.

6.1 Create script.js

Inside the same folder, create a new file called script.js:

// file: script.js

// Show a welcome message
alert("Welcome to JavaScript (from an external file)!");

// Log some information in the console
console.log("This is script.js speaking.");

// A small calculation example
let a = 10;
let b = 5;
let sum = a + b;
console.log("a =", a, ", b =", b, ", sum =", sum);

// A simple greeting function
function greetStudent(name) {
  console.log("Hello, " + name + "! Keep learning JavaScript.");
}

// Call the function
greetStudent("Coder & AccoTax Student");

6.2 Link script.js From index.html

Now remove the old inline script from index.html and instead link the external file at the end of the <body>: 

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Getting Started with JavaScript</title>
</head>
<body>
  <h1>JavaScript – Getting Started</h1>
  <p>This page is created for students of Coder &amp; AccoTax.</p>

  <!-- Link external JavaScript file -->
  <script src="script.js"></script>
</body>
</html>

Reload the page. You should see the new alert message and the console logs from script.js.

Best practice: Place the <script> tag just before the closing </body> tag. This ensures the HTML is loaded before JavaScript starts accessing it.
Step 7 · Debug Common Beginner Errors

7. Typical Errors in “Getting Started” Phase

Teaching students to read error messages early saves a lot of frustration later.

7.1 File Not Found

If you see something like Failed to load resource: net::ERR_FILE_NOT_FOUND:

  • Check that script.js is in the same folder as index.html.
  • Check the spelling: src="script.js" must match the exact filename.

7.2 Syntax Errors

Example mistake:

// WRONG
let a = 10
let b = 5;
console.log(a + b);

Missing semicolons usually don’t break JavaScript by themselves, but other syntax errors do. Consider this: 

// WRONG
let a = 10;
let b = 5
console.log(a + b;;

Here, the extra ; and missing parenthesis will cause a syntax error. The console will show you the line number and a brief message.

Always check the line number and file name in the error message. Teach students to read errors instead of ignoring them.

7.3 Using Console for Debugging

If your logic is not working, you can insert console.log() statements to see values: 

let marks = 80;
let result;

if (marks >= 90) {
  result = "Excellent";
} else if (marks >= 75) {
  result = "Very Good";
} else {
  result = "Keep Trying";
}

console.log("Marks:", marks);
console.log("Result:", result);

If the result is not what you expect, console.log() helps you see where the logic went wrong.

Step 8 · Recap & Practice Tasks

8. Summary of This Module

Before moving to the next module, make sure your students can comfortably perform all tasks in this checklist.

8.1 Concept Checklist

  • Explain the role of JavaScript compared to HTML and CSS.
  • Describe where JavaScript runs (browser, Node.js conceptually).
  • Open the browser console and run simple JavaScript commands.
  • Create an HTML file and embed JavaScript using <script> tags.
  • Separate JavaScript into an external .js file and link it correctly.
  • Read basic error messages from the console and fix simple syntax mistakes.

8.2 Practice Exercises (For Students)

  1. Create a page that shows an alert:
    • Message: “Hello, I am starting JavaScript with Coder & AccoTax!”.
    • Also log the same message in the console.
  2. In script.js, declare two variables: firstName and courseName. Log a sentence like: “Sukanta is learning JavaScript.” using console.log().
  3. Create a function square(number) that logs the square of a number. Call it for values 2, 5, and 10.
  4. Intentionally create a syntax error (e.g., remove a parenthesis), reload the page, read the error in the console, and then fix it. This builds confidence with errors.
