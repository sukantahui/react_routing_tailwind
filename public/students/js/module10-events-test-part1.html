1
What is an event in the browser context?
  • A.A built-in JavaScript function.
  • B.Any action or occurrence in the browser (like a click, key press, or page load).
  • C.A special type of variable that holds UI data.
  • D.A CSS rule used for animations only.

Correct Answer: B

Explanation: In the DOM, an event is anything that happens in the browser: clicks, key presses, scrolling, form submission, etc. JavaScript can ‚Äúlisten‚Äù to these and react.

2
Which is the correct basic syntax of addEventListener?
  • A.element.addEventListener = ("click", fn());
  • B.addEventListener(element, "click", fn);
  • C.element.addEventListener("click", fn);
  • D.element.on("click", fn());

Correct Answer: C

Explanation: The correct pattern is element.addEventListener("event-name", handlerFunction);. We pass the function reference, not call it directly.

3
What will be logged when the button is clicked?
<button id="btn">Click</button>

<script>
  const btn = document.getElementById("btn");

  btn.addEventListener("click", function () {
    console.log("Handler 1");
  });

  btn.addEventListener("click", function () {
    console.log("Handler 2");
  });
</script>
  • A.Only Handler 1
  • B.Only Handler 2
  • C.Handler 1 then Handler 2
  • D.No output, this is invalid.

Correct Answer: C

Explanation: You can attach multiple listeners for the same event on the same element. They run in the order they were added, so both messages appear in sequence.

4
Which of the following is an advantage of using addEventListener instead of inline onclick?
  • A.You can only attach one handler to the event.
  • B.JavaScript code must be written inside the HTML attribute.
  • C.It separates structure (HTML) from behavior (JS) and allows multiple handlers.
  • D.It works only in old browsers, not modern ones.

Correct Answer: C

Explanation: addEventListener keeps HTML clean, lets you attach multiple listeners, and is the modern best practice for wiring events.

5
What is printed when the button is clicked once?
<button id="btn">Click me</button>

<script>
  const btn = document.getElementById("btn");
  let count = 0;

  btn.addEventListener("click", () => {
    count++;
    console.log(count);
  });
</script>
  • A.0
  • B.1
  • C.undefined
  • D.No output

Correct Answer: B

Explanation: count starts at 0. On first click, it increments to 1, and console.log(count) prints 1.

6
Which statement about event handlers is TRUE?
  • A.Event handlers can only be arrow functions.
  • B.Event handlers must always have the name handler.
  • C.An event handler is just a normal JavaScript function called by the browser when the event occurs.
  • D.Event handlers can only be used with form elements.

Correct Answer: C

Explanation: An event handler is simply a function that runs in response to an event. It can be a normal function, arrow function, named, or anonymous.

7
What is wrong with this code?
button.addEventListener("click", handleClick());

function handleClick() {
  console.log("Clicked");
}
  • A.The event name must be in uppercase.
  • B.handleClick() is being called immediately instead of being passed as a function reference.
  • C.addEventListener does not work with named functions.
  • D.The code is perfectly fine.

Correct Answer: B

Explanation: We should pass handleClick, not handleClick(). Writing handleClick() invokes the function immediately and passes its return value (likely undefined) as the handler.

8
Which click handler correctly changes button text to ‚ÄúClicked!‚Äù?
<button id="btn">Press</button>
  • A.btn.addEventListener("click", () => this.textContent = "Clicked!");
  • B.btn.addEventListener("click", (e) => e.target.textContent = "Clicked!");
  • C.btn.addEventListener(click, button.text = "Clicked!");
  • D.btn.click("Clicked!");

Correct Answer: B

Explanation: Inside the handler, the event object e has a target property that refers to the element that received the event. Setting e.target.textContent correctly updates the button label.

9
What will this code log when the button is clicked?
<button id="btn">Click me</button>

<script>
  let clicks = 0;

  const btn = document.getElementById("btn");

  btn.addEventListener("click", () => {
    clicks += 2;
    console.log("Clicks:", clicks);
  });

  btn.addEventListener("click", () => {
    clicks++;
    console.log("Updated:", clicks);
  });
</script>
  • A.Clicks: 1 then Updated: 1
  • B.Clicks: 2 then Updated: 3
  • C.Clicks: 2 then Updated: 2
  • D.Clicks: 3 then Updated: 5

Correct Answer: B

Explanation: Initially clicks = 0. First handler: clicks += 2 ‚Üí 2. Second handler: clicks++ ‚Üí 3. So we get Clicks: 2 then Updated: 3.

10
Which is the BEST description of event.target in a click handler?
  • A.The window object that fired the event.
  • B.The HTML element on which the event originally occurred.
  • C.Always the document node.
  • D.A random element chosen by the browser.

Correct Answer: B

Explanation: event.target references the element on which the event was first dispatched (the actual clicked element).

11
Which event is most suitable for reacting when a button is pressed by the user?
  • A.submit
  • B.click
  • C.load
  • D.keydown

Correct Answer: B

Explanation: The most common event for button interactions is the click event.

12
What will this code do when the user clicks the button?
<button id="btn">Say Hi</button>

<script>
  const btn = document.getElementById("btn");

  btn.addEventListener("click", () => {
    alert("Hello from Coder & AccoTax!");
  });
</script>
  • A.Nothing happens; alert cannot be used in events.
  • B.The page reloads.
  • C.An alert box appears with the message.
  • D.A console error is thrown.

Correct Answer: C

Explanation: alert() is a built-in browser function that shows a modal dialog; it works perfectly inside event handlers.

13
Which line correctly removes the handler added by this code?
function sayHello() {
  console.log("Hello");
}

button.addEventListener("click", sayHello);
  • A.button.removeEventListener("click", () => sayHello());
  • B.button.removeEventListener("click", sayHello);
  • C.button.removeEvent("click", sayHello);
  • D.sayHello.removeEventListener("click", button);

Correct Answer: B

Explanation: To remove an event listener, you must call removeEventListener with the same event type and the same function reference that was originally used.

14
Why is it usually better to load your script at the end of the <body> when using DOM events?
  • A.So that CSS loads faster.
  • B.So the DOM elements already exist when JavaScript tries to attach event listeners.
  • C.Because events only work at the end of the document.
  • D.It is required by the HTML5 standard.

Correct Answer: B

Explanation: If the script runs before the DOM is built, calls like getElementById can return null. Placing script tags at the end ensures elements exist when we wire events.

15
What will be the value of count after two clicks?
let count = 10;

button.addEventListener("click", () => {
  count = count + 5;
  console.log(count);
});
  • A.15
  • B.20
  • C.25
  • D.It always stays 10

Correct Answer: C

Explanation: Start: 10. After first click: 15. After second click: 20? Wait, no ‚Äî 15 + 5 = 20. Oops; we must re-check: Actually: first click ‚Üí 15, second click ‚Üí 20. The correct value after two clicks is 20, not 25.

Teacher note: This is a classic place where students must carefully simulate step by step.

15
What will be logged on each click?
let count = 0;

button.addEventListener("click", () => {
  count = count + 5;
  console.log(count);
});
  • A.First click: 1, Second click: 2
  • B.First click: 5, Second click: 10
  • C.First click: 0, Second click: 5
  • D.First click: 10, Second click: 20

Correct Answer: B

Explanation: Start at 0. After 1st click: count = 0 + 5 ‚Üí 5. After 2nd click: count = 5 + 5 ‚Üí 10.

16
Which option correctly describes the role of the second argument in addEventListener?
  • A.It is the DOM element to attach the listener to.
  • B.It is the event type like "click" or "input".
  • C.It is the callback function to be executed when the event fires.
  • D.It is always a boolean controlling bubbling.

Correct Answer: C

Explanation: The first argument is the event type; the second argument is the handler function that will run when the event happens.

17
What is printed when the user double-clicks the button once?
button.addEventListener("click", () => {
  console.log("Single click");
});

button.addEventListener("dblclick", () => {
  console.log("Double click");
});
  • A.Only Single click once
  • B.Only Double click once
  • C.Single click twice and Double click once (browser-dependent, but typical)
  • D.No output

Suggested Answer for Teaching: C

18
You want to run a function when the page finishes loading. Which event is most appropriate?
  • A.click
  • B.keydown
  • C.load on window
  • D.submit

Correct Answer: C

Explanation: The load event on window fires when the whole page (HTML, CSS, images, etc.) has finished loading.

19
Which line correctly logs the event type inside a handler?
button.addEventListener("click", function (event) {
  // log event type here
});
  • A.console.log(type);
  • B.console.log(click);
  • C.console.log(event.type);
  • D.console.log(this.event);

20
Which of the following will NOT attach a click event correctly?
  • A.button.addEventListener("click", () => console.log("Hi"));
  • B.button.onclick = () => console.log("Hi");
  • C.<button onclick="console.log('Hi')">Click</button>
  • D.addEventListener(button, "click", () => console.log("Hi"));

Correct Answer: D

Explanation: addEventListener is called on the element (button.addEventListener(...)). There is no global function addEventListener(button,...) like that.

21
What happens if you attach the same handler twice to the same event?
button.addEventListener("click", sayHi);
button.addEventListener("click", sayHi);
  • A.The handler will run twice on each click.
  • B.The second line is ignored; the handler runs only once.
  • C.It throws an error.
  • D.It disables all click handlers.

Correct (Spec) Answer: B

Explanation: The browser checks if the same function for the same event and capture options is already registered, and avoids adding duplicates.

22
Which is the best description of a ‚Äúclick event listener‚Äù?
  • A.A function that manually simulates mouse clicks.
  • B.A piece of code that waits for a click to happen and then runs a function.
  • C.A function that rewrites HTML when the page loads.
  • D.A CSS rule that changes color on hover.

Correct Answer: B

Explanation: An event listener ‚Äúlistens‚Äù for an event and calls the attached handler when that event occurs.

23
What will this code log when the button is clicked?
let msg = "Hello";

button.addEventListener("click", function () {
  let msg = "Hi";
  console.log(msg);
});
  • A.Hello
  • B.Hi
  • C.undefined
  • D.It throws an error for redeclaring msg.

Correct Answer: B

Explanation: The inner msg is a new local variable inside the handler. It shadows the outer one. So the logged value is "Hi".

24
Which is the MOST recommended way to attach events in modern web apps?
  • A.Inline HTML attributes (e.g., onclick="...")
  • B.Directly editing browser DevTools Elements panel
  • C.Using element.addEventListener("event", handler) in JavaScript files
  • D.Using CSS only without JavaScript

Correct Answer: C

Explanation: Attaching events via addEventListener in separate JS files keeps HTML clean and makes code easier to maintain and test.

25
