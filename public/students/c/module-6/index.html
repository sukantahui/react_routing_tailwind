Arrays in C

Arrays store multiple values of the same type in contiguous memory locations. In this module, you‚Äôll learn fixed-size arrays, memory layout, traversal techniques, common operations, pointer relationships, 2D matrices, dynamic arrays using malloc(), and best practices used in real projects.

1. What is an Array?

A fixed-size collection of similar elements stored sequentially in memory.

  • Stores multiple values of the same data type under a single name.
  • Size is fixed at declaration time (cannot grow or shrink at runtime).
  • Each element is accessed using an index starting from 0.
  • No automatic bounds checking ‚Äî accessing invalid index is a bug.
  • Fast random access: element at index i can be accessed in O(1) time.
  • Good cache performance due to contiguous storage.

üìå Arrays Store Elements in Contiguous Memory

The most important property of a C array is that all elements are stored in contiguous (continuous) memory locations. Each element sits immediately next to the previous one in RAM.

  • Address difference between two elements = sizeof(type).
  • Element address formula: base_address + index * sizeof(type).
  • Enables pointer arithmetic and fast iteration.
// Example: int usually takes 4 bytes
int a[5] = {10, 20, 30, 40, 50};

// Suppose &a[0] is 1000 (for explanation)
Index :   0    1    2    3    4
Value :  10   20   30   40   50

Address:
a[0] ‚Üí 1000
a[1] ‚Üí 1004   (1000 + 1 * 4)
a[2] ‚Üí 1008   (1000 + 2 * 4)
a[3] ‚Üí 1012
a[4] ‚Üí 1016
In interviews, ‚ÄúWhat does contiguous memory mean for arrays?‚Äù is a very common question. Emphasise that indices are implemented using this formula.

2. Declaring & Initializing Arrays

Different ways to create and initialize arrays with examples.

// 1. Declaration only ‚Üí contains garbage values (uninitialized local array)
int a[5];

// 2. Declaration + full initialization
int b[5] = {10, 20, 30, 40, 50};

// 3. Partial initialization ‚Üí remaining elements become 0
int c[5] = {1, 2}; 
// c = {1, 2, 0, 0, 0}

// 4. Automatic size detection
int d[] = {4, 5, 6};  // size = 3

// 5. All zeros (common trick)
int z[10] = {0};      // all 10 elements become 0

// 6. Initialization of char array
char vowels[5] = {'a','e','i','o','u'};
For global or static arrays, uninitialized elements are automatically set to 0. For local (automatic) arrays, uninitialized elements contain garbage values.
‚ùó Incorrect: int n; int arr[n]; is not valid in old C standards. It is a variable-length array (VLA) in C99, but not supported by all compilers. Prefer fixed size or dynamic memory (malloc) for portability.

3. Accessing Array Elements

Using index numbers to read/write values.

int numbers[3] = {10, 20, 30};

printf("%d\n", numbers[0]);  // prints 10
numbers[2] = 99;
printf("%d\n", numbers[2]);  // prints 99

// Reading values from user
for(int i = 0; i < 3; i++) {
    printf("Enter element %d: ", i);
    scanf("%d", &numbers[i]);
}
‚ùó Accessing numbers[3] or any index < 0 or ‚â• size is undefined behaviour. The compiler will not give an error, but your program may crash or print garbage.

4. Traversing Arrays

Looping through array elements in different ways.

Forward Traversal

int arr[5] = {1, 2, 3, 4, 5};

for(int i = 0; i < 5; i++) {
    printf("%d ", arr[i]);
}

Reverse Traversal

for(int i = 4; i >= 0; i--) {
    printf("%d ", arr[i]);
}

Traversal with While Loop

int i = 0;
while(i < 5) {
    printf("%d ", arr[i]);
    i++;
}
For teaching, show students how changing the loop condition or increment wrongly can create infinite loops or index out-of-range errors.

5. Common Array Operations

Standard operations performed using loops.

5.1 Sum & Average

int arr[5] = {10, 20, 30, 40, 50};
int sum = 0;

for(int i = 0; i < 5; i++)
    sum += arr[i];

float avg = sum / 5.0;

printf("Sum = %d, Average = %.2f", sum, avg);

5.2 Maximum Element

int max = arr[0];
for(int i = 1; i < 5; i++) {
    if(arr[i] > max)
        max = arr[i];
}
printf("Max = %d", max);

5.3 Linear Search

int key = 30, found = 0;
for(int i = 0; i < 5; i++) {
    if(arr[i] == key) {
        printf("Found at index %d\n", i);
        found = 1;
        break;
    }
}
if(!found) {
    printf("Not found\n");
}

5.4 Copying Arrays

int src[5] = {1, 2, 3, 4, 5};
int dest[5];

for(int i = 0; i < 5; i++) {
    dest[i] = src[i];
}

5.5 Reversing an Array (In-place)

int start = 0, end = 4;
while(start < end) {
    int temp      = arr[start];
    arr[start]    = arr[end];
    arr[end]      = temp;
    start++;
    end--;
}

6. 2D Arrays (Matrix)

2D arrays represent tables or matrices. C stores them in row-major order.

int matrix[2][3] = {
    {1, 2, 3},
    {4, 5, 6}
};

printf("%d\n", matrix[1][2]); // prints 6

Reading & Printing a 2√ó3 Matrix

int a[2][3];

printf("Enter 6 elements:\n");
for(int i=0; i<2; i++) {
    for(int j=0; j<3; j++) {
        scanf("%d", &a[i][j]);
    }
}

printf("Matrix is:\n");
for(int i=0; i<2; i++) {
    for(int j=0; j<3; j++) {
        printf("%d ", a[i][j]);
    }
    printf("\n");
}

Row-Major Memory Layout (Concept)

// Logical layout:
[ [1, 2, 3],
  [4, 5, 6] ]

// Row-major physical order in memory:
1  2  3  4  5  6
In C, matrices are widely used for marksheets, images (pixel grids), games (board representation), and mathematical algorithms.

7. Multi-Dimensional Arrays

Arrays of arrays of arrays‚Ä¶ used for 3D data like 3D grids or multiple matrices.

int arr3D[2][3][4];  
// Total elements = 2 * 3 * 4 = 24
// Access example:

Practically, 1D and 2D arrays cover 90% of beginner-level problems. 3D arrays appear in advanced simulations, graphics, or scientific code.

8. Arrays & Pointers Relationship

In expressions, the array name often behaves like a pointer to its first element.

  • a ‚Üí value is ‚Äúaddress of a[0]‚Äù in most expressions.
  • a == &a[0] (value is same, type is different).
  • Pointer arithmetic: *(a + i) is same as a[i].
int a[5] = {10,20,30,40,50};

printf("%p\n", (void*)a);       // base address
printf("%p\n", (void*)&a[0]);   // same address

printf("%d\n", *(a + 2));       // 30
printf("%d\n", a[2]);           // 30
‚ùó a is not a normal pointer variable. You cannot do: 
a = a + 1;   // ‚ùå Error: array type is not assignable
To move through an array, use a separate pointer: 
int *p = a;
p = p + 1;   // ‚úÖ allowed

9. Dynamic Arrays (malloc)

Used when array size is not known at compile time.

#include <stdio.h>
#include <stdlib.h>

int main() {
    int n;
    printf("Enter number of elements: ");
    scanf("%d", &n);

    // Allocate memory for n integers
    int *ptr = (int*) malloc(n * sizeof(int));
    if(ptr == NULL) {
        printf("Memory allocation failed\n");
        return 1;
    }

    printf("Enter %d integers:\n", n);
    for(int i = 0; i < n; i++) {
        scanf("%d", &ptr[i]);   // same as *(ptr + i)
    }

    printf("You entered:\n");
    for(int i = 0; i < n; i++) {
        printf("%d ", ptr[i]);
    }

    free(ptr);   // important to avoid memory leak
    return 0;
}
Dynamic arrays are very useful in real applications (reading data from file, variable number of students, etc.), where size is not fixed.

10. Common Mistakes & Best Practices

Professional tips that prevent bugs in array-based programs.

  • Avoid accessing beyond the array size (0 to size-1 only).
  • Always initialize arrays before use (especially local arrays).
  • While using loops, check starting index, ending condition, and increment carefully.
  • When using malloc, always check for NULL and call free() later.
  • Prefer meaningful names like marks[ ], prices[ ] instead of a[ ], b[ ].
Teacher Tip: Ask students to dry-run small array programs on paper. This builds very strong logic for loops and indexing.
