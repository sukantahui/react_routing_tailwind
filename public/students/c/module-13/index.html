Storage Classes & Scope in C

Storage classes decide how long a variable exists, where it is stored, and where it can be accessed. Mastering them is crucial for professional C programming.

1. What Are Storage Classes?

They define four things about a variable:

  • Scope: Where the variable is accessible
  • Lifetime: How long the variable exists in memory
  • Visibility: Where it can be used in the program
  • Memory Location: Where it is stored (stack, data segment, CPU register)

2. auto (Default)

Local variables inside functions.

  • Default storage class
  • Scope: Block/function
  • Lifetime: Until function ends
  • Stored in: Stack
void test() {
    auto int x = 10;  // same as: int x = 10;
}

3. static

Remembers value between function calls.

  • Scope: Local to block/function
  • Lifetime: Entire program
  • Stored in: Data segment
void counter() {
    static int count = 0;
    count++;
    printf("%d ", count);
}

int main() {
    counter(); // 1
    counter(); // 2
    counter(); // 3
}

4. extern

Used for global variables across multiple files.

  • Declares a variable without creating memory
  • Used in multi-file projects
  • Lifetime: Entire program
  • Stored in: Data segment
// File1.c
int a = 10;

// File2.c
extern int a;
printf("%d", a);

5. register

Request compiler to store variable in CPU register.

  • Fastest access
  • May or may not be stored in register (compiler decides)
  • Cannot take address using &
register int i;
for(i = 0; i < 10000; i++);

6. Scope Types

  • Block scope – variables inside braces
  • Function scope – labels
  • File scope – global variables
int x = 10; // file scope

void test() {
    int y = 20; // block scope
}

7. Lifetime Summary Table

Storage Class   Lifetime       Scope        Memory
--------------------------------------------------------
auto            Function       Block        Stack
static          Program        Local/Global Data Segment
extern          Program        Global       Data Segment
register        Function       Block        CPU Register
