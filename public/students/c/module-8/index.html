Module 8 — Functions

Functions in C

Functions allow breaking large programs into smaller, reusable blocks. Learn declaration, definition, return values, parameter passing, recursion, and more.

1. What is a Function?

A function is a block of code that performs a specific task.

  • Improves modularity
  • Avoids repetition
  • Makes code cleaner & easier to debug
  • C has built-in library functions & user-defined functions
int add(int a, int b); // function declaration

int main() {
    printf("%d", add(3, 4));
}

int add(int x, int y) {  // function definition
    return x + y;
}

2. Function Declaration (Prototype)

Tells compiler about function name, return type, parameters.

return_type function_name(parameter_list);
    
float average(int a, int b);

3. Function Definition

Actual block of code executed when the function is called.

float average(int x, int y) {
    return (x + y) / 2.0;
}

4. Calling a Function

Invoke a function and pass required parameters.

int main() {
    float ans = average(10, 20);
    printf("%f", ans);
}

5. Return Values

Functions may return a value using return.

int square(int n){
    return n * n;
}
    
printf("%d", square(5));  // 25

6. Call by Value vs Call by Reference

How data is passed into functions.

  • Call by value: copies the value
  • Call by reference: sends address (using pointers)
// Call by reference example
void update(int *x){
    *x = 100;
}

int main() {
    int a = 10;
    update(&a);
    printf("%d", a); // 100
}

7. Functions with Multiple Parameters

void greet(char name[], int age){
    printf("Hello %s, Age %d", name, age);
}

8. Recursive Functions

A function calling itself.

// Factorial using recursion
int fact(int n){
    if(n == 0) return 1;
    return n * fact(n - 1);
}

9. When to Use Functions?

  • Whenever logic repeats
  • Large problems can be divided into modules
  • Enhances readability and debugging
  • Makes team-based coding easier
